# 유지보수 요구사항 분석

개발 완료 시스템에 사용자가 추가 요구하는 내용을 분석해 도메인을 판단하고, 유사 이력을 조회해 `designer`에게 전달한다.

## 절차

1. **단계적 분석**: 사용자 요구사항을 단계적으로 분석해 어느 도메인(들)에 해당하는지 판단. 여러 도메인에 걸치면 도메인별로 나누어 정리.
2. **모호함 확인**: 요구사항 범위·대상 화면/기능·우선순위 등이 모호하거나 결정이 필요하면, 임의 판단하지 말고 사용자에게 재질문.
3. **유사 이력 조회**: `docs/06_maintenance/CLAUDE.md` 인덱스를 먼저 확인. 인덱스에서 관련 있어 보이는 항목만 골라 해당 `report.md`를 필요한 만큼만 열람. (인덱스 거치지 않고 전체 이력 폴더 순회 금지 — 토큰 최소화)
4. **전달**: 분석한 도메인·요구사항과, 유사 이력이 있으면 [도메인 / 요구사항 / 해결 방법] 위주로 간략히 정리해 `designer`에게 `SendMessage`로 함께 전달.

## 주의사항

- 요구사항에 없는 내용은 추측·확장하지 않는다.
- 이력 조회는 반드시 `docs/06_maintenance/CLAUDE.md` 인덱스로 범위를 좁힌 뒤 필요한 report만 읽는다(전수 조사 금지).
