# 기능 명세서 작성

요구사항 정의서를 바탕으로 각 기능의 동작을 **직관적이고 간결하게** 명세한다.

## 작성 원칙

- 요구사항 정의서의 요구사항(REQ-ID)과 기능(FEAT-ID)을 추적 가능하게 연결한다.
- 요구사항에 명시된 동작만 기술한다. 추측·확장은 금지한다.
- 각 기능의 인수 기준은 EARS 표기법으로 작성하며, 예외/오류 처리(Unwanted Behaviour)를 반드시 포함한다.

## 양식

표준 양식은 [references/template.md](references/template.md)를 그대로 사용한다.

## 절차

1. 기능 목록 표를 작성하고 각 기능을 요구사항(REQ-ID)에 매핑한다.
2. 기능별 상세(설명·입출력·처리 흐름·인수 기준·예외 처리)를 작성한다.
3. 완성된 문서를 `docs/01_analyze/feature/{domain}.md`에 저장한다.
