# 역할 설계 (역할별 사용 범위 · 페르소나)

요구사항 정의서를 기반으로 사용자 역할을 정의한다.

## 작성 원칙

- **요구사항 정의서를 기반으로 사용자 역할 종류를 추출한다** (관리자 포함).
- 역할별로 **접근 가능한 화면과 API 요청**을 정의한다.
- 역할별 **페르소나**를 함께 기술한다.

## 접근 제어 규칙

- 로그인이 필요한 화면으로 이동하거나 로그인이 필요한 요청인 경우:
  1. Access Token의 `role`을 추출한다.
  2. 해당 역할이 접근 가능한 화면/요청인지 검증한다.
  3. **권한 부족 → 403**, **권한 충분 → 정상 수행**.
- **화면 이동마다 Backend로 API 요청을 보내 해당 화면에 대한 권한이 있는지 확인**해야 한다.

## 산출물 저장 위치

- 역할별로 `docs/02_plan/security/authorization/{역할명}.md`

표준 양식은 [references/template.md](references/template.md)를 사용한다.
