# 역할 설계 (역할별 사용 범위 · 페르소나)

요구사항 정의서 기반 사용자 역할 정의.

## 작성 원칙

- **요구사항 정의서 기반 사용자 역할 종류 추출** (관리자 포함).
- 역할별 **접근 가능 화면과 API 요청** 정의.
- 역할별 **페르소나** 함께 기술.

## 접근 제어 규칙

- 로그인 필요 화면 이동 또는 로그인 필요 요청인 경우:
  1. Access Token의 `role` 추출.
  2. 해당 역할이 접근 가능한 화면/요청인지 검증.
  3. **권한 부족 → 403**, **권한 충분 → 정상 수행**.
- **화면 이동마다 Backend로 API 요청 보내 해당 화면 권한 확인** 필요.

## 산출물 저장 위치

- 역할별로 `docs/02_plan/security/authorization/{역할명}.md`

## 양식

표준 양식은 [references/template.md](references/template.md) 사용.
