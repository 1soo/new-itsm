# 요구사항 정의서 작성

도메인별 사용자 요구사항을 **직관적이고 간결한** 요구사항 정의서로 작성.

## 작성 원칙

- 요구사항에 명시된 내용만 기술. 추측·확장 금지.
- 각 요구사항에 고유 ID(`REQ-{도메인약어}-{번호}`) 부여해 추적 가능하게 함.
- 인수 기준은 EARS 표기법으로 작성. (6가지 패턴은 분석 에이전트 프롬프트 표 참조)
- 한 요구사항은 여러 인수 기준 가질 수 있음.

## 양식

표준 양식은 [references/template.md](references/template.md) 그대로 사용.

## 절차

1. 도메인 개요와 범위(포함/미포함) 먼저 정리.
2. 요구사항 목록 표 작성.
3. 각 요구사항별 EARS 인수 기준 작성.
4. 완성 문서를 `docs/01_analyze/prd/{domain}.md`에 저장.
