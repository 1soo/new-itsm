# UI/UX 개발 표준 (Design System)

프레임워크와 무관하게 적용되는 UI/UX 구현 표준.

## 1. 디자인 토큰

화면 설계서의 색상 팔레트를 코드 토큰(CSS 변수/테마)으로 옮겨 관리한다.

| 토큰 그룹 | 내용 |
|-----------|------|
| Color | base + variation, neutral, 상태색(성공/경고/오류/정보). 설계서 팔레트 준수 |
| Typography | font-family, 크기 스케일, 굵기, line-height |
| Spacing | 간격 스케일(예: 4/8/12/16/24/32) |
| Radius | 모서리 반경 스케일 |
| Elevation | 그림자 단계 |

- 컴포넌트는 토큰만 참조한다. 색상·크기 하드코딩 금지.

## 2. 레이아웃 · 그리드

- 페이지 골격(헤더/사이드바/콘텐츠/푸터)과 그리드 컬럼 규칙 정의.
- 반응형 breakpoint 기준(mobile / tablet / desktop)을 고정하고 전 화면에 일관 적용.

## 3. 공통 컴포넌트 규격

- `docs/02_plan/screen/common.md`의 헤더·푸터·사이드바를 공통 컴포넌트로 구현.
- 기본 요소(버튼·입력·표·모달 등)는 **상태(variants)** 를 정의: 기본 / hover / focus / disabled / loading.
- shadcn 컴포넌트를 기반으로 하되 토큰으로 테마를 맞춘다.

## 4. 화면 상태 UX

각 화면은 아래 상태를 일관되게 표현한다.

- 로딩(스켈레톤/스피너), 빈 상태(empty), 오류 상태, `404`(설계서 `error_404.md`).

## 5. 인터랙션 · 피드백

- 폼 검증 메시지(인라인), 토스트/알림, 확인 다이얼로그의 패턴을 통일.
- 비동기 동작 중 버튼 로딩·중복 제출 방지.

## 6. 접근성 (a11y)

- 색 대비 기준(WCAG AA) 충족, 색상만으로 정보 전달 금지.
- 키보드 내비게이션 및 포커스 표시 제공.
- 시맨틱 태그 / 필요한 ARIA 속성 부여, 이미지 대체 텍스트.
