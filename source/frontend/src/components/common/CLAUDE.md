# CLAUDE.md

프로젝트 공통 컴포넌트. `ui/`의 프리미티브를 조합해 도메인 무관한 재사용 패턴(상태 표시·목록/상세 셸·차트·폼)을 제공하며, 프레젠테이션만 담당하고 데이터·권한·라우팅은 기능 레이어(FE)가 주입. 화면 설계서(common.md SCR-COM-*)의 공통 UX 구현.

## 파일
- `index.ts` — 공통 컴포넌트 배럴. 기능 화면은 이 모듈에서 import.
- `status-badge.tsx` — 상태 배지. tone(success/warning/danger/info/muted) + 라벨을 FE가 주입. `StatusTone` 타입 제공.
- `priority-badge.tsx` — 우선순위 배지(P1~P4 → Danger~Muted). `Priority` 타입 제공.
- `toast.ts` — 토스트 helper(success/error/info). 외부 API 불변, 내부는 SweetAlert2 toast mixin(우상단, `index.css`의 `itsm-swal-toast*` 클래스로 시맨틱 토큰 스타일링, 2026-07-12 SweetAlert2 도입).
- `confirm-dialog.tsx` — 파괴적/비가역 동작 확인 다이얼로그. 선언형 API(props) 불변, 내부에서 `open` 변화를 감지해 SweetAlert2를 명령형으로 호출하는 래퍼(`index.css`의 `itsm-swal-popup`/`itsm-swal-btn*` 클래스, 2026-07-12 SweetAlert2 도입). 확인 시에만 `onConfirm` 호출, 닫힘은 호출측이 `onOpenChange`로 제어. `confirmLabel`/`cancelLabel` 미지정 시 `common:dialog.confirm`/`common:dialog.cancel`로 폴백.
- `modal.tsx` — 범용 모달(폼/상세 등 비파괴 콘텐츠). 파괴적 확인은 ConfirmDialog 사용. SweetAlert2 도입 대상에서 제외되어 기존 Radix Dialog 그대로 유지.
- `user-guide-content.tsx` — 사용자 가이드 전용 화면(`/guide`, SCR-COM-012)의 섹션 콘텐츠. `UserGuideOverview`(개요 1절, Markdown 순차 렌더링)·`UserGuideDomainSection`(11개 도메인 아코디언)·`UserGuideRoleSection`(16개 역할 아코디언, `myRoles?: string[]`로 "내 역할" 상단 고정+기본 펼침) 3개를 개별 export. 콘텐츠는 `docs/01_analyze/feature/user-guide-content.md`(한국어)·`user-guide-content.en.md`(영어, 2026-07-12 다국어 지원) 원문을 가공 없이 이관한 정적 데이터(`react-markdown`으로 굵게 등 인라인 서식 렌더링), `i18n.language`에 따라 언어별 콘텐츠 세트 선택. 페이지 레이아웃(문서 헤더·TOC)은 담당하지 않음 — FE(`features/guide/GuidePage.tsx`)가 조립.
- `multi-select.tsx` — 다중 선택(Popover + 체크 리스트). 필터바용. `MultiSelectOption` 타입 제공. `placeholder` prop 미지정 시 `common:multiSelect.defaultPlaceholder`로 폴백, 선택 개수·선택 해제 aria-label·빈 목록 문구는 `common:multiSelect.*` 키(2026-07-12 다국어 지원).

- `dynamic-form-builder.tsx` — SRM 전용 관리자용 그리드 폼 설계 컴포넌트(SCR-SRM-007 "Form 설정" 팝업, form.io 완전 제거 → 자체 8×n 그리드로 전면 재작성 — 신규 빌드 의존성 없음, 네이티브 Pointer Events 기반). 팝업 내부를 **좌 팔레트(10종: text/textarea/select/radio/checkbox/date/file + 값 입력 없는 정적 컴포넌트 `label`/`guide-text`/`guide-file`) / 중앙 8칸 그리드 캔버스 / 우 전체 레이아웃 미리보기** 3분할로 항상 동시 표시한다(탭 전환 아님, 2026-07-18 후속 유지보수 요청 — 기존 `CatalogManagePage` 외부 45% 축소 pre-view를 이 팝업 안으로 이전). 우측 미리보기는 `dynamic-form-renderer.tsx`의 `DynamicFormRenderer`를 `disabled`+`hideFooter`로 재사용해 편집 중(적용 전) `components` 상태를 실시간 반영, 축소 비율은 `form-schema.ts`의 `GRID_PREVIEW_SCALE`(0.45) 공유. 캔버스는 스크롤 가능, 배치·리사이즈 1칸 단위 스냅, 겹침 배치는 차단+인라인 경고 "이미 배치된 컴포넌트와 겹칩니다" 1.5초 노출. 컴포넌트는 폭 1~2·높이 1~2칸(`textarea`만 높이 무제한, `gridMaxHeight`, `label`/`guide-text`/`guide-file`는 다른 비-textarea와 동일하게 2 제약). 카드가 1×1 크기면 캡션이 아이콘만 표시되고(텍스트 숨김), 1×1보다 크면 아이콘+텍스트를 함께 표시한다. 카드 hover 시 설정 아이콘(`Settings2`, Radix Popover, 팝업 폭 420px)·삭제 아이콘(`Trash2`) 노출 — `label`/`guide-text`는 "표시 텍스트"+정렬(동일 UI, `text`/`textAlign` 필드 공유), `guide-file`은 "첨부 파일"(선택, `FileReader.readAsDataURL()`로 base64 변환)만, 나머지 7종(입력 컴포넌트, `label`/`labelAlign` 속성 없음)은 input 폭%/정렬/Placeholder(`hasPlaceholderUi(type)`인 text/textarea/select/date/file만 노출)/기본값/읽기전용, 필수 여부, Validation 정규식, (select/radio/checkbox) 콤마 구분 옵션+"CI 연계" 라디오 자리(실제 동작 없음)를 편집한다. radio/checkbox는 추가로 옵션 배치 방향(가로/세로 토글, `hasOptionsLayoutUi(type)`만 노출)·옵션 간 여백 3단계(좁게/보통/넓게)를 편집한다(select 제외). 카드 캡션은 `label`/`guide-text` 타입이면 `component.text`, 나머지는 자기 타입의 팔레트 이름을 보여준다. Content 설정 팝업 하단에 편집 중인 컴포넌트 단독의 실시간 미리보기(팝업 전용 로컬 state)를 두며, 렌더러(`dynamic-form-renderer.tsx`)가 export하는 `GridComponentBody`를 그대로 재사용한다(로직 중복 구현 없음). `initialSchema`로 편집 모드 진입(로컬 state, 자동저장 없음), 하단 적용/취소 버튼 — 적용 시 `onApply(schema)`로 최신 그리드 스키마를 상위에 전달하고 팝업을 닫는 것은 호출측(`CatalogManagePage`)이 담당. 내부 UI 텍스트는 하드코딩 한국어(관리자 전용 팝업, 프로젝트 i18n 전환 범위 밖).
- `dynamic-form-renderer.tsx` — SRM 전용 요청자용 그리드 폼 렌더 컴포넌트(SCR-SRM-002, form.io 완전 제거 → 자체 구현). `schema.components`의 `position`/`size`를 CSS Grid(8열, 행 높이는 `GRID_ROW_HEIGHT_PX` 고정)로 그대로 배치해 렌더링(select/radio/checkbox는 `options`를 콤마로 파싱, 옵션 많으면 자동 줄바꿈+셀 내부 스크롤; radio/checkbox는 추가로 `optionsDirection`(기본 row=가로, wrap)/`optionsGap`(기본 1, gap-1~gap-3 매핑) 반영, 2026-07-18 후속 유지보수 요청). 컴포넌트별 렌더 로직은 `GridComponentBody`(component/value/onChange/disabled props, 그리드 포지셔닝 없이 본문만)로 export되어 메인 그리드와 빌더 팝업의 3분할 전체 미리보기·개별 실시간 미리보기가 동일 로직을 공유한다(로직 중복 구현 금지). 입력 컴포넌트는 `<label>` 캡션을 렌더링하지 않는다(값 입력 없는 `type=label` 컴포넌트가 대신 `text`/`textAlign`으로 정적 텍스트를 렌더, `type=guide-text`는 동일 구조에 안내 아이콘(`Info`)을 동반해 시각적으로 구분, `type=guide-file`은 첨부 파일 다운로드 링크(`file.dataUrl`)만 렌더(미첨부 시 빈 상태 안내) — 셋 다 입력과 `for`/`aria-label` 연결 없음, `guide` 통합 타입은 폐기됨). `date`/`file`은 입력 박스(`widthPercent`/`align` 적용, border/rounded)+박스 우측 끝 아이콘(`Calendar`/`FileIcon`) 형태로 표시하고, 박스 전체(텍스트 영역+아이콘)가 하나의 클릭 영역이라 어디를 클릭해도 숨겨진 `sr-only` 네이티브 input의 피커(`showPicker()`, 미지원 시 `click()` 폴백)/파일선택을 연다. 박스 안 텍스트는 값이 있으면 값(날짜 `yyyy-MM-dd`/파일명), 없으면 `component.input?.placeholder` 또는 유형별 기본 안내("날짜를 선택하세요"/"파일을 선택하세요")를 표시. `text`/`textarea`/`select`도 `input.placeholder`를 반영(select는 `SelectValue placeholder`, radio/checkbox는 placeholder UI 자체가 없음). 유효성 검증은 필드별 인라인이 아니라 제출 클릭 시 `components` 배열 순서대로 순회해 **첫 번째 위반 1건만** 단일 `formError` 문자열로 폼 하단에 표시(여러 오류 동시 표시 없음, `type=label`/`type=guide-text`/`type=guide-file`은 검증·제출 데이터 대상 제외, 서버 재검증도 동일 계약 — BE `FormSubmissionValidator`, api_spec/common.md 0-2절). `file` 타입은 선택한 파일을 `FileReader.readAsDataURL()`로 base64 데이터 URL 문자열로 변환해 값에 저장한다(api_spec/common.md 0-2절 "제출 데이터에 파일을 base64 문자열로 인라인 포함"). 제출/취소 버튼은 컴포넌트 자체 하단 푸터(우측 정렬, 취소→제출 순) — `onCancel?`(미지정 시 취소 버튼 숨김)/`submitLabel?`/`cancelLabel?`(기본값 "제출"/"취소", 도메인별 i18n 텍스트는 호출측이 전달)/`hideFooter?`(빌더 팝업 3분할 미리보기·pre-view 재사용 전용, `CatalogManagePage`의 외부 pre-view는 폐기됨) prop.

- `form-schema.ts` — SRM 전용 그리드 스키마 타입과 상수(`GRID_COLUMNS`=8, `GRID_ROW_HEIGHT_PX`, `GRID_PALETTE_TYPES`(10종), `GRID_PREVIEW_SCALE`(0.45, 빌더 팝업 3분할 미리보기 공유), `EMPTY_GRID_SCHEMA`), 헬퍼(`hasGridOptions`/`hasPlaceholderUi`/`hasOptionsLayoutUi`/`gridMaxHeight`). `hasPlaceholderUi(type)`는 text/textarea/select/date/file만 true(radio/checkbox는 placeholder UI 미노출). `hasOptionsLayoutUi(type)`는 radio/checkbox만 true(select 제외 — 옵션 배치 방향·여백 설정 UI 노출 대상). `GridComponent`는 판별 유니온으로 정의됨 — 7개 입력 타입(`GridInputComponentType`)은 `label`/`labelAlign` 없이 `input`(`placeholder` 포함)/`validation`/`options`/`ciLinked`/`optionsDirection`(row/column, 기본 row)/`optionsGap`(1~3, 기본 1)만 갖는 `GridInputComponent`, `type="label"`은 `text`/`textAlign`만 갖는 `GridLabelComponent`, `type="guide-text"`는 `GridLabelComponent`와 구조 동일한 `GridGuideTextComponent`(안내 텍스트, 정적), `type="guide-file"`은 `file: {name, dataUrl} | null`만 갖는 `GridGuideFileComponent`(첨부 가이드 파일, 정적)로 분리(`GridComponent = GridInputComponent | GridLabelComponent | GridGuideTextComponent | GridGuideFileComponent`). 기존 `guide`(안내 텍스트+첨부 파일 통합) 타입은 2026-07-18 후속 유지보수 요청으로 완전히 폐기되고 이 2종으로 대체됨(운영 데이터 없어 마이그레이션 불필요). 하단 레거시 구획(`FormFieldSchema`/`FormFieldType`/`FormValues`/`FormErrors`/`validateForm`/`hasOptions`)은 `field-builder.tsx`/`dynamic-form.tsx`(ESM 전용) 계약이라 SRM 전환과 무관하게 유지한다.
