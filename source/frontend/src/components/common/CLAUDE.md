# CLAUDE.md

프로젝트 공통 컴포넌트. `ui/`의 프리미티브를 조합해 도메인 무관한 재사용 패턴(상태 표시·목록/상세 셸·차트·폼)을 제공하며, 프레젠테이션만 담당하고 데이터·권한·라우팅은 기능 레이어(FE)가 주입. 화면 설계서(common.md SCR-COM-*)의 공통 UX 구현.

## 파일
- `index.ts` — 공통 컴포넌트 배럴. 기능 화면은 이 모듈에서 import.
- `status-badge.tsx` — 상태 배지. tone(success/warning/danger/info/muted) + 라벨을 FE가 주입. `StatusTone` 타입 제공.
- `priority-badge.tsx` — 우선순위 배지(P1~P4 → Danger~Muted). `Priority` 타입 제공.
- `toast.ts` — 토스트 helper(success/error/info). 외부 API 불변, 내부는 SweetAlert2 toast mixin(우상단, `index.css`의 `itsm-swal-toast*` 클래스로 시맨틱 토큰 스타일링, 2026-07-12 SweetAlert2 도입).
- `confirm-dialog.tsx` — 파괴적/비가역 동작 확인 다이얼로그. 선언형 API(props) 불변, 내부에서 `open` 변화를 감지해 SweetAlert2를 명령형으로 호출하는 래퍼(`index.css`의 `itsm-swal-popup`/`itsm-swal-btn*` 클래스, 2026-07-12 SweetAlert2 도입). 확인 시에만 `onConfirm` 호출, 닫힘은 호출측이 `onOpenChange`로 제어. `confirmLabel`/`cancelLabel` 미지정 시 `common:dialog.confirm`/`common:dialog.cancel`로 폴백.
- `modal.tsx` — 범용 모달(폼/상세 등 비파괴 콘텐츠). 파괴적 확인은 ConfirmDialog 사용. SweetAlert2 도입 대상에서 제외되어 기존 Radix Dialog 그대로 유지.
- `user-guide-content.tsx` — 사용자 가이드 전용 화면(`/guide`, SCR-COM-012)의 섹션 콘텐츠. `UserGuideOverview`(개요 1절, Markdown 순차 렌더링)·`UserGuideDomainSection`(11개 도메인 아코디언)·`UserGuideRoleSection`(16개 역할 아코디언, `myRoles?: string[]`로 "내 역할" 상단 고정+기본 펼침) 3개를 개별 export. 콘텐츠는 `docs/01_analyze/feature/user-guide-content.md`(한국어)·`user-guide-content.en.md`(영어, 2026-07-12 다국어 지원) 원문을 가공 없이 이관한 정적 데이터(`react-markdown`으로 굵게 등 인라인 서식 렌더링), `i18n.language`에 따라 언어별 콘텐츠 세트 선택. 페이지 레이아웃(문서 헤더·TOC)은 담당하지 않음 — FE(`features/guide/GuidePage.tsx`)가 조립.
- `multi-select.tsx` — 다중 선택(Popover + 체크 리스트). 필터바용. `MultiSelectOption` 타입 제공. `placeholder` prop 미지정 시 `common:multiSelect.defaultPlaceholder`로 폴백, 선택 개수·선택 해제 aria-label·빈 목록 문구는 `common:multiSelect.*` 키(2026-07-12 다국어 지원).

- `dynamic-form-builder.tsx` — SRM 전용 관리자용 그리드 폼 설계 컴포넌트(SCR-SRM-007 "Form 설정" 팝업, form.io 완전 제거 → 자체 8×n 그리드로 전면 재작성 — 신규 빌드 의존성 없음, 네이티브 Pointer Events 기반). 팝업 내부는 **좌 팔레트(9종: text/textarea/select/radio/checkbox/date/file + 값 입력 없는 정적 컴포넌트 `guide-text`/`guide-file`) / 우 8칸 그리드 캔버스** 2분할이다. 팔레트 최상단에 "라벨 추가" 버튼 + 생성된 라벨 칩 스트립(팝업 타이틀과 팔레트+캔버스 사이)이 있다. **캔버스=미리보기 통합**: 각 캔버스 카드가 컴포넌트 유형 아이콘·캡션 텍스트·점선 placeholder 대신 그 컴포넌트가 실제 렌더링될 모습을 카드 콘텐츠 영역에 그대로 보여준다(렌더러가 export하는 `GridComponentBody` 재사용). 이 렌더링은 **순수 시각적 표현**이며 콘텐츠 영역에 `pointer-events-none`을 적용해 값 입력·상호작용을 차단한다(카드 자체는 여전히 드래그 이동 대상이라 클릭이 카드 wrapper로 그대로 전파됨). 카드 hover 시 노출되는 설정(`Settings2`)·삭제(`Trash2`) 버튼은 렌더링된 콘텐츠 위에 절대 위치 오버레이(반투명 배경+블러)로 배치한다(별도 헤더 바 없음). 캔버스는 스크롤 가능. **배치(2026-07-18 유지보수 요청 4차)**: 팔레트 항목 클릭(첫 빈 칸 자동 배치, 기존)과 드래그앤드롭(팔레트 버튼 `onPointerDown`으로 시작, 이동 임계값(`DRAG_THRESHOLD_PX`) 초과 시에만 드래그로 간주해 클릭과 구분, 1칸 스냅 미리보기 고스트, 드롭 시 겹치면 차단+경고, 캔버스 밖 드롭 취소) 둘 다 지원한다. 리사이즈도 1칸 단위 스냅, 겹침 배치는 차단+인라인 경고 "이미 배치된 컴포넌트와 겹칩니다" 1.5초 노출. 컴포넌트는 폭 1~2칸, 높이 상한은 유형별 세분화(`gridMaxHeight`: text/date/file/select/guide-file은 1 고정, radio/checkbox/guide-text는 1~2, textarea는 무제한) — 변경 전 이미 2칸이던 컴포넌트는 강제 축소하지 않음(재편집 시에만 새 상한 적용). 1×1의 렌더링 축소 표시는 `GridComponentBody`가 이미 갖는 오버플로우 처리를 그대로 재사용. Content 설정 팝업에서는 `guide-text`가 "안내 텍스트"+정렬(9방향), `guide-file`은 "첨부 파일"(선택, `FileReader.readAsDataURL()`로 base64 변환)만, 나머지 7종(입력 컴포넌트)은 input 폭%/정렬(9방향)/Placeholder(`hasPlaceholderUi(type)`인 text/textarea/select/date/file만 노출)/기본값/읽기전용, 필수 여부, (`hasRegexUi(type)` — `text`만 true) Validation 정규식, (select/radio/checkbox) 콤마 구분 옵션+"CI 연계" 라디오 자리(실제 동작 없음)를 편집한다. **정렬은 가로×세로 3×3 앵커 그리드 위젯(`AnchorGridToggle`)** 하나로 통일(입력 7종은 `input.align`+`input.verticalAlign`, `guide-text`는 `textAlign`+`textVerticalAlign`, 기본값 모두 top) — 기존 가로 3버튼 `AlignToggle`은 폐기. radio/checkbox는 추가로 옵션 배치 방향(가로/세로 토글, `hasOptionsLayoutUi(type)`만 노출)·옵션 간 여백 3단계(좁게/보통/넓게)를 편집한다(select 제외). **Content 설정 팝업(`ComponentSettingsPopover`)**: 카드의 설정 아이콘 클릭 시 여는 미니 팝업으로, 2026-07-18 유지보수 요청 7차로 **Radix Popover에서 공용 `Modal`(Radix Dialog)로 전환**됐다 — 더 이상 트리거(설정 아이콘) 근처에 상대 배치되지 않고, 라벨 생성/수정 팝업과 동일한 화면 뷰포트 기준 고정 좌표(가로 정중앙, 세로는 `MINI_POPUP_POSITION_CLASS`(`top-[42%]`) 공용 상수로 정중앙보다 살짝 위)에 뜬다(폭 420px, `max-h-[80vh] overflow-y-auto`로 스크롤). **라벨(태그) 지정**: 입력 7종+guide-text+guide-file 공통 `labelId` Select(후보는 팝업 밖 최상위 `labels`, "없음" 포함, 최대 1개 참조)가 팝업 **최상단**(타입별 설정 항목보다 앞, 7차로 하단→최상단 이동)에 있다(prop drilling: `DynamicFormBuilder`→`BuilderComponentCard`→`ComponentSettingsPopover`). Content 설정 팝업에는 렌더링 미리보기가 없다(**2026-07-18 유지보수 요청 6차로 완전 제거** — 캔버스 카드 자체 렌더링과 SCR-SRM-007 축소 미리보기(A1)로 충분하다고 판단해 중복 기능 정리, `previewValue` state도 함께 제거됨. `GridComponentBody` import는 `BuilderComponentCard`의 캔버스 카드 렌더링에서 계속 쓰이므로 유지). **라벨(태그) 관리(2026-07-18 유지보수 요청 4차 신규, 5·6차로 세부 보정)**: 그리드 직접 배치형 `label` 컴포넌트는 완전히 폐기되고, 다른 컴포넌트에 선택적으로 부여하는 태그로 대체됐다. 팔레트 상단 "라벨 추가" 버튼 → 미니 `Modal`(텍스트/글자색/"테두리 없음" 체크박스(5차 신규, 체크 시 `showBorder: false`)/테두리색 `<input type="color">`(6차로 `showBorder=false`일 때 이 입력 UI 자체를 숨김, 체크 해제 시 다시 표시 — 값 자체는 숨겨져도 폼 상태에 보존)) → `labels` state(`id`/`text`/`textColor`/`borderColor`/`showBorder`)에 push. 칩 스트립에서 칩 클릭 시 값 프리필(체크박스 상태 포함)로 동일 미니 팝업 재오픈(수정), 칩의 × 버튼으로 라벨 삭제(참조하던 컴포넌트의 `labelId`를 null로 일괄 해제, 컴포넌트 자체는 유지). 칩 스트립 자체의 테두리+틴트 배경 스타일은 `showBorder`와 무관하게 항상 표시(변경 없음). 캔버스 위에는 라벨별 경계 테두리 오버레이 — **참조 컴포넌트가 1개 이상이면 오버레이 자체는 항상 렌더링**(6차로 렌더링 여부와 테두리 스타일 여부를 분리 — 5차 구현이 `showBorder=false`일 때 오버레이 전체를 안 그려서 텍스트까지 함께 사라지던 결함을 수정, 0개면 렌더링 없음). 좌표 최소/최대로 계산한 사각형을 CSS Grid 라인 좌표(`gridColumn`/`gridRow`, span 아님)로 배치하고 음수 마진(`-m-[2px]`)으로 각 변 2px(전체 4px) 바깥으로 시각적 확장하되, 테두리 관련 클래스(`border-2`)·`borderColor` 스타일은 `showBorder !== false`일 때만 적용(6차). 라벨 텍스트는 `showBorder`와 무관하게 항상 표시하며, **legend 스타일(6차)** — `fieldset`/`legend`처럼 테두리 선 위에 걸치도록 `absolute -top-[7px]`로 절반 정도 위로 올리고 `bg-background/80`(불투명도 있는 배경)로 아래 선을 가림, 색상은 `textColor`(`pointer-events-none`, 이동/리사이즈 중인 컴포넌트는 `movePreview`/`resizePreview` 반영값 사용) — 매 렌더 파생 계산, 별도 effect 없음. 이 오버레이는 Content 설정 팝업·요청 제출 폼·A1 축소 미리보기에는 나타나지 않는다(캔버스 전용). `initialSchema`로 편집 모드 진입(로컬 state, 자동저장 없음), 하단 적용/취소 버튼 — 적용 시 `onApply({components, labels})`로 최신 그리드 스키마를 상위에 전달하고 팝업을 닫는 것은 호출측(`CatalogManagePage`)이 담당. 내부 UI 텍스트는 하드코딩 한국어(관리자 전용 팝업, 프로젝트 i18n 전환 범위 밖).
