# 유지보수 이력 — SRM (서비스 카탈로그/서비스 요청)

> 유지보수 일시: 20260718-234334 · 도메인: srm

## 1. 요구사항

직전 유지보수(20260718-222005, 라벨 경계 테두리 보정 5차)에 이은 후속 보정 요청 4건이다.
설정 미니 팝업(Content 설정 팝업)에서 미리보기를 없앤다.
라벨 텍스트 위치를 박스 안(내부 좌측 상단)이 아니라 박스 테두리 선상에 걸치도록 수정한다.
fieldset의 legend처럼 테두리 라인 위에 걸쳐 보이는 형태로 변경한다.
테두리가 없어도(showBorder=false) 라벨 텍스트는 계속 보이도록 수정한다.
현재 텍스트가 테두리 렌더링에 종속되어 함께 숨겨지고 있어, 텍스트 표시와 테두리 표시를 분리한다.
라벨 설정 팝업에서 "테두리 없음" 체크박스 상태에 따라 테두리 색 설정 UI 자체의 렌더링 여부를 결정한다.
체크(테두리 없음)하면 테두리 색 설정 UI를 아예 렌더링하지 않고, 체크 해제하면 렌더링한다.

## 2. 해결 방법

### DB

변경 없음.

### BE

변경 없음.

### FE

`ComponentSettingsPopover` 최하단의 컴포넌트별 개별 실시간 미리보기 블록(2차 건 신규분, `GridComponentBody`+`previewValue` state)을 완전히 제거했다.
캔버스 카드 자체의 "캔버스=미리보기 통합"(비활성 시각적 표현)과 SCR-SRM-007 축소 미리보기(A1, 4차)는 이번 요청과 무관해 그대로 유지했다.
라벨 경계 테두리 오버레이의 텍스트 위치를 박스 안쪽 좌측 상단에서, fieldset/legend처럼 테두리 선 위에 걸치는 스타일로 변경했다(텍스트를 절반 정도 위로 올리고 배경으로 아래 테두리 선을 가리는 방식).
오버레이 렌더링 여부(참조 컴포넌트 1개 이상이면 항상 렌더링)와 테두리 스타일 적용 여부(showBorder 값으로만 제어)를 분리했다 — 직전 5차 구현이 `showBorder=false`일 때 오버레이 전체를 안 그려서 텍스트까지 함께 사라지던 결함을 수정했다.
라벨 생성/수정 미니팝업에서 "테두리 없음" 체크 상태에 따라 테두리색 `<input type="color">` UI 자체를 조건부 렌더링하도록 했다(체크 시 숨김, 체크 해제 시 다시 표시, `borderColor` 값 자체는 숨겨져도 폼 상태에 보존).
