# 유지보수 이력 — SRM (서비스 카탈로그/서비스 요청)

> 유지보수 일시: 20260718-222005 · 도메인: srm

## 1. 요구사항

직전 유지보수(20260718-214053, 그리드 배치형 label 폐기 → "라벨(태그)" 개편)에 대한 세부 보정 요청 4건이다.
라벨 테두리는 측정한 bounding box 범위보다 4px 더 크게 표시한다.
minX-2, maxX+2, minY-2, maxY+2로 각 변 2px씩 확장한다.
라벨 텍스트는 테두리 좌측 상단에 표시한다.
라벨이 컴포넌트 1개에만 적용된 경우에도 그 컴포넌트에 테두리를 적용한다.
이는 직전 4차 설계의 "라벨 부여 컴포넌트가 1개면 테두리 없음" 결정을 뒤집는 것이다.
테두리 색 지정 UI에 "테두리 없음" 체크박스 옵션을 추가한다.

## 2. 해결 방법

### DB

변경 없음.

### BE

변경 없음.

### FE

`form-schema.ts`의 `GridLabel`에 `showBorder`(기본값 true) 필드를 신규 추가했다.
`dynamic-form-builder.tsx`의 라벨 경계 테두리 오버레이를 그리드 좌표(셀 경계)는 그대로 두고, 시각적으로만 각 변 2px씩(전체 4px) 바깥으로 확장하도록 수정했다.
테두리 오버레이 내부 좌측 상단에 라벨 텍스트(textColor 적용)를 표시하도록 추가했다.
경계 테두리 표시 기준을 "참조 컴포넌트 2개 이상"에서 "1개 이상"으로 변경했다(직전 4차 설계의 "1개면 테두리 없음" 결정을 명시적으로 뒤집음).
라벨 생성/수정 미니팝업에 "테두리 없음" 체크박스를 추가해 `showBorder`를 제어하도록 했다.
`showBorder`가 false면 캔버스 위 컴포넌트 그룹 경계 테두리만 숨기고, 팝업 상단 칩 스트립의 배지(테두리+틴트 배경)는 체크 여부와 무관하게 그대로 유지했다.

### 코드 리뷰

Standards축은 diff를 직접 정독해 발견 사항이 없었다.
Spec축은 설계 4건 요구사항 전부 부합했고 누락·범위초과는 없었다.
