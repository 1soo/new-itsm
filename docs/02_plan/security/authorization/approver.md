# 역할 정의 — APPROVER

> 역할: APPROVER · 버전: 0.2 · 작성일: 2026-07-11 · 승인 프로세스 커스텀 기능(유지보수 요청) 반영 — SRM/CHANGE 전용 고정 승인자에서 "승인 프로세스 규칙에 지정될 수 있는 범용 역할"로 일반화. 승인 처리 권한은 역할 자체가 아니라 각 승인 인스턴스의 역할 요구사항(어느 티켓·어느 차수에 어떤 역할이 지정됐는지)으로 런타임에 결정됨

## 공통 기본 접근 (전 역할 공통)

앱 셸/공통 컴포넌트(SCR-COM-*), 로그인·프로필·비밀번호(SCR-AUTH-001~003), 404, 자기 인증 API(API-AUTH-003/004/005), 내 메뉴 조회(API-AUTH-022), 통합 검색 API(API-SEARCH-001, 접근 가능 도메인만 결과 포함). SYSTEM_ADMIN은 모든 역할의 화면·API에 예외적으로 접근 가능하다(system_admin.md 2절 참조). 이하 역할 고유 접근만 기술한다.

## 1. 페르소나

- **역할명**: APPROVER (승인자 · CAB 멤버 포함)
- **설명**: SYSTEM_ADMIN이 정의한 승인 프로세스 규칙(SCR-ADMIN-008)의 승인자 차수에 지정될 수 있는 범용 역할. 관리자가 어느 도메인·요청유형·차수에 APPROVER 역할을 지정했는지에 따라 실제로 승인하는 대상이 달라진다(예: 서비스요청 전용 규칙에만 지정되어 있으면 서비스요청만, 변경까지 지정되어 있으면 변경도 승인).
- **주요 목표**: 본인 역할이 지정된 승인 차수를 검토해 승인/반려하고 사유를 남긴다.

## 2. 접근 가능 화면

| 화면 ID | 화면명 | 비고 |
|---------|--------|------|
| SCR-COM-014 | 승인 대기함(전 도메인 공용) | 특정 역할 전용 화면이 아니라 인증된 모든 사용자가 접근 가능. 실제 표시되는 대기 목록은 로그인 사용자의 role claim과 각 승인 인스턴스의 현재 차수 필요 역할의 교집합으로 런타임 결정(APPROVER가 어느 규칙에도 지정되지 않았다면 빈 목록) |
| SCR-SRM-005 | 요청 상세 | 승인 대상 조회(읽기) |
| SCR-CHG-003 | 변경 상세 | 승인 대상 조회(읽기) |

## 3. 접근 가능 API

| API ID | Endpoint | Method | 비고 |
|--------|----------|--------|------|
| API-COM-003 | /api/v1/approvals?scope=mine | GET | 전 도메인 공용 승인 대기(특정 역할 전용 API 아님, 인증된 모든 사용자 호출 가능 — 결과는 role claim 기준으로 필터링됨) |
| API-COM-004 | /api/v1/approvals/{approvalRequestId} | GET | 승인 인스턴스 상세(차수별 진행 상태) |
| API-COM-005 | /api/v1/approvals/{approvalRequestId}/decisions | POST | 승인/반려 결정 |
| API-SRM-008 | /api/v1/service-requests/{id} | GET | 요청 상세 조회 |
| API-CHG-003 | /api/v1/changes/{id} | GET | 변경 상세 조회 |

## 4. 접근 제어 규칙

- 로그인 필요 화면/요청 접근 시 Access Token의 `role`을 검증한다.
- **SCR-COM-014·API-COM-003/004는 역할 제한 없이 인증된 모든 사용자가 접근할 수 있다**(화면·API 자체에 대한 403은 없음). 대신 **결정 처리(API-COM-005)는 인스턴스별 동적 판정**을 거친다: 대상 승인 인스턴스의 현재 대기 차수가 요구하는 역할 목록에 로그인 사용자의 role claim이 하나도 포함되지 않으면 → **403**. 포함되면 처리 가능(공유 대기함, 먼저 처리한 사용자 또는 AND 집계 기준으로 결정 — [common.md](../../02_plan/database/common.md) `approval_decision` 참조).
- 반려 시 사유 필수(누락 시 400). 이미 결정된 역할 슬롯 재처리 또는 이미 종료된 인스턴스 처리 시도 → 409.
- 화면 이동마다 Backend에 권한 확인 API를 호출한다.
