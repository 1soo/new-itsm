# 화면 설계서 — 변경 관리 (Change)

> 도메인: change · 버전: 0.1 · 작성일: 2026-07-09

## 1. 개요

변경 요청(RFC) 생성, 목록/상세(유형 분류·6단계 프로세스·승인 경로·CAB 승인·표준 변경 사전승인·구현 결과 기록·인시던트/문제 연계), 변경 일정(캘린더), 승인 대기함, 지표 대시보드 화면을 정의한다.

## 2. 색상 팔레트

[common.md](common.md) 공통 팔레트를 사용한다.

| 구분 | 색상 (HEX) | 용도 |
|------|-----------|------|
| Base | #2563EB | 생성/저장/전이 버튼 |
| Warning | #D97706 | 승인 대기·긴급 변경·고위험 |
| Success | #16A34A | 승인·성공 구현 |
| Danger | #DC2626 | 반려·실패 구현 |
| Info | #60A5FA | 캘린더·예정 변경 |

## 4. 화면 상세

### SCR-CHG-001 · 변경 목록

- **목적**: 변경을 필터·검색하고 상세로 진입.
- **레이아웃**: [SCR-COM-007] 목록 패턴. 필터(유형·상태·위험도·기간) / 표(식별키·요약·유형·상태·위험도·예정일).
- **구성 요소**:
  | 요소 | 유형 | 설명 | 색상 |
  |------|------|------|------|
  | 유형 배지 | 배지 | 표준/일반/긴급 | Info/Base/Warning |
  | 상태 배지 | 배지 | 요청~종료 6단계 | 상태 시맨틱 |
- **상태 · 인터랙션**: 행 클릭 시 상세.
- **연관 API**: `GET /api/v1/changes`

### SCR-CHG-002 · 변경 요청(RFC) 생성

- **목적**: RFC 생성 및 위험·구현·영향·롤백 기입, 변경 유형 지정.
- **레이아웃**: 폼 카드. 요약(필수)·설명·변경 유형(필수)·위험도·예상 구현·영향 시스템·롤백 방법·예정 일정 / 생성 버튼.
- **구성 요소**:
  | 요소 | 유형 | 설명 | 색상 |
  |------|------|------|------|
  | 변경 유형 셀렉트 | 셀렉트 | 표준/일반/긴급 필수 | Base |
  | 위험도 셀렉트 | 셀렉트 | 저/중/고 | Warning(고) |
  | 롤백 방법 입력 | 입력 | 롤백 계획 | Border/Base |
  | 생성 버튼 | 버튼 | 생성(상태=요청) | Base |
- **상태 · 인터랙션**: 요약·변경 유형 누락 시 400. 표준(사전승인 템플릿) 선택 시 승인 단계 생략 안내. 성공 시 상세 이동.
- **연관 API**: `POST /api/v1/changes`, `GET /api/v1/change-templates`(표준 템플릿)

### SCR-CHG-003 · 변경 상세

- **목적**: 변경 전 과정(프로세스·승인경로·구현결과·연계) 관리.
- **레이아웃**: [SCR-COM-008] 상세 패턴. 본문·타임라인·코멘트 / 우측 메타(유형·위험도·상태·승인경로·승인 이력·연결 인시던트/문제/컴플라이언스 요구사항).
- **구성 요소**:
  | 요소 | 유형 | 설명 | 색상 |
  |------|------|------|------|
  | 프로세스 상태 전이 | 버튼 | 요청~종료, 승인 전 구현 차단 | Base |
  | 승인 경로 표시 | 배지 | 자동/동료검토/CAB | Info/Base/Warning |
  | 구현 결과 기록 | 폼 | 성공/실패·롤백 여부·비고 | Success/Danger |
  | 인시던트/문제 연계 | 버튼 | 링크 | Base |
- **상태 · 인터랙션**: 승인 완료 전 구현 전이 거부. 표준 변경은 승인 단계 자동 통과. 위험도 미평가 시 기본 CAB 경로. 승인 안 된 변경에 결과 기록 거부. 미존재 대상 연계 시 400.
- **연관 API**: `GET /api/v1/changes/{id}`, `PATCH .../status`, `POST .../result`, `POST .../links`

### SCR-CHG-004 · CAB 승인 대기함

- **목적**: 승인 권한자(CAB/승인자)가 변경을 검토·승인·반려.
- **레이아웃**: 승인 대기 목록 / 항목 상세 요약·승인/반려(의견 입력).
- **구성 요소**:
  | 요소 | 유형 | 설명 | 색상 |
  |------|------|------|------|
  | 승인 대기 목록 | 표 | 변경·유형·위험도·요청자 | Warning(대기) |
  | 승인/반려 버튼 | 버튼 | 결정·의견 | Success/Danger |
- **상태 · 인터랙션**: 승인 권한 없으면 403. 결정 시 상태 갱신·이력 기록.
- **연관 API**: `GET /api/v1/approvals?type=change&scope=mine`, `POST /api/v1/changes/{id}/approval`

### SCR-CHG-005 · 변경 일정(캘린더)

- **목적**: 예정 변경을 기간별 캘린더로 조회.
- **레이아웃**: 월/주 캘린더 뷰. 날짜별 변경 항목(유형 색상). 상단 기간·유형 필터.
- **구성 요소**:
  | 요소 | 유형 | 설명 | 색상 |
  |------|------|------|------|
  | 캘린더 그리드 | 캘린더 | 예정 변경 표시 | Info |
  | 변경 항목 칩 | 칩 | 유형별 색상 | Info/Base/Warning |
- **상태 · 인터랙션**: 예정 변경 없으면 빈 일정. 항목 클릭 시 상세.
- **연관 API**: `GET /api/v1/changes/schedule?from=&to=`

### SCR-CHG-006 · 변경 지표 대시보드

- **목적**: 변경 성공률·실패율·긴급 변경 비율 조회.
- **레이아웃**: 기간 필터 / KPI 카드(성공률·실패율·긴급 비율).
- **구성 요소**:
  | 요소 | 유형 | 설명 | 색상 |
  |------|------|------|------|
  | 기간 필터 | 셀렉트/날짜 | 조회 기간 | Border/Base |
  | KPI 카드 | 카드 | 성공/실패/긴급 비율(집계값) | Success/Danger/Warning |
- **상태 · 인터랙션**: 데이터 없으면 빈 결과. (REQ-CHG-010은 집계·조회만 요구하므로 시계열 추이 차트는 이번 범위 제외.)
- **연관 API**: `GET /api/v1/changes/metrics`
