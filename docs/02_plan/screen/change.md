# 화면 설계서 — 변경 관리 (Change)

> 도메인: change · 버전: 0.3 · 작성일: 2026-07-14 · UI 축소 유지보수 요청(2026-07-14) 반영 — SCR-CHG-001 목록 표 컬럼 폭 고정
>
> 이전 버전: 승인 프로세스 커스텀 기능(유지보수 요청) 반영 — 승인 경로(AUTO/PEER_REVIEW/CAB) 배지 제거, 공통 승인 패널로 교체. SCR-CHG-004(CAB 승인 대기함)는 [common.md](common.md) SCR-COM-014로 대체되어 제거

## 1. 개요

변경 요청(RFC) 생성, 목록/상세(유형 분류·6단계 프로세스·커스텀 승인 프로세스·표준 변경 사전승인·구현 결과 기록·인시던트/문제 연계), 변경 일정(캘린더), 지표 대시보드 화면을 정의한다. 승인 대기·결정은 [common.md](common.md) SCR-COM-014(전 도메인 공용)를 사용한다.

## 4. 화면 상세

### SCR-CHG-001 · 변경 목록

- **목적**: 변경을 필터·검색하고 상세로 진입.
- **레이아웃**: [SCR-COM-007] 목록 패턴. 필터(유형·상태·위험도·기간) / 표(식별키·요약·유형·상태·위험도·예정일).
- **구성 요소**:
  | 요소 | 유형 | 설명 | 색상 |
  |------|------|------|------|
  | 유형 배지 | 배지 | 표준/일반/긴급 | Info/Base/Warning |
  | 상태 배지 | 배지 | 요청~종료 6단계 | 상태 시맨틱 |
- **상태 · 인터랙션**: 행 클릭 시 상세.
- **컬럼 폭(유지보수 요청, 2026-07-14, [common.md](common.md) SCR-COM-007 아키텍처 적용)**:
  | 컬럼 | 폭(px) | 근거 |
  |------|--------|------|
  | 식별키 | 130 | 코드(ticketKey) |
  | 요약 | 미지정(auto) | 길이 가변, 잔여 폭 흡수. `line-clamp-1` 적용(기존 유지) |
  | 유형 | 110 | 유형 뱃지 |
  | 상태 | 110 | 상태 뱃지 |
  | 위험도 | 110 | 위험도 뱃지/"미평가" |
  | 예정일 | 110 | 날짜(YYYY-MM-DD)/"-" |
- **페이지당 아이템 수(유지보수 요청, 2026-07-14)**: 10건 → 14건([common.md](common.md) 7절 패턴 B).
- **연관 API**: `GET /api/v1/changes`

### SCR-CHG-002 · 변경 요청(RFC) 생성

- **목적**: RFC 생성 및 위험·구현·영향·롤백 기입, 변경 유형 지정.
- **레이아웃**: 폼 카드. 요약(필수)·설명·변경 유형(필수)·위험도·예상 구현·영향 시스템·롤백 방법·예정 일정 / 생성 버튼.
- **구성 요소**:
  | 요소 | 유형 | 설명 | 색상 |
  |------|------|------|------|
  | 변경 유형 셀렉트 | 셀렉트 | 표준/일반/긴급 필수 | Base |
  | 위험도 셀렉트 | 셀렉트 | 저/중/고 | Warning(고) |
  | 롤백 방법 입력 | 입력 | 롤백 계획 | Border/Base |
  | 생성 버튼 | 버튼 | 생성(상태=요청) | Base |
- **상태 · 인터랙션**: 요약·변경 유형 누락 시 400. 표준(사전승인 템플릿) 선택 시 승인 단계 생략 안내. 성공 시 상세 이동.
- **연관 API**: `POST /api/v1/changes`, `GET /api/v1/change-templates`(표준 템플릿)

### SCR-CHG-003 · 변경 상세

- **목적**: 변경 전 과정(프로세스·커스텀 승인 프로세스 진행·구현결과·연계) 관리.
- **레이아웃**: [SCR-COM-008] 상세 패턴. 본문·타임라인·코멘트 / 우측 메타(유형·위험도·상태·승인 패널(공용)·연결 인시던트/문제/컴플라이언스 요구사항).
- **구성 요소**:
  | 요소 | 유형 | 설명 | 색상 |
  |------|------|------|------|
  | 프로세스 상태 전이 | 버튼 | 요청~종료, 승인 전 구현 차단 | Base |
  | 승인 패널(공용) | 패널 | 매칭되는 승인 프로세스가 있으면 차수 진행 상태 표시(API-COM-004), 없으면 "이 변경에는 승인 절차가 없습니다" 안내 | Info/Warning/Success |
  | 구현 결과 기록 | 폼 | 성공/실패·롤백 여부·비고 | Success/Danger |
  | 인시던트/문제 연계 | 버튼 | 링크 | Base |
- **상태 · 인터랙션**: 승인 완료 전 구현 전이 거부(매칭되는 프로세스가 없으면 게이트 없이 진행). 승인 안 된 변경에 결과 기록 거부. 미존재 대상 연계 시 400. 승인 대기 중인 건의 처리는 [common.md](common.md) SCR-COM-014에서 수행(이 화면은 진행 상태 조회 전용).
- **연관 API**: `GET /api/v1/changes/{id}`, `PATCH .../status`, `POST .../result`, `POST .../links`, `GET /api/v1/approvals/{approvalRequestId}`(API-COM-004)

### SCR-CHG-005 · 변경 일정(캘린더)

- **목적**: 예정 변경을 기간별 캘린더로 조회.
- **레이아웃**: 월/주 캘린더 뷰. 날짜별 변경 항목(유형 색상). 상단 기간·유형 필터.
- **구성 요소**:
  | 요소 | 유형 | 설명 | 색상 |
  |------|------|------|------|
  | 캘린더 그리드 | 캘린더 | 예정 변경 표시 | Info |
  | 변경 항목 칩 | 칩 | 유형별 색상 | Info/Base/Warning |
- **상태 · 인터랙션**: 예정 변경 없으면 빈 일정. 항목 클릭 시 상세.
- **연관 API**: `GET /api/v1/changes/schedule?from=&to=`

### SCR-CHG-006 · 변경 지표 대시보드

- **목적**: 변경 성공률·실패율·긴급 변경 비율 조회.
- **레이아웃**: 기간 필터 / KPI 카드(성공률·실패율·긴급 비율).
- **구성 요소**:
  | 요소 | 유형 | 설명 | 색상 |
  |------|------|------|------|
  | 기간 필터 | 셀렉트/날짜 | 조회 기간 | Border/Base |
  | KPI 카드 | 카드 | 성공/실패/긴급 비율(집계값) | Success/Danger/Warning |
- **상태 · 인터랙션**: 데이터 없으면 빈 결과. (REQ-CHG-010은 집계·조회만 요구하므로 시계열 추이 차트는 이번 범위 제외.)
- **연관 API**: `GET /api/v1/changes/metrics`
