# 화면 설계서 — 인증/계정 (Auth)

> 도메인: auth · 버전: 0.1 · 작성일: 2026-07-09

## 1. 개요

로그인, 토큰 재발급(백그라운드), 로그아웃, 내 프로필 조회, 비밀번호 변경 등 사용자 본인 인증·계정 화면을 정의한다. 관리자용 계정/역할/감사로그 화면은 [admin.md](admin.md)를 참조한다.

## 2. 색상 팔레트

[common.md](common.md) 공통 팔레트를 사용한다.

| 구분 | 색상 (HEX) | 용도 |
|------|-----------|------|
| Base | #2563EB | 로그인/저장 버튼·포커스 |
| Variation 1 | #1E40AF | 버튼 Hover |
| Danger | #DC2626 | 인증 실패·검증 오류 |
| Neutral Light | #F9FAFB | 배경 |

## 3. 화면 목록

| 화면 ID | 화면명 | 관련 요구사항/기능 | 설명 |
|---------|--------|--------------------|------|
| SCR-AUTH-001 | 로그인 | REQ-AUTH-002 / FEAT-AUTH-002 | 이메일·비밀번호 로그인 |
| SCR-AUTH-002 | 내 프로필 | REQ-AUTH-001 / FEAT-AUTH-001 | 본인 정보·역할 조회 |
| SCR-AUTH-003 | 비밀번호 변경 | REQ-AUTH-007 / FEAT-AUTH-007 | 현재·새 비밀번호 변경 |

## 4. 화면 상세

### SCR-AUTH-001 · 로그인

- **목적**: 이메일·비밀번호로 인증하고 토큰을 발급받아 진입.
- **레이아웃**: 중앙 정렬 카드. 로고 / 이메일·비밀번호 입력 / 로그인 버튼 / 오류 메시지 영역.
- **구성 요소**:
  | 요소 | 유형 | 설명 | 색상 |
  |------|------|------|------|
  | 이메일 입력 | 입력 | 형식 검증 | Border/Base(포커스) |
  | 비밀번호 입력 | 입력(마스킹) | 표시 토글 | Border/Base |
  | 로그인 버튼 | 버튼 | 제출 | Base |
  | 오류 메시지 | 텍스트 | 인증 실패·비활성 계정 안내 | Danger |
- **상태 · 인터랙션**: 제출 중 버튼 로딩. 401(불일치)·403(비활성)은 계정 열거 방지를 위해 "이메일 또는 비밀번호가 올바르지 않습니다"로 통일 표기. 성공 시 역할 기본 홈으로 이동.
- **연관 API**: `POST /api/v1/auth/login`

### SCR-AUTH-002 · 내 프로필

- **목적**: 로그인 사용자의 기본 정보와 부여된 역할을 확인.
- **레이아웃**: 상단 제목 / 정보 카드(이름·이메일·역할 목록·상태) / "비밀번호 변경" 버튼.
- **구성 요소**:
  | 요소 | 유형 | 설명 | 색상 |
  |------|------|------|------|
  | 프로필 카드 | 카드 | 이름·이메일·상태 | Neutral |
  | 역할 배지 | 배지 | 부여된 역할 목록 | Variation 2 |
  | 비밀번호 변경 버튼 | 버튼 | SCR-AUTH-003 이동 | Base |
- **상태 · 인터랙션**: 진입 시 본인 정보 로드. 역할 정보는 읽기 전용.
- **연관 API**: `GET /api/v1/auth/me`

### SCR-AUTH-003 · 비밀번호 변경

- **목적**: 현재 비밀번호 검증 후 새 비밀번호로 변경.
- **레이아웃**: 폼 카드. 현재 비밀번호 / 새 비밀번호 / 새 비밀번호 확인 / 저장 버튼.
- **구성 요소**:
  | 요소 | 유형 | 설명 | 색상 |
  |------|------|------|------|
  | 현재 비밀번호 | 입력(마스킹) | 검증 대상 | Border/Base |
  | 새 비밀번호/확인 | 입력(마스킹) | 정책·일치 검증 | Border/Base |
  | 저장 버튼 | 버튼 | 제출 | Base |
  | 오류 메시지 | 텍스트 | 현재 비밀번호 불일치·정책 위반 | Danger |
- **상태 · 인터랙션**: 새 비밀번호 정책(길이 등) 미충족 시 인라인 오류. 현재 비밀번호 불일치 시 Danger 메시지. 성공 시 토스트 후 프로필 복귀.
- **연관 API**: `PATCH /api/v1/auth/me/password`
