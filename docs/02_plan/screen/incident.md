## 1. 개요

인시던트 등록, 목록/상세(심각도·우선순위·상태 전이·역할 배정·에스컬레이션·상태 커뮤니케이션/타임라인·해결/시간지표·포스트모템·문제 연계), 지표 대시보드 화면을 정의한다.

## 2. 색상 팔레트

[common.md](common.md) 공통 팔레트를 사용한다. 심각도(SEV)는 Danger 계열 강조.

| 구분 | 색상 (HEX) | 용도 |
|------|-----------|------|
| Base | #2563EB | 등록/저장/전이 버튼 |
| Danger | #DC2626 | SEV1·미해결 강조 |
| Warning | #D97706 | SEV2·대응중·포스트모템 필요 |
| Success | #16A34A | 해결/종료 |
| Info | #60A5FA | 타임라인·내부 업데이트 |

## 4. 화면 상세

### SCR-INC-001 · 인시던트 목록

- **목적**: 인시던트를 필터·검색하고 상세로 진입.
- **레이아웃**: [SCR-COM-007] 목록 패턴. 필터(상태·심각도·담당자·기간·키워드) / 표(식별키·요약·SEV·상태·담당자·갱신일) / "인시던트 등록" 버튼.
- **구성 요소**:
  | 요소 | 유형 | 설명 | 색상 |
  |------|------|------|------|
  | 심각도 배지 | 배지 | SEV1~3 | Danger/Warning/Info |
  | 상태 배지 | 배지 | 신규/대응중/해결/종료 | 상태 시맨틱 |
  | 포스트모템 필요 태그 | 배지 | SEV1·2 해결·PM 미작성 | Warning |
- **상태 · 인터랙션**: 행 클릭 시 상세. 로딩 스켈레톤·빈 상태.
- **컬럼 폭([common.md](common.md) SCR-COM-007 아키텍처 적용)**:
  | 컬럼 | 폭(px) | 근거 |
  |------|--------|------|
  | 식별키 | 130 | 코드(ticketKey) |
  | 요약 | 미지정(auto) | 길이 가변, 잔여 폭 흡수. `line-clamp-1` 적용(기존 유지) |
  | 심각도 | 110 | SEV 뱃지 |
  | 상태 | 110 | 상태 뱃지 |
  | 담당자 | 120 | 사람 이름/"미배정" |
  | PM | 90 | "PM 필요" 뱃지, 헤더 라벨 짧음 |
  | 갱신일 | 110 | 날짜(YYYY-MM-DD) |
- **페이지당 아이템 수**: 14건([common.md](common.md) 7절 패턴 B).
- **연관 API**: `GET /api/v1/incidents`

### SCR-INC-002 · 인시던트 등록

- **목적**: 핵심 필드로 인시던트를 생성.
- **레이아웃**: 폼 카드. 요약(필수)·설명·심각도(필수)·영향 서비스·영향 제품 / 등록 버튼.
- **구성 요소**:
  | 요소 | 유형 | 설명 | 색상 |
  |------|------|------|------|
  | 요약 입력 | 입력 | 필수 | Border/Base |
  | 심각도 셀렉트 | 셀렉트 | SEV1~3 필수 | Danger 계열 |
  | 영향 서비스/제품 | 셀렉트/입력 | 영향 범위 | Border/Base |
  | 등록 버튼 | 버튼 | 생성(상태=신규) | Base |
- **상태 · 인터랙션**: 요약·심각도 누락 시 400·인라인 오류. 성공 시 상세 이동.
- **연관 API**: `POST /api/v1/incidents`

### SCR-INC-003 · 인시던트 상세

- **목적**: 인시던트 대응 전 과정을 관리.
- **레이아웃**: [SCR-COM-008] 상세 패턴. 본문·타임라인·코멘트 / 우측 메타(SEV·우선순위·상태·대응 역할·시간지표·연결 문제/자산).
- **구성 요소**:
  | 요소 | 유형 | 설명 | 색상 |
  |------|------|------|------|
  | 심각도/우선순위 편집 | 셀렉트 | 독립 관리·변경 이력 | Danger/Base |
  | 상태 전이 버튼 | 버튼 | 허용 전이만 노출. 라벨은 동작 동사형([common.md](common.md) SCR-COM-008 아키텍처 적용) — 아래 표 | Base |
  | 역할 배정 패널 | 폼 | IM이 Tech Lead·Comms·Scribe 배정 | Base |
  | 에스컬레이션 버튼 | 버튼 | 계층적/기능적 이관 | Warning |
  | 상태 업데이트 입력 | 입력+공개범위 토글 | 내부/외부 구분 | Info |
  | 해결 처리·시간지표 | 폼/표시 | 영향시작·탐지·영향종료 / MTTD·MTTA·MTTR | Success |
  | 문제 연계 버튼 | 버튼 | 신규/기존 문제 링크 | Base |
  | 승인 패널(공용) | 패널 | RESOLVED 전이에 매칭되는 승인 프로세스가 있으면 차수 진행 상태(API-COM-004) 표시, 없으면 패널 자체를 노출하지 않고 기존처럼 즉시 전이 | Info/Warning/Success |
- **상태 · 인터랙션**: 허용되지 않은 상태 전이 비노출. IM 권한 없으면 역할 배정 버튼 숨김/403. 시간 정보 없으면 해당 지표 "미산정" 표시. SEV1·2 해결·PM 미작성 시 "포스트모템 필요" 배너. 잘못된 심각도/미존재 문제 연계 시 400. RESOLVED 전이 시도 시 매칭되는 승인 프로세스가 있으면 409와 함께 승인 패널이 나타나며, 처리는 [common.md](common.md) SCR-COM-014에서 수행한다.
  - **전이 버튼 라벨**: `status.ts`의 `transitionLabel(t, target)`(i18n 키 `incident:transition.*`)이 버튼 텍스트를 결정한다(`statusLabel(t, target)`은 배지·토스트 문구에 별도로 사용).
    | 도착 상태 | 버튼 라벨 |
    |-----------|-----------|
    | IN_PROGRESS | 대응 시작 |
    | RESOLVED | 해결 처리 |
    | CLOSED | 종료 처리 |
  - **타임라인 actor 표시**: `IncidentDetailResponse.TimelineEntry`의 `actor` 필드가 행위 수행 주체자를 표시한다([common.md](common.md) SCR-COM-008 아키텍처 참조). 상태 변경 타임라인 메시지는 `target.name()`(코드) 대신 상태 라벨을 사용한다.
- **연관 API**: `GET /api/v1/incidents/{id}`, `PATCH .../severity`, `PATCH .../status`, `POST .../roles`, `POST .../escalate`, `POST .../updates`, `POST .../resolve`, `POST .../links`, `GET /api/v1/approvals/{approvalRequestId}`(API-COM-004)

### SCR-INC-004 · 포스트모템 편집

- **목적**: blameless 포스트모템 작성(요약·타임라인·5 Whys·근본원인·조치항목).
- **레이아웃**: 섹션형 폼. 요약 / 타임라인 요약 / 5 Whys 단계 입력 / 근본원인(필수) / 조치항목(반복) / 제출.
- **구성 요소**:
  | 요소 | 유형 | 설명 | 색상 |
  |------|------|------|------|
  | 5 Whys 입력 | 반복 입력 | Why 사슬 | Border/Base |
  | 근본원인 입력 | 입력 | 필수·개인 지정 강제 안 함 | Border/Base |
  | 조치항목 리스트 | 반복 입력 | 담당·기한·상태 | Base |
  | 제출 버튼 | 버튼 | 인시던트에 연결 저장 | Base |
- **상태 · 인터랙션**: 근본원인 비어 제출 시 거부. 저장 시 인시던트와 연결.
- **연관 API**: `GET/PUT /api/v1/incidents/{id}/postmortem`

### SCR-INC-005 · 인시던트 지표 대시보드

- **목적**: 기간별 건수·심각도 분포·평균 MTTR 조회.
- **레이아웃**: 기간 필터 / KPI 카드(건수·평균 MTTR) / 심각도 분포 차트.
- **구성 요소**:
  | 요소 | 유형 | 설명 | 색상 |
  |------|------|------|------|
  | KPI 카드 | 카드 | 건수·평균 MTTR | Base |
  | 심각도 분포 차트 | 차트 | SEV별 분포 | Danger/Warning/Info |
- **상태 · 인터랙션**: 데이터 없으면 빈 결과.
- **연관 API**: `GET /api/v1/incidents/metrics`
