# 화면 설계서 — 문제 관리 (Problem)

> 도메인: problem · 버전: 0.2 · 작성일: 2026-07-11 · 승인 프로세스 커스텀 기능(유지보수 요청) 반영 — 상세 화면에 공통 승인 패널 추가(WORKAROUND → RESOLVED_CLOSED 전이 게이트, 매칭 규칙 없으면 기존처럼 즉시 전이)

## 1. 개요

문제 등록, 목록/상세(6단계 프로세스 상태·우선순위(영향도×긴급도)·RCA·워크어라운드·인시던트/변경 연계·후속 조치·종료), 알려진 오류(KEDB) 검색 화면을 정의한다.

## 2. 색상 팔레트

[common.md](common.md) 공통 팔레트를 사용한다.

| 구분 | 색상 (HEX) | 용도 |
|------|-----------|------|
| Base | #2563EB | 등록/저장/전이 버튼 |
| Warning | #D97706 | 조사중·미해결 후속조치 경고 |
| Success | #16A34A | 종료·완료 조치 |
| Danger | #DC2626 | 높은 우선순위 |
| Info | #60A5FA | 연결 항목·KEDB |

## 4. 화면 상세

### SCR-PRB-001 · 문제 목록

- **목적**: 문제를 필터·검색하고 상세로 진입.
- **레이아웃**: [SCR-COM-007] 목록 패턴. 필터(상태·우선순위·출처(반응/선제)·담당자·기간) / 표(식별키·요약·상태·우선순위·담당자·갱신일).
- **구성 요소**:
  | 요소 | 유형 | 설명 | 색상 |
  |------|------|------|------|
  | 상태 배지 | 배지 | 6단계(탐지~종료) | 상태 시맨틱 |
  | 우선순위 배지 | 배지 | 영향도×긴급도 결과 | Danger~Muted |
- **상태 · 인터랙션**: 행 클릭 시 상세.
- **연관 API**: `GET /api/v1/problems`

### SCR-PRB-002 · 문제 등록

- **목적**: 표준 필드로 문제를 생성.
- **레이아웃**: 폼 카드. 요약(필수)·설명·출처(반응/선제)·조사사유·영향도·긴급도·구성요소 / 등록 버튼. 영향도·긴급도 입력 시 우선순위 실시간 표시.
- **구성 요소**:
  | 요소 | 유형 | 설명 | 색상 |
  |------|------|------|------|
  | 요약 입력 | 입력 | 필수 | Border/Base |
  | 영향도/긴급도 셀렉트 | 셀렉트 | 우선순위 산정 입력 | Border/Base |
  | 우선순위 미리보기 | 배지 | 매트릭스 산정 결과 | Danger~Muted |
  | 등록 버튼 | 버튼 | 생성(상태=탐지) | Base |
- **상태 · 인터랙션**: 요약 누락 시 400. 영향도/긴급도 중 하나라도 없으면 우선순위 "미산정" 표시. 성공 시 상세 이동.
- **연관 API**: `POST /api/v1/problems`

### SCR-PRB-003 · 문제 상세

- **목적**: 문제 조사·해결 전 과정을 관리.
- **레이아웃**: [SCR-COM-008] 상세 패턴. 본문·타임라인·코멘트 / 우측 메타(상태·우선순위·연결 인시던트/변경·KE 레코드·후속조치).
- **구성 요소**:
  | 요소 | 유형 | 설명 | 색상 |
  |------|------|------|------|
  | 프로세스 상태 전이 | 버튼 | 6단계 허용 전이만 | Base |
  | RCA 섹션 | 폼 | 근본원인·5 Whys·카테고리 | Border/Base |
  | 워크어라운드 입력 | 입력 | 임시 대응책·지식 연결 | Info |
  | 알려진 오류 생성 | 버튼 | 근본원인+워크어라운드→KEDB | Info |
  | 인시던트 연결 | 버튼 | 양방향 링크 | Base |
  | 변경 연계 | 버튼 | 신규/기존 변경 링크 | Base |
  | 후속 조치 리스트 | 반복 입력 | 항목·상태(진행중/완료) | Base/Success |
  | 종료 버튼 | 버튼 | 종료 전이 | Success |
  | 승인 패널(공용) | 패널 | RESOLVED_CLOSED 전이에 매칭되는 승인 프로세스가 있으면 차수 진행 상태(API-COM-004) 표시, 없으면 패널 자체를 노출하지 않고 기존처럼 즉시 전이 | Info/Warning/Success |
- **상태 · 인터랙션**: 순서 어긋난 전이 거부. RCA는 개인 강제 지정 안 함. 미존재 인시던트/변경 연결 시 400. 미해결 후속조치 남은 채 종료 시 경고 다이얼로그. 워크어라운드 빈 내용 저장 거부. 종료(RESOLVED_CLOSED) 전이 시도 시 매칭되는 승인 프로세스가 있으면 409와 함께 승인 패널이 나타나며, 처리는 [common.md](common.md) SCR-COM-014에서 수행한다.
- **연관 API**: `GET /api/v1/problems/{id}`, `PATCH .../status`, `PUT .../rca`, `POST .../workaround`, `POST .../known-errors`, `POST .../links`, `POST/PATCH .../actions`, `POST .../close`, `GET /api/v1/approvals/{approvalRequestId}`(API-COM-004)

### SCR-PRB-004 · KEDB 검색

- **목적**: 알려진 오류(근본원인+워크어라운드)를 키워드로 조회.
- **레이아웃**: 검색바 / 결과 목록(제목·근본원인 요약·워크어라운드·연결 문제).
- **구성 요소**:
  | 요소 | 유형 | 설명 | 색상 |
  |------|------|------|------|
  | 검색바 | 입력 | 키워드 | Border/Base |
  | KE 결과 목록 | 리스트 | 매칭 알려진 오류 | Info |
- **상태 · 인터랙션**: 결과 없으면 빈 목록.
- **연관 API**: `GET /api/v1/known-errors?keyword=`
