# 화면 설계서 — 컴플라이언스 관리 (Compliance Management)

> 도메인: compliance · 버전: 0.1 · 작성일: 2026-07-10

## 1. 개요

컴플라이언스 요구사항 등록·목록/상세(책임자 지정·시정조치 추적·변경 연계·감사 로그 조회), 준수 현황 대시보드 화면을 정의한다.

## 2. 색상 팔레트

[common.md](common.md) 공통 팔레트를 사용한다.

| 구분 | 색상 (HEX) | 용도 |
|------|-----------|------|
| Base | #2563EB | 등록/저장/전이 버튼 |
| Danger | #DC2626 | 미준수(NON_COMPLIANT)·책임자 미지정 |
| Warning | #D97706 | 시정조치 조치중 |
| Success | #16A34A | 준수(COMPLIANT)·시정조치 해결 |
| Info | #60A5FA | 변경 연계·감사 로그 |

## 4. 화면 상세

### SCR-COMP-001 · 컴플라이언스 요구사항 목록

- **목적**: 요구사항을 준수 상태·책임자 지정 여부로 필터링해 조회.
- **레이아웃**: [SCR-COM-007] 목록 패턴. 필터(준수 상태·책임자 지정 여부) / 표(식별키·이름·근거·책임자·준수 상태·갱신일).
- **구성 요소**:
  | 요소 | 유형 | 설명 | 색상 |
  |------|------|------|------|
  | 준수 상태 배지 | Lozenge | COMPLIANT/NON_COMPLIANT(미해결 시정조치 존재) | Success/Danger |
  | 책임자 미지정 배지 | Lozenge | 책임자 없음 강조 | Danger |
- **상태 · 인터랙션**: 행 클릭 시 상세.
- **연관 API**: `GET /api/v1/compliance/requirements`

### SCR-COMP-002 · 요구사항 등록

- **목적**: 이름·근거·적용 범위로 요구사항을 생성.
- **레이아웃**: 폼 카드. 이름(필수)·근거(규제/정책, 필수)·적용 범위 / 등록 버튼.
- **구성 요소**:
  | 요소 | 유형 | 설명 | 색상 |
  |------|------|------|------|
  | 이름 입력 | 입력 | 필수 | Border/Base |
  | 근거 입력 | 입력 | 규제 조항/내부 정책, 필수 | Border/Base |
  | 등록 버튼 | 버튼 | 생성 | Base |
- **상태 · 인터랙션**: 이름·근거 누락 시 400. 성공 시 상세 이동.
- **연관 API**: `POST /api/v1/compliance/requirements`

### SCR-COMP-003 · 요구사항 상세

- **목적**: 책임자 지정, 시정조치 등록·추적, 변경 요청 연계, 관련 감사 로그 확인.
- **레이아웃**: [SCR-COM-008] 상세 패턴. 본문(근거·적용범위)·시정조치 목록·감사 로그 / 우측 메타(책임자·준수 상태·연결 변경 요청).
- **구성 요소**:
  | 요소 | 유형 | 설명 | 색상 |
  |------|------|------|------|
  | 책임자 지정 | 셀렉트+버튼 | 오너 지정 | Base |
  | 시정조치 등록 | 폼 | 내용 입력 → 탐지 상태로 생성 | Base |
  | 시정조치 상태 전이 | 버튼 | 탐지→조치중→해결 | Warning/Success |
  | 변경 연계 | 버튼 | 변경 요청 링크 | Info |
  | 감사 로그 목록 | 리스트 | 요구사항 관련 활동 이력(수행자·시각) | Info |
- **상태 · 인터랙션**: 책임자 미지정 시 목록/상세에 "책임자 미지정" 표시. 정의되지 않은 시정조치 상태 전이 시 400. 미존재 변경 요청 연계 시 400.
- **연관 API**: `GET /api/v1/compliance/requirements/{id}`, `PATCH .../` , `POST .../owner`, `POST .../corrective-actions`, `PATCH /api/v1/compliance/corrective-actions/{actionId}/status`, `POST .../links`, `GET /api/v1/compliance/audit-logs`

### SCR-COMP-004 · 준수 현황 대시보드

- **목적**: 요구사항별 상태·미해결 시정조치 건수·전체 준수율 집계 조회.
- **레이아웃**: 상단 기간 필터 / KPI 카드(준수율·미해결 시정조치 건수) / 요구사항별 상태 표.
- **구성 요소**:
  | 요소 | 유형 | 설명 | 색상 |
  |------|------|------|------|
  | KPI 카드 | 카드 | 준수율·미해결 건수(집계값) | Success/Danger |
  | 요구사항별 상태 표 | 표 | 요구사항·책임자·준수 상태 | 상태 시맨틱 |
- **상태 · 인터랙션**: 데이터 없으면 빈 결과 표시.
- **연관 API**: `GET /api/v1/compliance/metrics`
