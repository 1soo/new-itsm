# 화면 설계서 — 관리자 (Admin)

> 도메인: admin(auth) · 버전: 0.5 · 작성일: 2026-07-14 · UI 축소 유지보수 요청(2026-07-14) 반영 — SCR-ADMIN-001/005/006 목록 표 컬럼 폭 고정(SCR-COM-007 아키텍처, common.md 참조)
>
> 이전 버전: 사이드바 메뉴 i18n 미적용 결함 수정(유지보수 요청) — SCR-ADMIN-006에 영문 메뉴명/그룹명 입력 추가 · 승인 프로세스 메타데이터 분리(유지보수 요청) — SCR-ADMIN-008 0/1단계(도메인·요청유형)를 "규칙 정보" 카드로 이관, 승인 단계는 승인 요청자부터 재시작

## 1. 개요

System Admin이 사용하는 관리 화면을 정의한다. 사용자 계정 관리, 역할 정의·부여, 감사 로그 조회, 승인 프로세스 커스텀 정의를 포함한다. auth 도메인 요구사항(REQ-AUTH-001·005·006·008)에 대응한다. 서비스 카탈로그 등 도메인별 관리 화면은 각 도메인 화면 설계서에 둔다.

## 2. 색상 팔레트

[common.md](common.md) 공통 팔레트를 사용한다.

## 4. 화면 상세

### SCR-ADMIN-001 · 계정 목록

- **목적**: 사용자 계정을 검색·필터하고 상세로 진입.
- **레이아웃**: [SCR-COM-007] 목록 패턴. 필터(이메일·이름·상태·역할) / 표(이름·이메일·역할·상태·생성일) / "계정 생성" 버튼.
- **구성 요소**:
  | 요소 | 유형 | 설명 | 색상 |
  |------|------|------|------|
  | 상태 배지 | 배지 | 활성/비활성 | Success/Warning |
  | 계정 생성 버튼 | 버튼 | SCR-ADMIN-002 이동 | Base |
- **상태 · 인터랙션**: 행 클릭 시 상세. System Admin만 접근(그 외 403).
- **컬럼 폭(유지보수 요청, 2026-07-14, SCR-COM-007 아키텍처 적용)**:
  | 컬럼 | 폭(px) | 근거 |
  |------|--------|------|
  | 이름 | 120 | 사람 이름 |
  | 이메일 | 미지정(auto) | 이메일 길이 가변, 잔여 폭 흡수. `truncate` 적용 |
  | 역할 | 220 | `ROLE_*` 뱃지 1~n개 wrap 표시 |
  | 상태 | 100 | 활성/비활성 뱃지 |
  | 생성일 | 110 | 날짜(YYYY-MM-DD) |
- **페이지당 아이템 수(유지보수 요청, 2026-07-14)**: 10건 → 14건([common.md](common.md) 7절 패턴 C).
- **연관 API**: `GET /api/v1/admin/users`

### SCR-ADMIN-002 · 계정 생성

- **목적**: 신규 사용자 계정 생성.
- **레이아웃**: 폼 카드. 이메일·이름·초기 역할(멀티 선택)·초기 비밀번호(입력, 정책 안내 동반) / 저장.
- **구성 요소**:
  | 요소 | 유형 | 설명 | 색상 |
  |------|------|------|------|
  | 이메일/이름 입력 | 입력 | 필수·형식 검증 | Border/Base |
  | 초기 역할 선택 | 멀티셀렉트 | 하나 이상 역할 | Base |
  | 초기 비밀번호 입력 | 입력(마스킹) | 필수·정책 검증·계정별 고유 | Border/Base |
  | 저장 버튼 | 버튼 | 생성 | Base |
  | 중복/정책 오류 | 텍스트 | 이메일 중복(409)·비밀번호 정책 위반(400) 안내 | Danger |
- **상태 · 인터랙션**: 초기 비밀번호는 관리자가 계정별로 지정(정책 미충족 시 400 인라인 오류)하며 사용자가 최초 로그인 후 SCR-AUTH-003에서 변경한다. 이메일 중복 시 409 → 인라인 오류. 성공 시 토스트 후 목록 복귀·감사 로그 기록.
- **연관 API**: `POST /api/v1/admin/users`

### SCR-ADMIN-003 · 계정 상세·수정

- **목적**: 계정 정보 수정, 비활성화, 역할 부여/회수.
- **레이아웃**: 상단 계정 요약 / 정보 수정 폼 / 역할 부여·회수 패널 / "비활성화" 위험 버튼.
- **구성 요소**:
  | 요소 | 유형 | 설명 | 색상 |
  |------|------|------|------|
  | 정보 수정 폼 | 폼 | 이름 등 수정 | Border/Base |
  | 역할 부여/회수 | 멀티셀렉트+칩 | 복수 역할 관리 | Base/Danger(회수) |
  | 비활성화 버튼 | 버튼 | 확인 다이얼로그 동반 | Danger |
- **상태 · 인터랙션**: 역할 부여/회수 시 즉시 반영. 비활성화는 확인 다이얼로그 후 처리(로그인 차단). 모든 변경 감사 로그 기록.
- **연관 API**: `GET/PATCH /api/v1/admin/users/{userId}`, `POST/DELETE /api/v1/admin/users/{userId}/roles`, `PATCH /api/v1/admin/users/{userId}/status`

### SCR-ADMIN-004 · 역할 관리

- **목적**: 역할 정의·조회.
- **레이아웃**: 역할 목록 표(역할명·설명·부여 사용자 수) / "역할 생성" 버튼 / 생성·수정 모달.
- **구성 요소**:
  | 요소 | 유형 | 설명 | 색상 |
  |------|------|------|------|
  | 역할 목록 표 | 표 | 역할명·설명 | Neutral |
  | 역할 생성/수정 모달 | 모달 | 역할명·설명 입력 | Base |
- **상태 · 인터랙션**: 존재하지 않는 역할 부여 시도는 400. System Admin 전용.
- **연관 API**: `GET/POST /api/v1/admin/roles`

### SCR-ADMIN-005 · 감사 로그 조회

- **목적**: 인증·인가·계정/역할 변경 이벤트 이력 조회.
- **레이아웃**: 필터(이벤트 유형·주체·대상·기간) / 표(시각·이벤트·주체·대상·결과) / 페이지네이션.
- **구성 요소**:
  | 요소 | 유형 | 설명 | 색상 |
  |------|------|------|------|
  | 로그 표 | 표 | 시각·유형·주체·대상 | Neutral |
  | 이벤트 유형 필터 | 셀렉트 | 로그인/로그아웃/재발급/계정·역할 변경 | Base |
- **상태 · 인터랙션**: 읽기 전용. 결과 0건 시 빈 상태. System Admin 전용.
- **컬럼 폭(유지보수 요청, 2026-07-14, SCR-COM-007 아키텍처 적용)**:
  | 컬럼 | 폭(px) | 근거 |
  |------|--------|------|
  | 시각 | 160 | 일시(YYYY-MM-DD HH:mm) |
  | 이벤트 | 130 | 이벤트 라벨(로그인/계정 생성 등) |
  | 주체 | 160 | 행위자 식별자(대개 이메일), `truncate` 적용 |
  | 대상 | 미지정(auto) | 대상 리소스 식별자 길이 가변, 잔여 폭 흡수. `truncate` 적용 |
  | 결과 | 100 | 성공/실패 뱃지 |
- **페이지당 아이템 수(유지보수 요청, 2026-07-14)**: 10건 → 14건([common.md](common.md) 7절 패턴 C).
- **연관 API**: `GET /api/v1/admin/audit-logs`

### SCR-ADMIN-006 · 메뉴 관리

- **목적**: 사이드바 메뉴 항목(마스터 데이터)을 등록·수정·삭제하고, 역할별로 어떤 메뉴를 노출할지 매핑한다(Main 요청, 2026-07-11). 기존 SCR-ADMIN-004(역할 관리)와는 별개 화면이며, 역할 자체의 정의·부여는 그대로 SCR-ADMIN-004/003을 사용한다.
- **레이아웃**: [SCR-COM-007] 목록 패턴. 상단 "메뉴 생성" 버튼 / 메뉴 목록 표(그룹·메뉴명·경로·아이콘·순서·노출 역할 수) / 행의 "역할 매핑" 버튼 클릭 시 우측 슬라이드 패널(해당 메뉴를 노출할 역할 체크박스 목록). 메뉴 생성·수정은 모달.
- **구성 요소**:
  | 요소 | 유형 | 설명 | 색상 |
  |------|------|------|------|
  | 메뉴 목록 표 | 표 | 그룹·메뉴명(한국어)·경로·아이콘·순서·노출 역할 수, 행 클릭 시 상세 아님(수정 버튼으로만 편집) | Neutral |
  | 메뉴 생성 버튼 | 버튼 | 생성 모달 오픈 | Base |
  | 메뉴 생성/수정 모달 | 모달 | 그룹(선택 또는 신규 입력, 그룹 선택 시 그룹 영문명도 함께 입력)·메뉴명(한국어)·메뉴 영문명·경로·아이콘(lucide 아이콘 이름 선택)·순서(숫자) 입력(사이드바 메뉴 i18n 미적용 결함 수정, 유지보수 요청 2026-07-13 — 메뉴 영문명·그룹 영문명 필드 신규) | Base |
  | 삭제 버튼 | 버튼 | 메뉴 삭제(확인 다이얼로그 동반) | Danger |
  | 역할 매핑 버튼 | 버튼 | 행별로 우측 역할 매핑 패널 오픈 | Base |
  | 역할 매핑 패널 | 슬라이드 패널+체크박스 리스트 | 전체 역할 목록에 체크박스, 체크된 역할만 해당 메뉴 노출 | Base(체크)/Neutral |
- **상태 · 인터랙션**:
  - 메뉴 생성/수정 시 그룹·메뉴명(한국어)·메뉴 영문명·경로는 필수. 그룹을 지정하면 그룹 영문명도 필수(신규 그룹 입력 시 함께 입력, 기존 그룹 선택 시 해당 그룹의 기존 영문명 자동 표시 후 필요 시 수정). 경로 중복 시 409 → 인라인 오류.
  - 삭제는 확인 다이얼로그(SCR-COM-009) 후 처리하며, 삭제된 메뉴는 모든 역할의 매핑에서도 함께 제거된다(사이드바에서 즉시 사라짐).
  - 역할 매핑 패널에서 체크박스를 켜거나 끄면 즉시 반영(저장 버튼 없이 토글마다 API 호출), 성공/실패 토스트 피드백.
  - 메뉴 순서(순서 컬럼) 값 오름차순으로 사이드바 그룹 내 정렬. 동일 그룹의 메뉴는 그룹명으로 묶여 표시.
  - System Admin만 접근(그 외 403). 모든 변경은 감사 로그(SCR-ADMIN-005, ROLE_CHANGE 계열 이벤트)에 기록.
- **컬럼 폭(유지보수 요청, 2026-07-14, SCR-COM-007 아키텍처 적용)**:
  | 컬럼 | 폭(px) | 근거 |
  |------|--------|------|
  | 그룹 | 110 | 그룹 라벨(짧은 한국어) |
  | 메뉴명 | 미지정(auto) | 메뉴명 길이 가변, 경로와 잔여 폭 분할. `truncate` 적용 |
  | 경로 | 미지정(auto) | URL 경로 길이 가변, 메뉴명과 잔여 폭 분할. `truncate` 적용 |
  | 아이콘 | 140 | 아이콘+lucide 아이콘명 텍스트 |
  | 순서 | 80 | 숫자, 우측 정렬 |
  | 노출 역할 수 | 130 | "전체 공개"/"N개 역할" 뱃지 |
  | 액션 | 260 | 버튼 3개(역할 매핑·수정·삭제), 우측 정렬 |
- **페이지당 아이템 수(유지보수 요청, 2026-07-14)**: 20건 → 16건([common.md](common.md) 7절 패턴 D).
- **연관 API**: 메뉴 CRUD API(신규, `api_spec/auth.md`), 역할-메뉴 매핑 API(신규, `api_spec/auth.md`)

### SCR-ADMIN-007 · 승인 프로세스 목록

- **목적**: 등록된 승인 프로세스 규칙을 도메인별로 조회하고 편집 화면(SCR-ADMIN-008)으로 진입.
- **레이아웃**: [SCR-COM-007] 목록 패턴. 상단 도메인 필터(셀렉트) + "프로세스 생성" 버튼 / 목록 표(규칙명·도메인·요청유형·요청자 역할·우선순위 tier·차수 수).
- **구성 요소**:
  | 요소 | 유형 | 설명 | 색상 |
  |------|------|------|------|
  | 도메인 필터 | 셀렉트 | SERVICE_REQUEST/CHANGE/KNOWLEDGE/INCIDENT/PROBLEM 등(API-AUTH-023 목록) | Border/Base |
  | 규칙 목록 표 | 표 | 규칙명·도메인·요청유형(전체/구체값)·요청자 역할 배지·우선순위 tier 배지(도메인 기본/요청유형 전용/요청자 역할 전용)·차수 수(0이면 "승인자 없음" 표시) | Neutral |
  | 프로세스 생성 버튼 | 버튼 | SCR-ADMIN-008(생성) 이동 | Base |
- **상태 · 인터랙션**: 행 클릭 시 SCR-ADMIN-008(편집)로 이동. 결과 0건 시 빈 상태 안내. System Admin만 접근(그 외 403).
- **연관 API**: `GET /api/v1/admin/approval-processes`(API-AUTH-025)

### SCR-ADMIN-008 · 승인 프로세스 생성/편집

- **목적**: 도메인·요청유형·승인 요청자·n차 승인자를 단계적으로 구성해 커스텀 승인 프로세스를 정의한다. 복잡도(다단계 플로우·역할 Drag&Drop·승인자 박스 재정렬)를 고려해 모달이 아닌 **별도 전용 페이지**로 구성한다.
- **레이아웃**: 상단 "규칙 정보" 카드(도메인·요청유형·규칙명·설명 — 메타데이터) / 그 아래 승인 단계 카드 스택(1→2단계 순서, 카드 사이 여유로운 간격(`space.300`)으로 직관적이고 여유 있게 구성). 완료된 단계는 상단에 요약으로 접히고, 진행 중 단계가 포커스된다. 우측 상단에 "생성 완료"(또는 편집 시 "저장") 버튼 고정.
  > **메타데이터 분리(유지보수 요청, 2026-07-13)**: 기존에는 도메인 선택(구 0단계)·요청 유형 선택(구 1단계)이 승인 단계 카드 스택 안에 포함되어 있었으나, 규칙명·설명과 같은 메타데이터임에도 승인 단계와 뒤섞여 있어 혼동을 유발했다. 이번 개편으로 두 항목을 "규칙 정보" 카드(규칙명·설명과 동일한 카드)로 이관하고, 승인 단계 카드 스택은 **승인 요청자부터 1단계로 재시작**한다(구 2단계 → 1단계, 구 3단계 → 2단계). 카드 내 필드 순서는 원래 선택 흐름(도메인→요청유형이 이름·설명보다 먼저 결정되던 항목)을 살려 **도메인·요청유형을 규칙명·설명보다 위에** 배치한다. 카드 스택 자체의 상호작용(드래그 앤 드롭 재정렬·역할 선택 패널·박스별 필수역할 검증·승인자 0개 확인 다이얼로그)은 변경하지 않는다.
- **구성 요소**:
  | 요소 | 유형 | 설명 | 색상 |
  |------|------|------|------|
  | 규칙 정보 카드 | 카드(폼) | 순서대로: 도메인 선택(필수, API-AUTH-023 목록) → 요청 유형 선택(선택한 도메인이 `hasRequestSubtype=true`일 때만 노출, "전체" 옵션 포함, API-AUTH-024 후보) → 규칙명(필수) → 설명(선택) | Base |
  | 1단계: 승인 요청자 박스 | 고정 카드(비-드래그) | 항상 1개, 드래그 핸들 없음·드롭 대상 아님을 시각적으로 구분(둥근 실선 테두리 대비 승인자 박스는 점선+드래그 핸들 아이콘). "역할 선택" 버튼 클릭 시 우측 슬라이드 패널 오픈. 선택된 역할은 박스 안에 칩으로 표시(1개 이상) | Info |
  | 역할 선택 사이드 패널 | 슬라이드 패널 | 상단 검색 input(버튼 없음, 프론트 contains 필터) + 전체 역할 목록. 목록 항목을 박스 영역으로 드래그(draggable)하면 해당 박스에 역할 칩 추가 | Base |
  | 2단계: 승인자 박스 스택 | 카드 반복(가변) | "승인자 추가" 버튼 클릭마다 새 박스가 스택 하단에 추가되며 실제로 카드가 늘어나는 애니메이션으로 표현(텍스트 카운터가 아님). 각 박스는 드래그 핸들 아이콘 보유(다른 승인자 박스 위에 드롭 시 두 박스 순서 교체). 박스 안에 "역할 선택" 버튼(1단계와 동일한 사이드 패널 재사용)·선택된 역할 칩·AND/OR 체크박스(역할 2개 이상일 때만 노출) | Base |
  | 승인자 추가 버튼 | 버튼 | 클릭마다 차수 박스 1개 추가, 10차 도달 시 비활성화(툴팁 "최대 10차까지 추가할 수 있습니다") | Base |
  | 차수 배지 | 배지 | 각 승인자 박스 좌상단에 "N차" 표시(드래그 앤 드롭으로 순서가 바뀌면 배지 번호도 즉시 갱신) | Info |
  | 승인자 없음 안내 | 안내 텍스트 | 승인자 박스가 0개인 채로 "생성 완료" 클릭 시 확인 문구("승인자 없이 바로 진행됩니다")를 다이얼로그로 표시 후 진행 | Warning |
  | 생성 완료/저장 버튼 | 버튼 | 전체 검증 후 API 호출 | Base |
  | 우선순위 충돌 오류 | 인라인 오류 | 409 응답 시 저장 실패 사유(겹치는 규칙명·역할 조합)를 폼 상단에 표시 | Danger |
- **상태 · 인터랙션**:
  - 규칙 정보 카드에서 도메인 미선택 시 승인 단계 카드 스택 비활성화.
  - 승인 요청자 박스는 드래그 시작 자체가 되지 않으며(핸들 미제공), 다른 박스의 드롭 대상으로도 반응하지 않는다(드롭 시 무시).
  - 승인자 박스 A를 박스 B 위로 드래그해 놓으면 두 박스의 순서(차수)가 교체된다(드래그 중 드롭 가능 영역 하이라이트).
  - 각 박스(요청자·승인자 공통)는 역할이 1개 이상 선택되어야 저장 가능(0개 상태로 "생성 완료" 시도 시 인라인 오류, 단 승인자 박스 자체가 0개인 것은 허용 — 요청자만 있는 프로세스).
  - 저장 성공 시 토스트 후 SCR-ADMIN-007 목록으로 복귀. 우선순위 충돌(같은 tier·스코프 중복) 시 409 인라인 오류로 저장 차단.
  - System Admin만 접근(그 외 403). 모든 생성·수정·삭제는 감사 로그(SCR-ADMIN-005, ROLE_CHANGE 계열과 별도로 승인 프로세스 변경 이벤트)에 기록.
