# 화면 설계서 — IT 인프라 모니터링 & 용량관리 (Infra Monitoring & Capacity Management)

> 도메인: infra-monitoring · 버전: 0.1 · 작성일: 2026-07-10

## 1. 개요

인프라 자산(HW/네트워크 장비)의 가동률·성능 지표 수동 등록·시계열 대시보드, 임계치 설정·초과 알림, SLA 대비 가동률 비교, 팀/서비스별 용량 계획, 인프라 지표 리포팅 화면을 정의한다.

## 2. 색상 팔레트

[common.md](common.md) 공통 팔레트를 사용한다.

| 구분 | 색상 (HEX) | 용도 |
|------|-----------|------|
| Base | #2563EB | 등록/저장 버튼 |
| Danger | #DC2626 | 임계치 초과·SLA 미달 |
| Warning | #D97706 | 용량 활용률 높음(역량 근접) |
| Success | #16A34A | SLA 목표 달성·정상 범위 |
| Info | #60A5FA | 시계열 차트·용량 계획 |

## 4. 화면 상세

### SCR-IOM-001 · 인프라 지표 등록

- **목적**: 운영 담당자가 인프라 자산의 가동률·성능 지표를 수동 입력.
- **레이아웃**: 폼 카드. 대상 자산 선택·측정 시각·지표 항목(가동률/CPU/메모리/응답시간)·값 / 등록 버튼.
- **구성 요소**:
  | 요소 | 유형 | 설명 | 색상 |
  |------|------|------|------|
  | 자산 선택 | 검색 셀렉트 | 자산(ITAM) 도메인 조회 | Border/Base |
  | 지표 항목 셀렉트 | 셀렉트 | UPTIME/CPU/MEMORY/RESPONSE_TIME | Border/Base |
  | 값 입력 | 입력 | 숫자 | Border/Base |
  | 등록 버튼 | 버튼 | 저장, 임계치 초과 시 알림 생성 | Base |
- **상태 · 인터랙션**: 자산·값 누락 시 400. 저장 시 임계치 초과 여부 즉시 판정해 초과 시 토스트 안내("임계치 초과 알림이 생성되었습니다").
- **연관 API**: `POST /api/v1/infra/metrics`

### SCR-IOM-002 · 지표 대시보드

- **목적**: 특정 자산의 지표를 시계열로 조회하고 SLA 목표 대비 실제 가동률을 비교.
- **레이아웃**: 상단 자산·기간 선택 / 지표별 시계열 차트 / SLA 대비 가동률 카드.
- **구성 요소**:
  | 요소 | 유형 | 설명 | 색상 |
  |------|------|------|------|
  | 시계열 차트 | 차트 | 측정 시각 순 지표값 | Info |
  | SLA 대비 가동률 카드 | 카드 | 목표%·실제%·달성 여부 | Success/Danger |
- **상태 · 인터랙션**: 조회 기간 내 지표 없으면 빈 결과 표시. 가동률 목표 미설정 자산은 실제 가동률만 표시.
- **연관 API**: `GET /api/v1/infra/metrics`, `GET /api/v1/infra/assets/{assetId}/uptime`

### SCR-IOM-003 · 임계치 설정·알림 목록

- **목적**: 지표 항목별 임계치(상한/하한)를 설정하고, 초과 알림을 확인·처리.
- **레이아웃**: 상단 임계치 설정 폼(지표 항목·상한·하한) / 하단 알림 목록(자산·지표·초과값·발생시각·확인 여부).
- **구성 요소**:
  | 요소 | 유형 | 설명 | 색상 |
  |------|------|------|------|
  | 임계치 설정 폼 | 입력 | 지표 항목별 상한/하한 | Border/Base |
  | 알림 목록 | 표 | 자산·지표·초과값·발생시각 | Danger |
  | 확인 처리 버튼 | 버튼 | 알림 확인 처리 | Base |
- **상태 · 인터랙션**: 임계치 미설정 지표 항목은 알림이 생성되지 않음을 안내. 확인 처리된 알림은 목록에서 흐리게 표시.
- **연관 API**: `GET/PUT /api/v1/infra/metric-thresholds`, `GET /api/v1/infra/metric-alerts`, `PATCH /api/v1/infra/metric-alerts/{id}/acknowledge`

### SCR-IOM-004 · 용량 계획 관리

- **목적**: 팀/서비스 단위로 처리 역량 대비 예상 수요를 등록·조회.
- **레이아웃**: 등록 폼(팀/서비스명·역량·예상 수요) / 목록(팀/서비스·역량·수요·활용률).
- **구성 요소**:
  | 요소 | 유형 | 설명 | 색상 |
  |------|------|------|------|
  | 등록 폼 | 입력 | 팀/서비스·역량·수요 | Border/Base |
  | 활용률 배지 | Lozenge | 수요/역량 비율 | Success/Warning/Danger |
- **상태 · 인터랙션**: 역량·수요 누락 시 400. 활용률 100% 초과 시 Danger 강조.
- **연관 API**: `POST /api/v1/infra/capacity-plans`, `GET /api/v1/infra/capacity-plans`

### SCR-IOM-005 · 인프라 지표 리포팅

- **목적**: 자산별·기간별 가동률·성능 지표 평균과 용량 활용률을 집계 조회.
- **레이아웃**: 상단 기간·자산 필터 / KPI 카드(평균 가동률·평균 성능 지표·평균 용량 활용률).
- **구성 요소**:
  | 요소 | 유형 | 설명 | 색상 |
  |------|------|------|------|
  | 기간·자산 필터 | 셀렉트/날짜 | 조회 조건 | Border/Base |
  | KPI 카드 | 카드 | 집계 수치 | Base |
- **상태 · 인터랙션**: 데이터 없으면 빈 결과 표시.
- **연관 API**: `GET /api/v1/infra/metrics/report`
