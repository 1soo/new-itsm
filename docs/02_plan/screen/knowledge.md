# 화면 설계서 — 지식 관리 (Knowledge)

> 도메인: knowledge · 버전: 0.3 · 작성일: 2026-07-14 · UI 축소 유지보수 요청(2026-07-14) 반영 — SCR-KM-001 목록 표 컬럼 폭 고정
>
> 이전 버전: 승인 프로세스 커스텀 기능(유지보수 요청) 반영 — 게이트키퍼 단일승인을 커스텀 프로세스로 대체. SCR-KM-004(검토·게시 승인함)는 [common.md](common.md) SCR-COM-014로 대체되어 제거

## 1. 개요

지식 기사 작성·편집, 상태 관리(초안/검토/게시), 커스텀 승인 프로세스 기반 게시 승인, 지식 검색, 셀프서비스 열람, 유용성 평가, 분류(카테고리/라벨), KCS 티켓 연계, 지표 대시보드 화면을 정의한다. 승인 대기·결정은 [common.md](common.md) SCR-COM-014(전 도메인 공용)를 사용한다.

## 4. 화면 상세

### SCR-KM-001 · 지식베이스 검색/목록

- **목적**: 키워드·분류로 기사를 검색·탐색.
- **레이아웃**: 상단 검색바 / 좌측 카테고리·라벨 필터 / 우측 결과 목록(제목·요약·상태·유용성 평점).
- **구성 요소**:
  | 요소 | 유형 | 설명 | 색상 |
  |------|------|------|------|
  | 검색바 | 입력 | 관련도 순 결과 | Border/Base |
  | 분류 필터 | 리스트 | 카테고리/라벨 | Base |
  | 상태 배지 | 배지 | 초안/검토/게시 | Muted/Warning/Success |
- **상태 · 인터랙션**: 매칭 없으면 빈 결과·무결과 검색 기록. 최종 사용자에게는 게시 기사만 노출. 기여자/게이트키퍼는 초안·검토 포함 조회.
- **컬럼 폭(유지보수 요청, 2026-07-14, [common.md](common.md) SCR-COM-007 아키텍처 적용)**:
  | 컬럼 | 폭(px) | 근거 |
  |------|--------|------|
  | 제목 | 미지정(auto) | 길이 가변, 요약과 잔여 폭 분할. `line-clamp-1` 적용(기존 유지) |
  | 요약 | 미지정(auto) | 길이 가변, 제목과 잔여 폭 분할. `line-clamp-1` 적용(기존 유지) |
  | 카테고리 | 110 | 카테고리명(짧음)/"-" |
  | 상태 | 110 | 상태 뱃지 |
  | 유용성 | 90 | "N%" 짧은 수치 |
- **페이지당 아이템 수(유지보수 요청, 2026-07-14)**: 10건 → 14건([common.md](common.md) 7절 패턴 B).
- **연관 API**: `GET /api/v1/knowledge/articles?keyword=&category=`

### SCR-KM-002 · 기사 열람(셀프서비스)

- **목적**: 게시된 기사를 열람하고 유용성 평가.
- **레이아웃**: 본문 영역(제목·카테고리·본문) / 하단 "도움이 되었나요?" 위젯·피드백 입력.
- **구성 요소**:
  | 요소 | 유형 | 설명 | 색상 |
  |------|------|------|------|
  | 기사 본문 | 문서 | 게시 내용 | Neutral |
  | 유용성 위젯 | 버튼(예/아니오)+코멘트 | 평가 저장·집계 | Success |
- **상태 · 인터랙션**: 미게시 기사에 최종 사용자 접근 시 차단(403). 미게시 기사 평가 거부. 평가 저장 시 토스트.
- **연관 API**: `GET /api/v1/knowledge/articles/{id}`, `POST .../feedback`

### SCR-KM-003 · 기사 작성·편집

- **목적**: 기여자가 기사를 작성·수정·삭제하고 검토를 요청.
- **레이아웃**: 좌측 에디터(제목·본문) / 우측 메타(카테고리·라벨·상태) / 상단 액션(저장·검토 요청·삭제).
- **구성 요소**:
  | 요소 | 유형 | 설명 | 색상 |
  |------|------|------|------|
  | 제목/본문 입력 | 입력/에디터 | 필수 | Border/Base |
  | 카테고리/라벨 | 셀렉트/태그 | 분류 지정 | Base |
  | 검토 요청 버튼 | 버튼 | 초안→검토 전이 | Warning |
  | 삭제 버튼 | 버튼 | 확인 다이얼로그 | Danger |
  | 승인 패널(공용) | 패널 | 검토 대기 중이면 차수 진행 상태(API-COM-004) 표시, 반려 시 사유 노출 | Info/Warning/Danger |
- **상태 · 인터랙션**: 제목/본문 비면 저장 거부. 허용되지 않은 상태 전이 거부. 미존재 카테고리 지정 시 400. "검토 요청" 클릭 시 매칭되는 승인 프로세스가 없으면 **즉시 게시(PUBLISHED)** 되고(승인 절차 없음 안내 토스트), 매칭되는 프로세스가 있으면 IN_REVIEW로 전환되어 승인 대기(진행 상태는 이 화면의 승인 패널로 조회, 처리는 [common.md](common.md) SCR-COM-014에서 수행). 승인 완료 시 PUBLISHED로, 반려 시 사유와 함께 DRAFT로 자동 복귀.
- **연관 API**: `POST/PATCH/DELETE /api/v1/knowledge/articles`, `PATCH .../status`, `GET /api/v1/approvals/{approvalRequestId}`(API-COM-004)

### SCR-KM-005 · 지식 지표 대시보드

- **목적**: 사용량·무결과 검색·유용성 평점·티켓 차단율 조회.
- **레이아웃**: 기간 필터 / KPI 카드(사용량·무결과 검색·유용성·차단율) / 무결과 키워드 랭킹.
- **구성 요소**:
  | 요소 | 유형 | 설명 | 색상 |
  |------|------|------|------|
  | KPI 카드 | 카드 | 핵심 지표(집계값) | Base |
  | 무결과 검색 목록 | 표 | 개선 후보 키워드(API topNoResultKeywords) | Warning |
- **상태 · 인터랙션**: 데이터 없으면 빈 결과. (시계열 추이는 범위 제외; API-KM-012 집계값·무결과 키워드 랭킹만 표시.)
- **연관 API**: `GET /api/v1/knowledge/metrics`
