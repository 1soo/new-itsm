## 1. 개요

자산 등록/조회/수정/폐기, 생애주기 관리, 유형/속성 분류, 만료 추적·알림, 구성 항목(CI) 및 CMDB 관계 관리, 티켓 연계, 영향 범위 조회, 지표 대시보드 화면을 정의한다.

## 2. 색상 팔레트

## 4. 화면 상세

### SCR-ITAM-001 · 자산 목록

- **목적**: 자산을 필터·검색하고 상세로 진입.
- **레이아웃**: [SCR-COM-007] 목록 패턴. 필터(유형·상태·소유자·만료 임박·기간) / 표(식별키·이름·유형·상태·소유자·만료일) / "자산 등록" 버튼.
- **구성 요소**:
  | 요소 | 유형 | 설명 | 색상 |
  |------|------|------|------|
  | 유형 배지 | 배지 | HW/SW/클라우드 | Base 계열 |
  | 상태 배지 | 배지 | 생애주기 단계 | 상태 시맨틱 |
  | 만료 배지 | 배지 | 임박/경과 | Warning/Danger |
- **상태 · 인터랙션**: 행 클릭 시 상세. 만료 임박 필터로 대상 자산 조회.
- **컬럼 폭(유지보수 요청, 2026-07-14, [common.md](common.md) SCR-COM-007 아키텍처 적용)**:
  | 컬럼 | 폭(px) | 근거 |
  |------|--------|------|
  | 식별키 | 130 | 코드(assetKey) |
  | 이름 | 미지정(auto) | 길이 가변, 잔여 폭 흡수. `line-clamp-1` 적용(기존 유지) |
  | 유형 | 110 | 유형 뱃지 |
  | 상태 | 110 | 상태 뱃지 |
  | 소유자 | 120 | 사람 이름/"미지정" |
  | 만료일 | 150 | 날짜(YYYY-MM-DD)+만료 임박/경과 뱃지 병기 |
- **페이지당 아이템 수(유지보수 요청, 2026-07-14)**: 10건 → 14건([common.md](common.md) 7절 패턴 B).
- **연관 API**: `GET /api/v1/assets`

### SCR-ITAM-002 · 자산 등록/수정

- **목적**: 자산을 유형별 속성으로 등록·수정.
- **레이아웃**: 폼 카드. 이름(필수)·유형(필수)·소유자·위치·상태·구매일·비용·계약/라이선스/보증 만료일. 유형 선택 시 유형별 속성 필드 동적 표시.
- **구성 요소**:
  | 요소 | 유형 | 설명 | 색상 |
  |------|------|------|------|
  | 유형 셀렉트 | 셀렉트 | HW/SW/클라우드 필수 | Base |
  | 유형별 속성 | 동적 입력 | 유형 스키마 기반 | Border/Base |
  | 만료일 입력 | 날짜 | 라이선스/보증/계약 | Border/Base |
  | 저장 버튼 | 버튼 | 생성/수정 | Base |
- **상태 · 인터랙션**: 이름·유형·유형별 필수 속성 누락 시 400. 만료일 과거 입력 시 경고. 자산 관리 권한 없으면 403.
- **연관 API**: `POST/PATCH /api/v1/assets`

### SCR-ITAM-003 · 자산 상세

- **목적**: 자산 생애주기·만료·연계를 관리.
- **레이아웃**: 상단 요약(이름·유형·상태) / 좌측 속성·생애주기 이력 / 우측 메타(만료·연결 티켓·연결 CI).
- **구성 요소**:
  | 요소 | 유형 | 설명 | 색상 |
  |------|------|------|------|
  | 생애주기 단계 전이 | 버튼 | 계획~유지보수(비종료 단계) 허용 전이. 라벨은 동작 동사형([common.md](common.md) SCR-COM-008 아키텍처 적용) — 아래 표 | Base |
  | 폐기 버튼 | 버튼 | 확인 다이얼로그·상태=폐기(라벨 "폐기"는 도착 상태명과 동작 동사형이 동일) | Danger |
  | 만료 정보 | 표시 | 임박/경과 강조 | Warning/Danger |
  | 티켓 연계 | 버튼 | 요청·인시던트·문제·변경 링크 | Base |
  | 승인 패널(공용) | 패널 | 폐기 전이에 매칭되는 승인 프로세스가 있으면 차수 진행 상태(API-COM-004) 표시, 없으면 패널 자체를 노출하지 않고 기존처럼 즉시 전이 | Info/Warning/Success |
- **상태 · 인터랙션**: 정의되지 않은 단계 전이 거부. 미존재 티켓 연계 시 400. 이력 기록. 폐기 시도 시 매칭되는 승인 프로세스가 있으면 409와 함께 승인 패널이 나타나며, 처리는 [common.md](common.md) SCR-COM-014에서 수행한다.
  - **전이 버튼 라벨**: `status.ts`의 `transitionLabel(t, target)`(i18n 키 `asset:transition.*`)이 버튼 텍스트를 결정한다(`statusLabel(t, target)`은 배지·토스트 문구에 별도로 사용).
    | 도착 상태 | 버튼 라벨 |
    |-----------|-----------|
    | PLANNING | 계획 수립 |
    | PROCUREMENT | 구매 진행 |
    | OPERATION | 운영 전환 |
    | MAINTENANCE | 유지보수 전환 |
- **연관 API**: `GET /api/v1/assets/{id}`, `PATCH .../lifecycle`, `POST .../links`, `PATCH .../retire`, `GET /api/v1/approvals/{approvalRequestId}`(API-COM-004)

### SCR-ITAM-004 · CI·CMDB 관계 뷰

- **목적**: CI 등록·의존 관계 관리 및 영향 범위 조회.
- **레이아웃**: 좌측 CI 목록 / 우측 관계 그래프(선택 CI 중심 의존 노드) / 관계 추가 폼.
- **구성 요소**:
  | 요소 | 유형 | 설명 | 색상 |
  |------|------|------|------|
  | CI 목록 | 리스트 | CI 검색·선택 | Neutral |
  | 관계 그래프 | 그래프 | 의존 관계 시각화 | Info |
  | 관계 추가 폼 | 폼 | 대상 CI·관계 유형 | Base |
  | 영향 범위 패널 | 리스트 | 연결된 CI 목록 | Info |
- **상태 · 인터랙션**: 미존재 CI와 관계 맺기 시 400. 관계 없는 CI 영향 조회 시 빈 목록.
- **연관 API**: `GET/POST /api/v1/cis`, `POST /api/v1/cis/{id}/relations`, `GET /api/v1/cis/{id}/impact`

### SCR-ITAM-005 · 자산 지표 대시보드

- **목적**: 활용률·만료 임박 건수·유형별 분포 조회.
- **레이아웃**: 기간 필터 / KPI 카드(활용률·만료 임박) / 유형 분포 차트.
- **구성 요소**:
  | 요소 | 유형 | 설명 | 색상 |
  |------|------|------|------|
  | KPI 카드 | 카드 | 활용률·만료 임박 | Base/Warning |
  | 유형 분포 차트 | 차트 | HW/SW/클라우드 분포 | Base 계열 |
- **상태 · 인터랙션**: 데이터 없으면 빈 결과.
- **연관 API**: `GET /api/v1/assets/metrics`
