# 화면 설계서 — 서비스 요청 관리 (Service Request)

> 도메인: service-request · 버전: 0.2 · 작성일: 2026-07-11 · 승인 프로세스 커스텀 기능(유지보수 요청) 반영 — 카탈로그 관리의 "승인 필요 토글" 제거, 상세 화면 승인 상태를 공통 승인 패널로 교체. SCR-SRM-006(승인 대기함)은 [common.md](common.md) SCR-COM-014로 대체되어 제거

## 1. 개요

셀프서비스 포털에서의 요청 제출(동적 양식·지식 추천), 요청 큐/상세(검증·라우팅·커스텀 승인·이행·검증·종료), SLA·코멘트·CSAT, 서비스 카탈로그 관리, 지표 리포팅 화면을 정의한다. 승인 대기·결정은 [common.md](common.md) SCR-COM-014(전 도메인 공용)를 사용한다.

## 2. 색상 팔레트

[common.md](common.md) 공통 팔레트를 사용한다.

| Success | #16A34A | 이행 완료·종료·SLA 준수 |
| Warning | #D97706 | 승인 대기·SLA 임박 |

## 4. 화면 상세

### SCR-SRM-001 · 서비스 포털(카탈로그 브라우즈)

- **목적**: 요청자가 카탈로그에서 요청 유형을 탐색·선택.
- **레이아웃**: 상단 검색바 / 카테고리 그룹 카드 그리드 / 각 카드에 요청 유형명·설명.
- **구성 요소**:
  | 요소 | 유형 | 설명 | 색상 |
  |------|------|------|------|
  | 카탈로그 카드 | 카드 | 요청 유형 선택 진입 | Base(강조) |
  | 검색바 | 입력 | 유형/키워드 검색 | Border/Base |
- **상태 · 인터랙션**: 카드 클릭 시 SCR-SRM-002 이동. 검색 시 관련 지식 기사 추천 배너 노출(있을 때만).
- **연관 API**: `GET /api/v1/service-catalog/items`

### SCR-SRM-002 · 요청 제출(동적 양식)

- **목적**: 선택한 요청 유형의 동적 양식을 작성해 제출.
- **레이아웃**: 좌측 동적 폼(요청 유형 스키마 기반) / 우측 "관련 지식 기사" 추천 패널 / 하단 제출 버튼.
- **구성 요소**:
  | 요소 | 유형 | 설명 | 색상 |
  |------|------|------|------|
  | 동적 필드 | 입력/셀렉트/첨부 | 유형별 양식 스키마 렌더 | Border/Base |
  | 지식 추천 패널 | 리스트 | 제목/유형 기반 기사 추천(deflection) | Variation 2 |
  | 제출 버튼 | 버튼 | 요청 생성 | Base |
- **상태 · 인터랙션**: 필수 필드 미입력 시 제출 차단·인라인 오류. 추천 기사 없으면 패널 숨김. 제출 성공 시 접수번호 토스트·상세 이동.
- **연관 API**: `GET /api/v1/service-catalog/items/{id}`, `GET /api/v1/knowledge/suggestions`, `POST /api/v1/service-requests`

### SCR-SRM-003 · 내 요청 목록

- **목적**: 요청자가 본인 요청 상태를 추적.
- **레이아웃**: [SCR-COM-007] 목록 패턴. 필터(상태·기간) / 표(접수번호·유형·상태·SLA·갱신일).
- **구성 요소**:
  | 요소 | 유형 | 설명 | 색상 |
  |------|------|------|------|
  | 상태 배지 | 배지 | 제출/승인대기/이행중/종료 | 상태 시맨틱 |
  | SLA 배지 | 배지 | 준수/임박/위반 | Success/Warning/Danger |
- **상태 · 인터랙션**: 본인 요청만 조회. 행 클릭 시 상세.
- **연관 API**: `GET /api/v1/service-requests?scope=mine`

### SCR-SRM-004 · 요청 큐(상담원)

- **목적**: 상담원이 큐별 요청을 배정받아 처리.
- **레이아웃**: 좌측 큐 목록 / 우측 [SCR-COM-007] 요청 표. 미분류 큐 표시.
- **구성 요소**:
  | 요소 | 유형 | 설명 | 색상 |
  |------|------|------|------|
  | 큐 목록 | 리스트 | 큐별 건수 | Neutral |
  | 배정 버튼 | 버튼 | 나에게 배정 | Base |
  | 미분류 표시 | 배지 | 매칭 큐 없음 | Warning |
- **상태 · 인터랙션**: 권한 없는 사용자는 배정 불가(403). 배정 시 담당자 갱신.
- **연관 API**: `GET /api/v1/service-requests?queue=`, `POST /api/v1/service-requests/{id}/assign`

### SCR-SRM-005 · 요청 상세

- **목적**: 요청 처리 전 과정(검증·승인 상태·이행·검증/종료·코멘트·SLA·CSAT) 관리.
- **레이아웃**: [SCR-COM-008] 상세 패턴. 본문(양식 값)·코멘트·타임라인 / 우측 메타(상태·담당자·SLA 타이머·승인 패널(공용)·연결 지식).
- **구성 요소**:
  | 요소 | 유형 | 설명 | 색상 |
  |------|------|------|------|
  | 상태 전이 버튼 | 버튼 | 이행완료/종료 등 허용 전이 | Base |
  | 승인 패널(공용) | 패널 | 매칭되는 승인 프로세스가 있으면 차수 진행 상태(API-COM-004)·반려 사유 표시, 없으면 "이 요청에는 승인 절차가 없습니다" 안내 | Info/Warning/Success/Danger |
  | SLA 타이머 | 배지 | 응답·해결 경과/위반 | Success/Warning/Danger |
  | 코멘트 입력 | 입력 | 요청자-상담원 소통 | Border/Base |
  | CSAT 위젯 | 별점/버튼 | 종료 시 요청자 노출 | Success |
- **상태 · 인터랙션**: 승인 대기 중에는 이행 전이 버튼 숨김(매칭되는 프로세스가 없으면 게이트 없이 진행). 이미 종료된 요청 재종료 차단. CSAT는 종료 후 요청자에게만 노출. 권한 없는 코멘트/이행 시 403. 승인 대기 중인 건의 처리는 [common.md](common.md) SCR-COM-014에서 수행(이 화면은 진행 상태 조회 전용).
- **연관 API**: `GET /api/v1/service-requests/{id}`, `PATCH .../status`, `POST .../comments`, `POST .../csat`, `GET /api/v1/approvals/{approvalRequestId}`(API-COM-004)

### SCR-SRM-007 · 서비스 카탈로그 관리

- **목적**: 프로세스 오너가 요청 유형(양식·SLA·큐)을 정의·정리. 승인 여부·경로 설정은 이 화면이 아닌 SCR-ADMIN-008(승인 프로세스 생성/편집, SYSTEM_ADMIN 전용)에서 도메인=SERVICE_REQUEST·요청유형=이 카탈로그 항목으로 별도 구성한다(승인 프로세스 커스텀 기능으로 완전 대체).
- **레이아웃**: 카탈로그 항목 목록 / 항목 편집 폼(이름·설명·양식 필드 빌더·담당 큐·SLA 목표).
- **구성 요소**:
  | 요소 | 유형 | 설명 | 색상 |
  |------|------|------|------|
  | 양식 필드 빌더 | 반복 입력 | 동적 필드 정의(라벨·유형·필수) | Border/Base |
  | SLA 목표 입력 | 입력 | 응답·해결 시간 | Border/Base |
- **상태 · 인터랙션**: 이름·양식 누락 시 400. 프로세스 오너 권한 없으면 403.
- **연관 API**: `GET/POST/PATCH /api/v1/service-catalog/items`

### SCR-SRM-008 · 요청 지표 대시보드

- **목적**: CSAT·응답/해결시간·SLA 준수율 집계 조회.
- **레이아웃**: 상단 기간 필터 / KPI 카드(CSAT·평균 응답·평균 해결·SLA 준수율).
- **구성 요소**:
  | 요소 | 유형 | 설명 | 색상 |
  |------|------|------|------|
  | 기간 필터 | 셀렉트/날짜 | 조회 기간 선택 | Border/Base |
  | KPI 카드 | 카드 | 핵심 지표 수치(집계값) | Base |
- **상태 · 인터랙션**: 데이터 없으면 0값 표시. (REQ-SRM-012는 집계·조회만 요구하므로 시계열 추이 차트는 이번 범위 제외.)
- **연관 API**: `GET /api/v1/service-requests/metrics`
