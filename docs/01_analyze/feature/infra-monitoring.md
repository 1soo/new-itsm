## 1. 개요

인프라 자산의 가동률·성능 지표 수동 등록·시계열 조회, 임계치 알림, 팀/서비스별 용량 계획, SLA 대비 가동률 비교, 지표 리포팅 기능을 제공한다.

## 3. 기능 상세

### FEAT-IOM-001 · 지표 수동 등록

- **설명**: 운영 담당자가 인프라 자산의 가동률·성능 지표를 수동으로 입력한다.
- **관련 요구사항**: REQ-IOM-001
- **입력**: 대상 자산, 측정 시각, 지표값(가동률/CPU/메모리/응답시간)
- **출력**: 생성된 지표 레코드
- **처리 흐름**:
  1. 필수 값 검증
  2. 지표 레코드 저장
- **인수 기준 (EARS)**:
  - (Event-driven) **WHEN** 유효한 지표값을 입력하면, 시스템은 지표 레코드를 생성해야 한다.
- **예외/오류 처리 (EARS · Unwanted Behaviour)**:
  - **IF** 대상 자산 또는 지표값이 누락되면, **THEN** 시스템은 등록을 거부해야 한다.

### FEAT-IOM-002 · 지표 대시보드 조회

- **설명**: 특정 자산의 지표를 시계열로 조회한다.
- **관련 요구사항**: REQ-IOM-002
- **입력**: 자산 ID, 조회 기간
- **출력**: 측정 시각 순 지표 목록
- **처리 흐름**:
  1. 기간 내 지표 조회
  2. 시각 순 정렬 반환
- **인수 기준 (EARS)**:
  - (Event-driven) **WHEN** 사용자가 대시보드를 조회하면, 시스템은 지표를 측정 시각 순으로 표시해야 한다.
- **예외/오류 처리 (EARS · Unwanted Behaviour)**:
  - **IF** 조회 기간 내 지표가 없으면, **THEN** 시스템은 빈 결과를 표시해야 한다.

### FEAT-IOM-003 · 임계치 알림

- **설명**: 지표값이 설정된 임계치를 벗어나면 담당자에게 알림을 생성한다.
- **관련 요구사항**: REQ-IOM-003
- **입력**: 지표 항목별 임계치(상한/하한), 등록되는 지표값
- **출력**: 알림
- **처리 흐름**:
  1. 지표 등록 시 임계치 비교
  2. 초과 시 알림 생성
- **인수 기준 (EARS)**:
  - (Ubiquitous) 시스템은 지표 항목별 임계치를 저장해야 한다.
  - (Event-driven) **WHEN** 지표값이 임계치를 벗어나면, 시스템은 알림을 생성해야 한다.
- **예외/오류 처리 (EARS · Unwanted Behaviour)**:
  - **IF** 임계치가 설정되지 않은 지표 항목이면, **THEN** 시스템은 알림을 생성하지 않아야 한다.

### FEAT-IOM-004 · 용량 계획 관리

- **설명**: 팀/서비스 단위로 처리 역량 대비 수요를 등록·조회한다.
- **관련 요구사항**: REQ-IOM-004
- **입력**: 팀/서비스, 처리 역량, 예상 수요
- **출력**: 용량 계획, 역량 대비 수요 비율
- **처리 흐름**:
  1. 역량·수요 값 입력
  2. 용량 계획 저장
  3. 조회 시 비율 계산
- **인수 기준 (EARS)**:
  - (Event-driven) **WHEN** 역량·수요를 입력하면, 시스템은 용량 계획을 생성해야 한다.
  - (Event-driven) **WHEN** 용량 계획을 조회하면, 시스템은 역량 대비 수요 비율을 표시해야 한다.
- **예외/오류 처리 (EARS · Unwanted Behaviour)**:
  - **IF** 역량 또는 수요 값이 누락되면, **THEN** 시스템은 저장을 거부해야 한다.

### FEAT-IOM-005 · SLA 대비 가동률 비교

- **설명**: 인프라 자산의 가동률 목표(SLA) 대비 실제 가동률을 비교해 표시한다.
- **관련 요구사항**: REQ-IOM-005
- **입력**: 자산 ID, 가동률 목표
- **출력**: 목표 대비 실제 가동률
- **처리 흐름**:
  1. 자산의 등록된 가동률 지표 조회
  2. 목표값과 비교해 반환
- **인수 기준 (EARS)**:
  - (Ubiquitous) 시스템은 자산별 가동률 목표를 저장해야 한다.
  - (Event-driven) **WHEN** 가동률 현황을 조회하면, 시스템은 목표 대비 실제 가동률을 표시해야 한다.
- **예외/오류 처리 (EARS · Unwanted Behaviour)**:
  - **IF** 목표가 설정되지 않은 자산이면, **THEN** 시스템은 실제 가동률만 표시해야 한다.

### FEAT-IOM-006 · 인프라 지표 리포팅

- **설명**: 자산별·기간별 가동률·성능 지표 평균과 용량 활용률을 집계한다.
- **관련 요구사항**: REQ-IOM-006
- **입력**: 조회 기간·필터
- **출력**: 가동률·성능 지표 평균, 용량 활용률
- **처리 흐름**:
  1. 지표·용량 계획 데이터 집계
  2. 지표 계산·반환
- **인수 기준 (EARS)**:
  - (Ubiquitous) 시스템은 가동률·성능 지표 평균과 용량 활용률을 조회할 수 있어야 한다.
- **예외/오류 처리 (EARS · Unwanted Behaviour)**:
  - **IF** 데이터가 없으면, **THEN** 시스템은 빈 결과를 반환해야 한다.
