# 기능 명세서 — 컴플라이언스 관리 (Compliance Management)

> 도메인: compliance · 버전: 0.1 · 작성일: 2026-07-10

## 1. 개요

컴플라이언스 요구사항 등록·책임자 지정·시정조치 추적·감사 로그·변경 관리 연계·준수 현황 리포팅 기능을 제공한다.

## 3. 기능 상세

### FEAT-COMP-001 · 요구사항 관리

- **설명**: 컴플라이언스 담당자가 요구사항을 등록·조회·수정한다.
- **관련 요구사항**: REQ-COMP-001
- **입력**: 이름, 근거(규제·정책), 적용 범위
- **출력**: 생성/수정된 요구사항(식별키)
- **처리 흐름**:
  1. 필수 필드 검증
  2. 요구사항 저장
- **인수 기준 (EARS)**:
  - (Event-driven) **WHEN** 유효한 정보로 등록하면, 시스템은 요구사항을 생성해야 한다.
- **예외/오류 처리 (EARS · Unwanted Behaviour)**:
  - **IF** 이름·근거가 누락되면, **THEN** 시스템은 생성을 거부해야 한다.

### FEAT-COMP-002 · 책임자 지정

- **설명**: 요구사항에 책임자(오너)를 지정한다.
- **관련 요구사항**: REQ-COMP-002
- **입력**: 요구사항 ID, 책임자
- **출력**: 지정 결과
- **처리 흐름**:
  1. 책임자 지정
  2. 지정 이력 기록
- **인수 기준 (EARS)**:
  - (Event-driven) **WHEN** 책임자를 지정하면, 시스템은 지정 내역을 기록·표시해야 한다.
- **예외/오류 처리 (EARS · Unwanted Behaviour)**:
  - **IF** 책임자가 미지정 상태이면, **THEN** 시스템은 목록에서 "책임자 미지정"으로 표시해야 한다.

### FEAT-COMP-003 · 시정조치 추적

- **설명**: 요구사항 위반/이슈를 시정조치 항목으로 등록하고 상태를 추적한다.
- **관련 요구사항**: REQ-COMP-003
- **입력**: 시정조치 내용, 상태 전이 요청
- **출력**: 시정조치 상태·이력
- **처리 흐름**:
  1. 시정조치 등록(탐지)
  2. 조치중 전이
  3. 해결 전이
- **인수 기준 (EARS)**:
  - (Ubiquitous) 시스템은 시정조치를 탐지·조치중·해결 상태로 관리해야 한다.
  - (Event-driven) **WHEN** 상태를 전이하면, 시스템은 이력을 기록해야 한다.
- **예외/오류 처리 (EARS · Unwanted Behaviour)**:
  - **IF** 정의되지 않은 상태로 전이하려 하면, **THEN** 시스템은 거부해야 한다.

### FEAT-COMP-004 · 감사 추적 로그

- **설명**: 컴플라이언스 관련 활동을 감사 로그에 기록한다.
- **관련 요구사항**: REQ-COMP-004
- **입력**: 수행 작업 유형, 수행자, 시각
- **출력**: 감사 로그 항목
- **처리 흐름**:
  1. 대상 작업 수행
  2. 감사 로그에 수행자·시각과 함께 기록
- **인수 기준 (EARS)**:
  - (Ubiquitous) 시스템은 요구사항 등록·수정·상태 전이·시정조치 처리를 감사 로그에 기록해야 한다.
- **예외/오류 처리 (EARS · Unwanted Behaviour)**:
  - **IF** 감사 로그 기록에 실패하면, **THEN** 시스템은 원본 작업도 실패로 처리해야 한다.

### FEAT-COMP-005 · 변경 연계

- **설명**: 변경 요청과 컴플라이언스 요구사항을 연결한다.
- **관련 요구사항**: REQ-COMP-005
- **입력**: 변경 요청 ID, 요구사항 ID
- **출력**: 저장된 링크
- **처리 흐름**:
  1. 변경 요청 존재 확인
  2. 링크 저장
- **인수 기준 (EARS)**:
  - (Event-driven) **WHEN** 변경 요청을 요구사항에 연결하면, 시스템은 링크를 저장하고 변경 요청 상세에서 조회 가능하게 해야 한다.
- **예외/오류 처리 (EARS · Unwanted Behaviour)**:
  - **IF** 존재하지 않는 변경 요청과 연결하려 하면, **THEN** 시스템은 400 오류를 반환해야 한다.

### FEAT-COMP-006 · 준수 현황 리포팅

- **설명**: 요구사항별 상태·미해결 시정조치 건수·전체 준수율을 집계한다.
- **관련 요구사항**: REQ-COMP-006
- **입력**: 조회 기간·필터
- **출력**: 요구사항별 상태, 미해결 건수, 준수율
- **처리 흐름**:
  1. 요구사항·시정조치 데이터 집계
  2. 지표 계산·반환
- **인수 기준 (EARS)**:
  - (Ubiquitous) 시스템은 요구사항별 상태·미해결 시정조치 건수·전체 준수율을 조회할 수 있어야 한다.
- **예외/오류 처리 (EARS · Unwanted Behaviour)**:
  - **IF** 데이터가 없으면, **THEN** 시스템은 빈 결과를 반환해야 한다.
