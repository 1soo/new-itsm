# 기능 명세서 — 지식 관리 (Knowledge Management)

> 도메인: knowledge · 버전: 0.1 · 작성일: 2026-07-09

## 1. 개요

## 3. 기능 상세

### FEAT-KM-001 · 기사 작성·편집

- **설명**: 기여자가 지식 기사를 작성·수정·삭제한다.
- **관련 요구사항**: REQ-KM-001
- **입력**: 제목, 본문, 카테고리/라벨
- **출력**: 생성/수정된 기사(초안)
- **처리 흐름**:
  1. 입력값 검증
  2. 초안 저장
- **인수 기준 (EARS)**:
  - (Event-driven) **WHEN** 기사를 작성·저장하면, 시스템은 초안을 생성해야 한다.
- **예외/오류 처리 (EARS · Unwanted Behaviour)**:
  - **IF** 제목/본문이 비었으면, **THEN** 시스템은 저장을 거부해야 한다.

### FEAT-KM-002 · 기사 상태 관리

- **설명**: 기사를 초안→검토→게시로 전이한다.
- **관련 요구사항**: REQ-KM-002
- **입력**: 기사 ID, 목표 상태
- **출력**: 갱신된 상태
- **처리 흐름**:
  1. 전이 허용 여부 확인
  2. 상태 전이
- **인수 기준 (EARS)**:
  - (Ubiquitous) 시스템은 기사 상태를 관리해야 한다.
  - (Event-driven) **WHEN** 검토를 요청하면, 시스템은 검토 상태로 전이해야 한다.
- **예외/오류 처리 (EARS · Unwanted Behaviour)**:
  - **IF** 허용되지 않은 전이면, **THEN** 시스템은 거부해야 한다.

### FEAT-KM-003 · 검토·게시 승인

- **설명**: 게이트키퍼가 기사를 검토·게시 승인 또는 반려한다.
- **관련 요구사항**: REQ-KM-003
- **입력**: 기사 ID, 승인/반려·사유
- **출력**: 게시 또는 초안 복귀
- **처리 흐름**:
  1. 게이트키퍼 권한 확인
  2. 승인 시 게시, 반려 시 초안 복귀
- **인수 기준 (EARS)**:
  - (Event-driven) **WHEN** 게이트키퍼가 승인하면, 시스템은 기사를 게시해야 한다.
- **예외/오류 처리 (EARS · Unwanted Behaviour)**:
  - **IF** 게이트키퍼 권한이 없으면, **THEN** 시스템은 403을 반환해야 한다.
  - **IF** 반려되면, **THEN** 시스템은 기사를 초안으로 되돌리고 사유를 표시해야 한다.

### FEAT-KM-004 · 지식 검색

- **설명**: 키워드로 지식베이스를 검색한다.
- **관련 요구사항**: REQ-KM-004
- **입력**: 검색 키워드
- **출력**: 관련도 순 기사 목록
- **처리 흐름**:
  1. 키워드 매칭
  2. 관련도 순 반환
- **인수 기준 (EARS)**:
  - (Event-driven) **WHEN** 키워드로 검색하면, 시스템은 매칭 기사를 관련도 순으로 반환해야 한다.
- **예외/오류 처리 (EARS · Unwanted Behaviour)**:
  - **IF** 매칭 기사가 없으면, **THEN** 시스템은 빈 결과를 반환하고 무결과 검색으로 기록해야 한다.

### FEAT-KM-005 · 셀프서비스 열람

- **설명**: 최종 사용자가 포털에서 게시된 기사를 열람한다.
- **관련 요구사항**: REQ-KM-005
- **입력**: 기사 ID/검색
- **출력**: 게시 기사 내용
- **처리 흐름**:
  1. 게시 상태 확인
  2. 기사 표시
- **인수 기준 (EARS)**:
  - (State-driven) **WHILE** 기사가 게시 상태인 동안, 시스템은 최종 사용자 열람을 허용해야 한다.
- **예외/오류 처리 (EARS · Unwanted Behaviour)**:
  - **IF** 미게시 기사에 최종 사용자가 접근하면, **THEN** 시스템은 접근을 차단해야 한다.

### FEAT-KM-006 · 유용성 평가

- **설명**: 사용자가 기사 유용성을 평가한다.
- **관련 요구사항**: REQ-KM-006
- **입력**: 기사 ID, 평가(도움됨/안됨), 코멘트
- **출력**: 저장된 평가·집계
- **처리 흐름**:
  1. 평가 저장
  2. 집계 반영
- **인수 기준 (EARS)**:
  - (Event-driven) **WHEN** 유용성을 평가하면, 시스템은 평가를 저장·집계해야 한다.
- **예외/오류 처리 (EARS · Unwanted Behaviour)**:
  - **IF** 미게시 기사에 평가하면, **THEN** 시스템은 이를 거부해야 한다.

### FEAT-KM-007 · 분류(카테고리/라벨)

- **설명**: 기사를 카테고리·라벨로 분류·조회한다.
- **관련 요구사항**: REQ-KM-007
- **입력**: 기사 ID, 카테고리/라벨
- **출력**: 분류된 기사, 분류별 목록
- **처리 흐름**:
  1. 분류 지정 저장
  2. 분류별 조회 지원
- **인수 기준 (EARS)**:
  - (Event-driven) **WHEN** 카테고리·라벨을 지정하면, 시스템은 분류를 저장하고 분류별 조회를 지원해야 한다.
- **예외/오류 처리 (EARS · Unwanted Behaviour)**:
  - **IF** 존재하지 않는 카테고리를 지정하면, **THEN** 시스템은 400 오류를 반환해야 한다.

### FEAT-KM-008 · KCS 티켓 연계

- **설명**: 요청/인시던트/문제 처리 중 기사를 작성하거나 연결한다.
- **관련 요구사항**: REQ-KM-008
- **입력**: 원 티켓 ID, (신규 기사 내용 또는 기존 기사 ID)
- **출력**: 생성/연결된 기사, 티켓-기사 링크
- **처리 흐름**:
  1. 신규 기사 작성 또는 기존 기사 선택
  2. 원 티켓과 링크
- **인수 기준 (EARS)**:
  - (Event-driven) **WHEN** 티켓에서 기사 작성/연결을 실행하면, 시스템은 기사를 생성/연결하고 링크해야 한다.
- **예외/오류 처리 (EARS · Unwanted Behaviour)**:
  - **IF** 존재하지 않는 티켓에 연결하면, **THEN** 시스템은 400 오류를 반환해야 한다.

### FEAT-KM-009 · 지표 리포팅

- **설명**: 지식 지표를 집계·조회한다.
- **관련 요구사항**: REQ-KM-009
- **입력**: 조회 기간·필터
- **출력**: 사용량·무결과 검색·유용성 평점·차단율
- **처리 흐름**:
  1. 데이터 집계
  2. 지표 계산·반환
- **인수 기준 (EARS)**:
  - (Ubiquitous) 시스템은 기사 사용량·무결과 검색·유용성 평점·티켓 차단율을 조회할 수 있어야 한다.
- **예외/오류 처리 (EARS · Unwanted Behaviour)**:
  - **IF** 데이터가 없으면, **THEN** 시스템은 빈 결과를 반환해야 한다.
