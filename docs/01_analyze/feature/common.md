## 1. 개요

전 도메인을 가로지르는 공통 헤더/레이아웃 기능(알림 팝오버, 사용자 가이드)을 제공한다.

## 2. 기능 목록

| ID | 기능명 | 관련 요구사항 | 설명 |
|----|--------|---------------|------|
| FEAT-COM-001 | 헤더 알림 팝오버 | REQ-COM-001 | 알림 벨 클릭 시 최근 알림 목록을 도메인·내용·상세보기 버튼과 함께 팝오버로 표시 |
| FEAT-COM-002 | 사용자 가이드 | REQ-COM-002 | 헤더 "?" 아이콘 클릭 시 전용 화면으로 이동해 개요·도메인 및 원칙·역할별 수행 내용과 방법을 표시 |

## 3. 기능 상세

### FEAT-COM-001 · 헤더 알림 팝오버

- **설명**: 헤더 알림 벨을 클릭하면 여러 도메인의 최근 알림을 모아 팝오버 목록으로 보여주고, 각 알림에서 해당 상세 화면으로 이동할 수 있다.
- **관련 요구사항**: REQ-COM-001
- **입력**: 로그인 사용자
- **출력**: 알림 목록(도메인·내용·상세보기 버튼)
- **처리 흐름**:
  1. 알림 벨 클릭
  2. 최근 알림 목록을 팝오버로 표시(각 항목에 도메인·내용·상세보기 버튼 포함)
  3. 상세보기 버튼 클릭 시 해당 알림의 상세 화면으로 이동
- **인수 기준 (EARS)**:
  - (Event-driven) **WHEN** 사용자가 헤더 알림 벨을 클릭하면, 시스템은 최근 알림 목록을 팝오버로 표시해야 한다.
  - (Ubiquitous) 시스템은 팝오버의 각 알림 항목에 도메인·내용·상세보기 버튼을 표시해야 한다.
- **예외/오류 처리 (EARS · Unwanted Behaviour)**:
  - **IF** 표시할 알림이 없으면, **THEN** 시스템은 빈 상태 안내를 표시해야 한다.

### FEAT-COM-002 · 사용자 가이드

- **설명**: 헤더의 "?" 아이콘을 클릭하면 전용 화면으로 이동해 플랫폼 개요, 도메인 및 원칙, 역할별 수행 내용과 방법을 안내하는 사용자 가이드를 표시한다. 콘텐츠는 `docs/01_analyze/feature/user-guide-content.md`에 정의하며, Markdown 형식의 상세 문서로 작성한다.
- **관련 요구사항**: REQ-COM-002
- **입력**: 로그인 사용자의 "?" 아이콘 클릭
- **출력**: 사용자 가이드 전용 화면(개요/도메인 및 원칙/역할별 수행 내용과 방법)
- **처리 흐름**:
  1. 헤더 "?" 아이콘 클릭
  2. 사용자 가이드 전용 화면으로 이동(개요 → 도메인 및 원칙 → 역할별 수행 내용과 방법 순 구성)
- **인수 기준 (EARS)**:
  - (Event-driven) **WHEN** 사용자가 헤더의 "?" 아이콘을 클릭하면, 시스템은 사용자 가이드 전용 화면으로 이동해야 한다.
  - (Ubiquitous) 시스템은 사용자 가이드 화면에 개요·도메인 및 원칙·역할별 수행 내용과 방법 3개 섹션을 포함해야 한다.
- **예외/오류 처리 (EARS · Unwanted Behaviour)**:
  - **IF** 사용자 가이드 콘텐츠 로드에 실패하면, **THEN** 시스템은 오류 안내를 표시해야 한다.
