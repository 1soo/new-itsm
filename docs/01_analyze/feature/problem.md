## 3. 기능 상세

### FEAT-PRB-001 · 문제 등록

- **설명**: 사용자가 문제를 생성하고 표준 필드를 기입한다.
- **관련 요구사항**: REQ-PRB-001
- **입력**: 요약, 설명, 출처, 조사사유, 영향도, 긴급도, 구성요소
- **출력**: 생성된 문제(식별키·초기 상태 탐지)
- **처리 흐름**:
  1. 입력값 검증
  2. 문제 생성
- **인수 기준 (EARS)**:
  - (Event-driven) **WHEN** 필수 필드로 등록하면, 시스템은 문제를 생성해야 한다.
- **예외/오류 처리 (EARS · Unwanted Behaviour)**:
  - **IF** 요약이 누락되면, **THEN** 시스템은 400 오류를 반환해야 한다.

### FEAT-PRB-002 · 문제 프로세스 상태 관리

- **설명**: 문제를 6단계 프로세스에 따라 전이한다.
- **관련 요구사항**: REQ-PRB-002
- **입력**: 문제 ID, 목표 단계
- **출력**: 갱신된 상태
- **처리 흐름**:
  1. 전이 허용 여부 확인
  2. 전이·이력 기록
- **인수 기준 (EARS)**:
  - (Event-driven) **WHEN** 허용된 단계 전이를 요청하면, 시스템은 상태를 갱신해야 한다.
- **예외/오류 처리 (EARS · Unwanted Behaviour)**:
  - **IF** 허용되지 않은 전이면, **THEN** 시스템은 거부해야 한다.

### FEAT-PRB-003 · 우선순위 산정

- **설명**: 영향도·긴급도로 우선순위를 산정한다.
- **관련 요구사항**: REQ-PRB-003
- **입력**: 영향도, 긴급도
- **출력**: 산정된 우선순위
- **처리 흐름**:
  1. 영향도×긴급도 매트릭스 조회
  2. 우선순위 결정·표시
- **인수 기준 (EARS)**:
  - (Event-driven) **WHEN** 영향도·긴급도가 입력되면, 시스템은 우선순위를 산정해 표시해야 한다.
- **예외/오류 처리 (EARS · Unwanted Behaviour)**:
  - **IF** 영향도·긴급도 중 하나라도 없으면, **THEN** 시스템은 우선순위를 미산정으로 표시해야 한다.

### FEAT-PRB-004 · 근본 원인 분석(RCA)

- **설명**: 근본 원인과 5 Whys 사슬을 기록한다.
- **관련 요구사항**: REQ-PRB-004
- **입력**: 문제 ID, 근본원인, 5 Whys 단계, 근본원인 카테고리
- **출력**: 저장된 RCA
- **처리 흐름**:
  1. RCA 내용 저장
  2. 문제와 연결
- **인수 기준 (EARS)**:
  - (Event-driven) **WHEN** 근본 원인을 기록하면, 시스템은 이를 문제에 저장해야 한다.
  - (Ubiquitous) 시스템은 개인을 근본 원인으로 강제 지정하지 않아야 한다.
- **예외/오류 처리 (EARS · Unwanted Behaviour)**:
  - **IF** 문제 접근 권한이 없으면, **THEN** 시스템은 403을 반환해야 한다.

### FEAT-PRB-005 · 알려진 오류(KEDB)

- **설명**: 근본원인+워크어라운드를 알려진 오류로 등록하고 KEDB에서 검색한다.
- **관련 요구사항**: REQ-PRB-005
- **입력**: 근본원인, 워크어라운드, (조회)키워드
- **출력**: KE 레코드, 검색 결과
- **처리 흐름**:
  1. KE 레코드 생성·KEDB 등록
  2. 키워드 검색 지원
- **인수 기준 (EARS)**:
  - (Event-driven) **WHEN** KE 레코드를 생성하면, 시스템은 KEDB에 등록해야 한다.
  - (Event-driven) **WHEN** KEDB를 조회하면, 시스템은 매칭 결과를 반환해야 한다.
- **예외/오류 처리 (EARS · Unwanted Behaviour)**:
  - **IF** 검색 결과가 없으면, **THEN** 시스템은 빈 목록을 반환해야 한다.

### FEAT-PRB-006 · 워크어라운드 관리

- **설명**: 임시 대응책을 등록하고 지식베이스와 연결한다.
- **관련 요구사항**: REQ-PRB-006
- **입력**: 문제 ID, 워크어라운드 내용
- **출력**: 저장된 워크어라운드
- **처리 흐름**:
  1. 워크어라운드 저장
  2. 지식 기사 연결(선택)
- **인수 기준 (EARS)**:
  - (Event-driven) **WHEN** 워크어라운드를 등록하면, 시스템은 이를 문제에 저장해야 한다.
- **예외/오류 처리 (EARS · Unwanted Behaviour)**:
  - **IF** 내용이 비었으면, **THEN** 시스템은 저장을 거부해야 한다.

### FEAT-PRB-007 · 인시던트 연결

- **설명**: 문제에 인시던트를 연결한다.
- **관련 요구사항**: REQ-PRB-007
- **입력**: 문제 ID, 인시던트 ID
- **출력**: 양방향 링크
- **처리 흐름**:
  1. 대상 인시던트 확인
  2. 링크 저장
- **인수 기준 (EARS)**:
  - (Event-driven) **WHEN** 인시던트를 연결하면, 시스템은 양방향으로 링크·표시해야 한다.
- **예외/오류 처리 (EARS · Unwanted Behaviour)**:
  - **IF** 존재하지 않는 인시던트면, **THEN** 시스템은 400 오류를 반환해야 한다.

### FEAT-PRB-008 · 변경 연계

- **설명**: 문제 해결을 위한 변경 요청을 생성/연계한다.
- **관련 요구사항**: REQ-PRB-008
- **입력**: 문제 ID, (신규/기존)변경 ID
- **출력**: 문제-변경 링크
- **처리 흐름**:
  1. 변경 생성 또는 선택
  2. 링크 저장
- **인수 기준 (EARS)**:
  - (Event-driven) **WHEN** 변경을 생성/연계하면, 시스템은 문제와 변경을 링크해야 한다.
- **예외/오류 처리 (EARS · Unwanted Behaviour)**:
  - **IF** 존재하지 않는 변경에 연계하면, **THEN** 시스템은 400 오류를 반환해야 한다.

### FEAT-PRB-009 · 후속 조치 추적

- **설명**: 문제의 후속(시정) 조치 상태를 추적한다.
- **관련 요구사항**: REQ-PRB-009
- **입력**: 문제 ID, 조치 항목·상태
- **출력**: 조치 목록·상태
- **처리 흐름**:
  1. 조치 항목 등록
  2. 상태 갱신 추적
- **인수 기준 (EARS)**:
  - (Ubiquitous) 시스템은 후속 조치 항목의 상태를 추적할 수 있어야 한다.
- **예외/오류 처리 (EARS · Unwanted Behaviour)**:
  - **IF** 조치 항목 없이 조회하면, **THEN** 시스템은 빈 목록을 반환해야 한다.

### FEAT-PRB-010 · 문제 종료

- **설명**: 문제가 완전히 제거되면 종료한다.
- **관련 요구사항**: REQ-PRB-010
- **입력**: 문제 ID
- **출력**: 종료된 문제
- **처리 흐름**:
  1. 종료 조건 확인
  2. 상태=종료 전환·기록
- **인수 기준 (EARS)**:
  - (Event-driven) **WHEN** 문제를 종료하면, 시스템은 상태를 종료로 전환하고 기록해야 한다.
- **예외/오류 처리 (EARS · Unwanted Behaviour)**:
  - **IF** 미해결 후속 조치가 남아 있으면, **THEN** 시스템은 경고를 표시해야 한다.
