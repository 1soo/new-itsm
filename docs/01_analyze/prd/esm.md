# 요구사항 정의서 — 엔터프라이즈 서비스 관리 (Enterprise Service Management)

> 도메인: esm · 버전: 0.1 · 작성일: 2026-07-10

## 1. 개요

ITSM의 서비스 관리 원칙(서비스 카탈로그·셀프서비스 포털·워크플로우·지식 관리)을 IT 외 부서(HR·법무·시설·재무)로 확장한다. 부서별 서비스 카탈로그, HR 케이스 관리, 온보딩/오프보딩 체크리스트(부서 간 하위 작업 오케스트레이션)를 제공한다. (출처: `enterprise-service-management.md`)

## 2. 범위

### 포함 (In Scope)

- 부서별(HR/법무/시설/재무) 서비스 카탈로그 항목 관리(담당 부서 지정)
- 부서 포털에서 부서별 요청 제출·처리·상태 추적(기존 `service-request` 요청 처리 엔진을 재사용하되 담당 부서로 구분)
- HR 케이스 관리(접수→기록→조사→해결 4단계), 민감 정보 접근 제한(RBAC)
- 온보딩 체크리스트 생성 및 IT/시설/HR 등 관련 부서 하위 작업 자동 배정
- 오프보딩 체크리스트 생성 및 자산 회수·접근 권한 해제 하위 작업 배정(자산 도메인 연계)
- 체크리스트 하위 작업 진행 상태 추적 및 전체 완료 여부 표시
- 부서별 처리 건수·평균 처리시간·온보딩/오프보딩 완료율 리포팅

### 미포함 (Out of Scope)

- Slack/Teams 등 대화형 티케팅(conversational ticketing) 연동
- 직원 경험 플랫폼(EXP)의 설문·성과관리·인정(recognition) 기능
- 데이터 거버넌스(데이터 카탈로그·데이터 스튜어드십)
- 재무 시스템 연동 지출 승인·정산 자동화
- 조직 변화 관리 방법론(ADKAR·Lewin·Kotter 8단계 등) 도구화

## 4. 인수 기준 (Acceptance Criteria · EARS)

### REQ-ESM-001

- (Event-driven) **WHEN** 프로세스 오너가 담당 부서·이름·설명·양식 필드를 포함해 카탈로그 항목을 생성하면, 시스템은 이를 부서별 카탈로그에 등록해야 한다.
- (Unwanted) **IF** 담당 부서가 지정되지 않으면, **THEN** 시스템은 저장을 거부해야 한다.

### REQ-ESM-002

- (Event-driven) **WHEN** 요청자가 부서 포털에서 요청 유형을 선택해 제출하면, 시스템은 요청을 생성하고 담당 부서 큐로 라우팅해야 한다.
- (Event-driven) **WHEN** 담당 부서 처리자가 상태를 갱신하면, 시스템은 요청자에게 변경된 상태를 표시해야 한다.

### REQ-ESM-003

- (Ubiquitous) 시스템은 HR 케이스를 접수(Intake)·기록(Documentation)·조사(Investigation)·해결(Resolution) 상태로 관리해야 한다.
- (Event-driven) **WHEN** HR 담당자가 케이스 상태를 전이하면, 시스템은 전이 이력을 기록해야 한다.

### REQ-ESM-004

- (Unwanted) **IF** HR 역할이 아니고 SYSTEM_ADMIN도 아닌 사용자가 HR 케이스를 조회·수정하려 하면, **THEN** 시스템은 403을 반환해야 한다.

### REQ-ESM-005

- (Event-driven) **WHEN** 신규 입사 관련 요청 유형이 제출되면, 시스템은 온보딩 체크리스트를 생성하고 IT 계정 준비·시설 워크스페이스 준비·HR 서류 처리 등 하위 작업을 관련 부서에 배정해야 한다.
- (Unwanted) **IF** 온보딩 체크리스트 템플릿이 정의되지 않으면, **THEN** 시스템은 생성을 거부하고 오류를 표시해야 한다.

### REQ-ESM-006

- (Event-driven) **WHEN** 퇴사 관련 요청 유형이 제출되면, 시스템은 오프보딩 체크리스트를 생성하고 자산 회수·계정/접근 권한 해제 하위 작업을 관련 부서에 배정해야 한다.
- (Event-driven) **WHEN** 오프보딩 체크리스트가 자산 회수 하위 작업을 포함하면, 시스템은 이를 자산(ITAM) 도메인의 대상 자산과 연결해야 한다.

### REQ-ESM-007

- (Event-driven) **WHEN** 사용자가 체크리스트를 조회하면, 시스템은 각 하위 작업의 담당 부서·상태를 표시해야 한다.
- (Event-driven) **WHEN** 모든 하위 작업이 완료되면, 시스템은 체크리스트 전체 상태를 완료로 표시해야 한다.

### REQ-ESM-008

- (Ubiquitous) 시스템은 부서별 요청 처리 건수·평균 처리시간·온보딩/오프보딩 완료율을 집계·조회할 수 있어야 한다.
