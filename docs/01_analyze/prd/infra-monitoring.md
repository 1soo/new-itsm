## 1. 개요

인프라 자산(서버·네트워크 장비)의 가동률·성능 지표를 수동 입력 기반으로 기록·조회하고, 임계치 초과 시 알림하며, 팀 단위 용량 계획(처리 역량 대비 수요)을 관리한다. (출처: `it-operations.md` IT 인프라 관리, `it-operations-management.md` 용량 계획)

## 2. 범위

### 포함 (In Scope)

- 인프라 자산(기존 `asset` 도메인의 HW/네트워크 장비)에 대한 가동률·성능 지표(가동률%·CPU·메모리·응답시간 등) 수동 등록
- 등록된 지표의 시계열 대시보드 조회
- 지표 임계치 설정 및 임계치 초과 시 알림 생성
- 인프라 자산의 가동률 목표(SLA) 대비 실제 가동률 비교 표시
- 팀/서비스별 용량 계획(처리 역량·과거 처리량 대비 수요) 등록·조회
- 인프라 지표(가동률·성능·용량 활용률) 리포팅

### 미포함 (Out of Scope)

- 실시간 에이전트/모니터링 도구 연동(자동 수집)
- 자동 알림 발송(이메일·SMS·Slack 등 외부 채널 연동)
- 클라우드 제공자 API 연동, 자동 오토스케일링
- 실제 서버 성능 카운터 수집

## 4. 인수 기준 (Acceptance Criteria · EARS)

### REQ-IOM-001

- (Event-driven) **WHEN** 운영 담당자가 대상 인프라 자산·측정 시각·지표값(가동률/CPU/메모리/응답시간 등)을 입력하면, 시스템은 지표 레코드를 생성해야 한다.
- (Unwanted) **IF** 대상 자산 또는 지표값이 누락되면, **THEN** 시스템은 등록을 거부해야 한다.

### REQ-IOM-002

- (Event-driven) **WHEN** 사용자가 특정 자산의 지표 대시보드를 조회하면, 시스템은 등록된 지표를 측정 시각 순으로 정렬해 표시해야 한다.
- (Unwanted) **IF** 조회 기간 내 등록된 지표가 없으면, **THEN** 시스템은 빈 결과를 표시해야 한다.

### REQ-IOM-003

- (Ubiquitous) 시스템은 지표 항목별 임계치(상한/하한)를 설정·저장해야 한다.
- (Event-driven) **WHEN** 등록된 지표값이 임계치를 벗어나면, 시스템은 담당자에게 알림을 생성해야 한다.

### REQ-IOM-004

- (Event-driven) **WHEN** 운영 담당자가 팀/서비스의 처리 역량과 예상 수요를 입력하면, 시스템은 용량 계획을 생성·저장해야 한다.
- (Event-driven) **WHEN** 사용자가 용량 계획을 조회하면, 시스템은 역량 대비 수요 비율을 계산해 표시해야 한다.

### REQ-IOM-005

- (Ubiquitous) 시스템은 인프라 자산별 가동률 목표(SLA)를 저장해야 한다.
- (Event-driven) **WHEN** 사용자가 자산의 가동률 현황을 조회하면, 시스템은 목표 대비 실제 가동률(등록된 지표 기반 산출값)을 비교해 표시해야 한다.

### REQ-IOM-006

- (Ubiquitous) 시스템은 자산별·기간별 가동률·성능 지표 평균과 용량 활용률을 집계·조회할 수 있어야 한다.
