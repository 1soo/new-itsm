# 요구사항 정의서 — 공통 (Common)

> 도메인: common · 버전: 0.1 · 작성일: 2026-07-11

## 1. 개요

전 도메인이 공유하는 공통 레이아웃(헤더·사이드바·푸터 등, `docs/02_plan/screen/common.md` SCR-COM-*)에 대응하는, 특정 업무 도메인에 속하지 않는 공통 요구사항을 정의한다.

## 2. 범위

### 포함 (In Scope)

- 여러 업무 도메인을 가로지르는 공통 헤더/레이아웃 기능(예: 알림 팝오버, 사용자 가이드)

### 미포함 (Out of Scope)

- 개별 도메인에서 알림을 생성하는 로직(예: REQ-ITAM-004 만료 알림, REQ-IOM-003 임계치 알림) — 이미 각 도메인 문서에서 정의됨
- 이메일·SMS·Slack 등 외부 채널로의 알림 발송

## 3. 요구사항 목록

| ID | 유형 | 요구사항 | 우선순위 | 출처 |
|----|------|----------|----------|------|

## 4. 인수 기준 (Acceptance Criteria · EARS)

### REQ-COM-001

- (Event-driven) **WHEN** 사용자가 헤더 알림 벨을 클릭하면, 시스템은 최근 알림 목록을 팝오버로 표시해야 한다.
- (Ubiquitous) 시스템은 팝오버의 각 알림 항목에 도메인·내용·상세보기 버튼을 표시해야 한다.
- (Event-driven) **WHEN** 사용자가 알림 항목의 상세보기 버튼을 클릭하면, 시스템은 해당 알림의 상세 화면으로 이동해야 한다.

### REQ-COM-002

- (Event-driven) **WHEN** 사용자가 헤더의 "?" 아이콘을 클릭하면, 시스템은 사용자 가이드 전용 화면으로 이동해야 한다.
- (Ubiquitous) 사용자 가이드 화면은 개요, 도메인 및 원칙, 역할별 수행 내용과 방법의 3개 섹션으로 구성되어야 한다.
- (Ubiquitous) 시스템은 정의된 16개 역할 각각의 수행 내용과 방법을 역할별 수행 내용과 방법 섹션에서 확인할 수 있도록 제공해야 한다.
