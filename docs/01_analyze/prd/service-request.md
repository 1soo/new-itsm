# 요구사항 정의서 — 서비스 요청 관리 (Service Request Management)

> 도메인: service-request · 버전: 0.1 · 작성일: 2026-07-09

## 1. 개요

직원·고객의 일상적 요청(접근 권한, 장비, 정보 등)을 표준화된 워크플로우(제출→검증→분류/라우팅→승인→이행→검증→종료)로 처리한다. 서비스 데스크의 단일 접점, 서비스 카탈로그, 셀프서비스 포털, 승인, SLA를 제공한다. (출처: `service-request-management.md`)

## 2. 범위

### 포함 (In Scope)

- 서비스 카탈로그(요청 유형) 관리
- 셀프서비스 포털을 통한 요청 제출(동적 요청 양식)
- 요청 7단계 워크플로우(제출·검증·분류/라우팅·승인·이행·검증·종료)
- 큐(queue) 기반 분류(triage)·담당자 배정
- 승인 프로세스(필요 요청 유형에 한함)
- SLA 목표(응답·해결) 설정 및 준수 추적
- 요청 상태 추적·코멘트, CSAT(만족도) 확인
- 지식베이스 연계(요청 생성 시 관련 기사 노출로 티켓 차단)

### 미포함 (Out of Scope)

- 대화형 티켓팅(Slack/Teams 연동), 이메일→티켓 전환
- 가상 에이전트(AI 챗봇)
- 외부 고객 조직(Organizations) 관리

## 4. 인수 기준 (Acceptance Criteria · EARS)

### REQ-SRM-001

- (Event-driven) **WHEN** 프로세스 오너가 요청 유형(이름·설명·양식 필드·승인 필요 여부·SLA)을 생성하면, 시스템은 이를 서비스 카탈로그에 등록해야 한다.
- (Unwanted) **IF** 필수 항목(이름·양식)이 누락되면, **THEN** 시스템은 저장을 거부하고 오류를 반환해야 한다.

### REQ-SRM-002

- (Event-driven) **WHEN** 요청자가 요청 유형을 선택하고 양식을 작성해 제출하면, 시스템은 요청(work item)을 생성하고 접수번호를 반환해야 한다.
- (Unwanted) **IF** 양식 필수 필드가 미입력이면, **THEN** 시스템은 제출을 거부해야 한다.

### REQ-SRM-003

- (Event-driven) **WHEN** 요청이 제출되면, 시스템은 완전성·자격 요건을 검증해야 한다.
- (Unwanted) **IF** 요청자가 해당 요청의 자격이 없으면, **THEN** 시스템은 요청을 반려하고 사유를 표시해야 한다.

### REQ-SRM-004

- (Event-driven) **WHEN** 요청이 검증을 통과하면, 시스템은 분류 규칙에 따라 적절한 팀·큐로 라우팅해야 한다.
- (Unwanted) **IF** 매칭되는 큐가 없으면, **THEN** 시스템은 기본(미분류) 큐로 배정하고 표시해야 한다.

### REQ-SRM-005

- (State-driven) **WHILE** 요청이 승인 대기 상태인 동안, 시스템은 이행 단계로 진행하지 않아야 한다.
- (Event-driven) **WHEN** 승인자가 승인/반려하면, 시스템은 요청 상태를 갱신해야 한다.
- (Unwanted) **IF** 승인자가 반려하면, **THEN** 시스템은 요청을 반려 처리하고 요청자에게 사유를 통지해야 한다.

### REQ-SRM-006

- (Event-driven) **WHEN** 상담원이 요청을 배정받아 이행을 완료하면, 시스템은 상태를 "이행 완료"로 갱신해야 한다.
- (Unwanted) **IF** 권한 없는 사용자가 요청을 배정/이행하려 하면, **THEN** 시스템은 403을 반환해야 한다.

### REQ-SRM-007

- (Event-driven) **WHEN** 요청자 또는 상담원이 완료를 확인하면, 시스템은 요청을 종료 처리해야 한다.
- (Unwanted) **IF** 이미 종료된 요청을 다시 종료하려 하면, **THEN** 시스템은 중복 처리를 거부해야 한다.

### REQ-SRM-008

- (Ubiquitous) 시스템은 각 요청에 대해 요청 유형의 SLA 목표를 기준으로 경과 시간을 계산해야 한다.
- (Unwanted) **IF** SLA 목표 시간을 초과하면, **THEN** 시스템은 해당 요청을 SLA 위반으로 표시해야 한다.

### REQ-SRM-009

- (Event-driven) **WHEN** 요청자 또는 상담원이 코멘트를 등록하면, 시스템은 코멘트를 요청 이력에 추가하고 상대에게 표시해야 한다.

### REQ-SRM-010

- (Event-driven) **WHEN** 요청이 종료되면, 시스템은 요청자에게 CSAT 평가를 요청해야 한다.

### REQ-SRM-011

- (Event-driven) **WHEN** 요청자가 요청 유형을 선택하거나 제목을 입력하면, 시스템은 관련 지식 기사를 추천 노출해야 한다.

### REQ-SRM-012

- (Ubiquitous) 시스템은 CSAT·평균 응답시간·평균 해결시간·SLA 준수율 지표를 집계·조회할 수 있어야 한다.
