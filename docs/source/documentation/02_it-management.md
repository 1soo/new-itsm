# IT 관리 (IT Management)

> **IT 관리(IT Management)는 기업의 네트워크·하드웨어·소프트웨어 등 기술 시스템 전반을 관리·감독·유지하는 활동으로, 현대 기업 운영의 근간(foundation)이자 ITSM·ITOM·ITAM·ITIL·거버넌스를 아우르는 상위(umbrella) 관리 규율이다.**

---

## 📌 개요 (무엇을 & 왜 — IT 관리의 정의와 범위)

### 정의
현대 기업은 비즈니스 기능과 매끄럽게 정렬되는 기술 인프라를 필요로 한다. **IT 관리(IT Management)** 는 기업의 기술 시스템(네트워크, 하드웨어, 소프트웨어)을 **관리(administer)하고 모니터링(monitor)** 하는 활동이며, 취약점 없는 인프라를 유지하는 일은 이 시스템을 얼마나 효과적으로 관리하느냐에 달려 있다. 따라서 IT 관리는 성공적인 모든 기업의 **토대(keystone)** 다.

기업의 정보기술(IT) 시스템은 **하드웨어 + 소프트웨어 + 네트워크 통합(integration)** 으로 구성되며, 이는 다양한 역할과 기능에 걸쳐 비즈니스 운영을 지원한다. IT 관리는 이 시스템을 감독하는 일을 맡고, IT 구성요소의 **감사(auditing), 관리, 유지보수** 및 정보 시스템이 효과적·효율적으로 작동하도록 하는 해법 설계를 포함한다.

> 💡 **핵심 관계**: "IT 관리를 한 단계 더 발전시킨 것이 ITSM(IT Service Management)이다." IT 관리가 *시스템을 감독*한다면, ITSM은 *기업이 IT 서비스를 어떻게 제공하는지*를 포괄한다.

### 왜 중요한가 (Why It Matters)
기술이 어디에나 존재(ubiquitous)하게 되면서 IT 팀은 기업·직원·시장의 변화하는 요구에 맞추어 스스로 변모하고 있다.

- 과거 IT의 역할은 주로 **헬프데스크**(비밀번호 재설정, 네트워크/폴더 접근 제공)로 분류할 수 있었다.
- 오늘날 IT 팀은 사이버 범죄와 인공지능(AI)으로 가득 찬 **매우 빠르고 역동적인 산업** 환경에서 고도로 숙련되고 전략적으로 사고해야 한다.
- IT는 이제 **리더십 테이블에 자리를 차지**하며, 요청에 단순 반응하는 것을 넘어 비즈니스 의사결정을 이끈다.
- IT 팀의 역할이 **"비즈니스를 지원(supporting)"** 하는 것에서 **"비즈니스를 차별화(differentiating)"** 하는 것으로 전환되었다.

## 🧩 IT 관리의 핵심 기능(Functions)

Atlassian은 IT 관리의 기능을 **IT 관리자(IT Manager)의 책임**과 **IT 관리가 기여하는 역할**의 두 축으로 설명한다.

### IT 관리자의 4대 책임(Responsibilities)

| 기능 | 설명 |
|------|------|
| **IT 전략 기획 (IT Strategic Planning)** | 기업 요구에 맞는 가장 효과적인 소프트웨어·하드웨어·시스템 선정. 내부 협업 시스템의 온라인 유지 및 확장성 확보. 여기에는 **ITIL 서비스 전략**(더 높은 품질을 더 낮은 비용으로 서비스를 전달하는 ITSM 접근법)과 **업무 연속성 계획(business continuity planning)** 이 포함된다. |
| **네트워크 관리 (Network Management)** | 확장 가능하고 보호된 데이터·파일을 저장할 수 있는 네트워크의 구축·유지. 사용량 모니터링, 트래픽 트렌드 파악, 유연성 확보를 포함. |
| **위험 평가 및 완화 (Risk Assessment & Mitigation)** | 잠재적 위험 평가, 안전장치 배치, 특히 값비싼 사이버 공격에 대한 노출·위험 감소 조치. |
| **예산 관리 (Budget Management)** | 운영/자본 예산에 비용 배정, 혁신·개선을 위한 자금 재배분, 데이터 유출 등 예기치 못한 상황 대비 자원 편성. |

### IT 관리가 기여하는 역할(Role)

| 역할 | 설명 |
|------|------|
| **생산성 향상 (Improved Productivity)** | 플랫폼·프로세스·사람으로부터 생산성과 산출이 직접 발생. Jira Service Management 같은 서비스 관리 소프트웨어로 마찰을 줄여 더 빠르고 효율적인 상호작용과 생산적인 인력 확보. |
| **혁신과 적응성 (Innovation & Adaptability)** | 신흥 기술을 평가하고 비즈니스 목표·인프라에 맞는 혁신적 솔루션 배치. |
| **데이터 컴플라이언스 보장 (Ensuring Data Compliance)** | 데이터 프라이버시·보안 우려 증가에 따라 견고하고 안전한 데이터 수집·저장·활용 보장. IT가 데이터 저장·전송과 신기술 도입 및 컴플라이언스 확보를 담당. |
| **의사결정 (Decision-Making)** | 내부 시스템이 무엇을 지원할 수 있는지 안내, 프로젝트 확장을 위한 검증된 데이터 분석·기술 사양 제공. |

## 🗺️ IT 관리 지형도 (ITSM · ITOM · ITAM · ITIL · 거버넌스의 관계)

IT 관리는 **우산(umbrella) 규율**이며, 그 아래에 서비스·운영·자산·프레임워크·거버넌스가 자리한다.

```
┌─────────────────────────────────────────────────────────────┐
│                    IT 관리 (IT Management)                    │
│        기술 시스템 전반의 관리·감독·유지 (우산 규율)             │
│                                                               │
│  ┌────────────┐  ┌────────────┐  ┌────────────┐              │
│  │   ITSM     │  │   ITOM     │  │   ITAM     │              │
│  │ 서비스 제공 │  │ 운영·인프라 │  │ 자산·CMDB  │              │
│  └────────────┘  └────────────┘  └────────────┘              │
│        └── 인시던트 / 문제 / 변경 / 지식 / 서비스요청 관리       │
│                                                               │
│  ┌──────────────────────┐   ┌──────────────────────────┐    │
│  │ 방법론/프레임워크      │   │ IT 거버넌스 & 컴플라이언스 │    │
│  │ ITIL · COBIT · Agile │   │ (규제·정책·감사·리포팅)     │    │
│  └──────────────────────┘   └──────────────────────────┘    │
└─────────────────────────────────────────────────────────────┘
```

### 각 영역의 정의와 관계

| 영역 | 정의 | IT 관리와의 관계 |
|------|------|------------------|
| **ITSM** (IT Service Management) | IT 팀이 IT 서비스의 **엔드투엔드(end-to-end) 전달**을 관리하는 방식. 서비스의 설계·계획·전달을 관리하는 확립된 프로세스 집합. | IT 관리를 "한 단계 더 발전"시킨 것. IT 관리가 시스템을 감독하면, ITSM은 서비스를 *어떻게* 제공하는지를 다룬다. |
| **ITOM** (IT Operations Management) | IT 인프라·운영 관리. 인프라 관리, 네트워크 인프라, 서비스 매핑, 애플리케이션 의존성 매핑, 시스템 업그레이드 등. | IT 관리의 **운영 실행 계층**. 시스템이 안정적으로 돌아가도록 하는 일상 운영. |
| **ITAM** (IT Asset Management) | IT 자산의 전 생애주기 관리. **CMDB(구성관리 데이터베이스)**, 자산 추적, 하드웨어 자산 관리, 자산 생애주기 포함. | IT 관리의 **자산·구성 계층**. "무엇을 보유·관리하는가"를 추적. |
| **ITIL** (Information Technology Infrastructure Library) | IT 팀에 유연성과 안정성을 제공하면서 엔터프라이즈 IT 서비스의 계획·선정·전달·유지·전체 생애주기를 표준화하는 **프레임워크**. | IT 관리/ITSM을 구현하는 **대표 방법론**. 서비스 전략·전환·지속적 개선 등의 실무를 규정. |
| **IT 거버넌스 & 컴플라이언스** | 규제·정책·통제를 IT 활동에 적용하고 감사·리포팅으로 이를 입증하는 활동. | IT 관리의 **통제·책임 계층**. 데이터 컴플라이언스 보장은 IT 관리의 핵심 역할 중 하나. |

### 핵심 ITSM 프랙티스 간 관계 (ITIL 관점)

Atlassian은 문제 관리(Problem Management) 문서에서 프랙티스 간 상호작용을 다음과 같이 정리한다.

- **인시던트 관리 (Incident Management)**: IT 서비스의 계획되지 않은 중단 또는 품질 저하에 대응. ITIL은 인시던트를 "IT 서비스의 계획되지 않은 중단 또는 IT 서비스 품질의 저하"로 정의.
- **문제 관리 (Problem Management)**: 하나 이상의 인시던트의 **원인(cause) 또는 잠재적 원인**을 식별·관리. 근본 원인을 찾아 장기적 해결책 또는 우회책(workaround)을 구현하여 재발을 방지. Atlassian은 인시던트 관리와 문제 관리를 **점점 더 긴밀하게 통합**할 것을 권장.
- **변경 관리 (Change Management)**: 서비스 중단·다운타임 없이 변경을 계획·추적·릴리스하는 프로세스. 변경이 중단을 유발하면 인시던트·문제 관리 프로세스에서 분석.
- **지식 관리 (Knowledge Management)**: 해법·문서·인시던트 우회책의 저장소를 구축. 건강한 지식 관리는 더 빠른 인시던트 해결과 인시던트 감소를 가능케 함. (KCS = 지식 중심 서비스)
- **서비스 요청 관리 (Service Request Management)**: 접근 권한, 소프트웨어 개선, 정보 제공 등 사용자의 요청을 처리. ITIL V3(2007)부터 서비스 요청이 인시던트와 구분됨.

## 🏛️ IT 거버넌스 & 컴플라이언스 (Compliance 관리·리포팅 소프트웨어)

IT 관리의 "데이터 컴플라이언스 보장" 역할을 실무 도구로 뒷받침하는 두 가지 소프트웨어 범주가 있다. 둘은 **초점이 다르다**.

> 🎯 **핵심 구분**
> - **컴플라이언스 관리 소프트웨어** = 컴플라이언스를 *운영(operationalize)* 하는 **워크플로·거버넌스 도구** (일이 실제로 수행되게 만드는 것)
> - **컴플라이언스 리포팅 소프트웨어** = 컴플라이언스를 *추적·문서화·입증(demonstrate)* 하는 **추적·리포팅 도구** (감사·검토 대응)

### 1) 컴플라이언스 관리 소프트웨어 (Compliance Management Software)
> *"Operationalize compliance at scale — 규모에 맞게 컴플라이언스를 운영한다."*

**정의**: IT 팀이 서비스 운영 내에서 컴플라이언스 활동을 **조율(coordinate)·추적(track)·집행(enforce)** 하도록 돕는 도구. 감사·리포팅 도구라기보다 **워크플로·거버넌스 도구**에 가깝다. 운영 위험 관리, 서비스 거버넌스, 프로세스 표준화를 지원한다.

**서비스 워크플로의 일부로서의 컴플라이언스 관리 (4단계)** — 주기적 검토가 아니라 서비스 관리 프로세스 전반에서 지속적으로 처리:

| 단계 | 핵심 활동 |
|------|-----------|
| **① 컴플라이언스 요구사항 식별** | 규제 의무·업계 표준 추적, 내부 정책·통제 요구사항 식별, 서비스 전달·인프라 변경에 요구사항 매핑 |
| **② 컴플라이언스 책임 할당** | 컴플라이언스 과업의 명확한 소유자 지정, 서비스 워크플로에 과업 내재화, 책임을 명확히 문서화·가시화 |
| **③ 시정·교정 조치 관리** | 탐지부터 해결까지 이슈 추적, 취한 교정 조치 문서화, 향후 위험 감소·감사 지원을 위한 결과 기록 |
| **④ 서비스 운영 중 지속 유지** | 서비스 워크플로의 컴플라이언스 위험 모니터링, 인프라·프로세스 변경 검토, 시스템 진화에도 통제의 유효성 유지 |

**IT 팀을 위한 이점(Benefits)**:
- **운영 위험 감소**: 구조화된 워크플로로 서비스 전달 중 컴플라이언스 공백 방지 → 벌금·과징금 회피, 일관된 접근.
- **책임성·가시성 향상**: 중앙집중 시스템으로 소유권과 컴플라이언스 상태를 명확히 파악, 누락 방지.
- **컴플라이언스 이슈에 대한 빠른 대응**: 컴플라이언스 워크플로가 서비스 워크플로에 통합되어 상시 유지 → 인시던트 관리에 쓰는 시간 절감.
- **확장 가능한 거버넌스**: 표준화된 워크플로로 팀·서비스·요구사항 증가에도 거버넌스를 확장, 특정 팀·인원 의존도 감소.

**주요 기능(Key Features)**:

| 기능 | 왜 중요한가 |
|------|-------------|
| **워크플로 자동화** | 반복적 컴플라이언스 과업 자동화, 인적 오류 위험 감소, 실시간 알림 |
| **인시던트·변경 프로세스와의 통합** | 인프라 변경이 규제 요구사항을 충족하도록 보장, 컴플라이언스 워크플로를 서비스 관리와 연결 |
| **감사 대비 활동 추적 (Audit-ready tracking)** | 컴플라이언스 활동의 상세 기록 유지 → 감사 대비 |
| **IT 생태계 전반 통합** | 개발·운영 도구와 컴플라이언스 워크플로를 연결하여 지속적 컴플라이언스 유지 |

**시작하기(Getting Started)**: ① 서비스 프로세스 전반에 컴플라이언스 요구사항 매핑 → ② 컴플라이언스를 IT 운영과 연결하는 구조화된 플랫폼 도입 → ③ 컴플라이언스 워크플로를 서비스 워크플로에 직접 내재화.

### 2) 컴플라이언스 리포팅 소프트웨어 (Compliance Reporting Software)
> *"Benefits & Features — 컴플라이언스를 추적·문서화하고 감사에 대비한다."*

**정의**: 조직이 컴플라이언스 요구사항을 **추적**하고, 관련 활동을 **문서화**하며, 감사·내부 검토용 **리포트를 생성**하도록 돕는 시스템. 기록을 정돈해 진행 상황 모니터링, 공백 식별, 통제·프로세스 준수 입증을 가능케 한다.

**누가 사용하나(Who Uses)**:
- **컴플라이언스·리스크 팀**: 규제 의무 관리, 벌금·과징금 회피.
- **IT·엔지니어링 팀**: 소프트웨어 개발에 컴플라이언스·거버넌스를 내장, 팀 간 컴플라이언스 작업 조율.
- **재무·법무·운영 리더**: 거버넌스 감독과 리포팅 책임성 지원.

**작동 방식(How It Works)** — 팀이 이미 쓰는 도구(프로젝트 관리 플랫폼, 클라우드 스토리지, 커뮤니케이션 앱)를 연결해 데이터를 하나의 중앙 플랫폼으로 수집:
- **통합·동기화**: 정책·통제·증거·과업 업데이트를 자동으로 한곳에 모음.
- **워크플로 자동화**: 승인·검토·증거 수집 등을 규칙 기반으로 라우팅, 단계 완료 시 다음 단계 자동 트리거.
- **지속적 활동 로깅**: 모든 변경·승인·제출을 발생 즉시 기록하여 구조화된 감사 추적(audit trail) 구축.
- **실시간 모니터링**: 팀 전반의 컴플라이언스 상태를 추적하고 대시보드·알림으로 위험을 즉시 표면화.

**이점(Benefits)**: 조직 전반의 가시성 향상, 비용·수작업 감소, 컴플라이언스 위험에 대한 빠른 대응, 강화된 팀 간 협업, 확장 가능한 컴플라이언스 프로세스.

**주요 기능(Key Features)**: 워크플로 자동화 · 감사 추적 및 변경 추적 · 실시간 대시보드와 알림 · 기존 도구와의 통합 · 권한 및 접근 제어(access controls).

**전통적 방식 vs. 소프트웨어**:

| 접근 | 가장 적합 | 주요 단점 |
|------|-----------|-----------|
| **전통적 방식(수작업 추적)** | 더 작고 단순한 컴플라이언스 요구 | 가시성 제한, 확장 어려움 |
| **컴플라이언스 리포팅 소프트웨어** | 복잡하고 빠르게 변하는 환경 | 초기 설정·도입(adoption) 필요 |

## ✅ 모범 사례 (Best Practices)

### IT 관리자를 위한 모범 사례
- **교육과 새로운 기술 학습**: 독서, 웨비나, 신기술 강좌·훈련, 자격증 취득으로 업계 변화를 따라가기.
- **호기심 유지**: 오래된 문제를 새로운 방식으로 해결. 사례 연구를 검토하고 리더십에 제시할 유즈케이스 구축.
- **질문하기**: IT 전문가가 기업 운영의 모든 것을 알 순 없으나, 과제를 정확히 진단하고 해법을 도출할 수 있어야 함. 질문은 협업을 촉진하고 프로세스에 대한 엔드투엔드 이해를 보장.

### 컴플라이언스 관점의 모범 사례
- 주기적 검토가 아니라 **서비스 워크플로에 컴플라이언스를 상시 내재화**.
- **명확한 소유권 할당**과 문서화·가시화로 책임성 확보.
- **표준화된 워크플로**로 조직 전반의 공백 최소화 및 확장성 확보.
- **감사 대비 기록**을 중앙집중 위치에 유지.

## 📊 주요 지표(Metrics)

Atlassian IT 관리 문서는 "낮은 비용으로 고품질 서비스 전달"을 성과 기준으로 제시하며, ITSM/인시던트 관리 영역에서 다음 지표군을 다룬다.

| 범주 | 대표 지표 |
|------|-----------|
| **비용/가치** | 다운타임 비용(Gartner 기준 다수 조직에서 **시간당 30만 달러 초과**), 낮은 비용의 고품질 서비스 전달 |
| **신뢰성/가용성** | Reliability vs. Availability, MTTF(평균 고장 시간), Error Budget |
| **서비스 수준** | SLA vs. SLO vs. SLI, 첫 통화 해결률(First Call Resolution) |
| **인시던트** | 심각도 수준(Severity Levels), 해결 시간(Time to Resolution), 재발 인시던트 감소 |
| **컴플라이언스** | 감사 대비 상태, 컴플라이언스 이슈 대응 시간, 미해결 통제 공백 수 |

> 💡 컴플라이언스 리포팅 소프트웨어의 **실시간 대시보드**는 이러한 지표를 상시 가시화하여 "상태 리포트를 기다리지 않고" 현황을 파악하게 한다.

## 🛠️ 도구 & Jira Service Management 연계

세 문서 모두 **Jira Service Management(JSM)** 를 IT 관리·컴플라이언스의 핵심 실행 도구로 제시한다.

- **IT 관리 전반**: 워크플로에 맞게 프로젝트를 유연하게 구성, 팀 간 협업으로 워크플로 간소화. 진행 상황 추적, 타 도구 데이터 연결로 인시던트·요청·변경 대응 시간 단축. **노코드 폼·자동화·AI**로 수작업 제거. 팀 관리형 프로젝트(team-managed projects)로 가시성·책임성 확보.
- **컴플라이언스 관리**: 서비스 운영 전반에 컴플라이언스 워크플로를 표준화, 인시던트·변경 프로세스와 통합, 중앙집중 감사 추적 로그와 인시던트 관리 KPI 제공, 개발·운영 시스템과 통합.
- **컴플라이언스 리포팅**: 부서 간 컴플라이언스 워크플로 표준화·자동화, 요청·승인·프로세스 변경의 구조화된 추적, 개발·모니터링·협업 도구와 통합.

**함께 언급되는 방법론/프레임워크**: **Agile**(단계별 분할, 지속적 협업·개선), **COBIT**(비즈니스 리스크·관리 요구·기술 문제 간 간극을 메움), **ITIL**(계획·선정·전달·유지·생애주기 표준화).

## 🔗 다른 ITIL 프랙티스와의 관계

IT 관리는 아래 ITIL/ITSM 프랙티스를 포괄하며, 각 프랙티스는 서로 맞물려 전체 ITSM 전략을 형성한다.

- **서비스 전략(ITIL Service Strategy)**: IT 전략 기획의 근간. 더 높은 품질을 더 낮은 비용으로.
- **인시던트 관리 ↔ 문제 관리**: 서비스 복구(인시던트)와 근본 원인 제거(문제)를 점점 통합. "인시던트는 미래 신뢰성에 대한 계획되지 않은 투자."
- **변경 관리**: 무중단 변경 실행. 컴플라이언스 관리 소프트웨어는 변경 프로세스와 통합되어 인프라 변경의 규제 준수를 보장.
- **지식 관리 / KCS**: 해법 저장소로 더 빠른 해결·인시던트 감소.
- **서비스 요청 관리**: 사용자 요청 처리(인시던트와 구분).
- **거버넌스·컴플라이언스**: 데이터 컴플라이언스 보장은 IT 관리의 핵심 역할이며, 컴플라이언스 관리/리포팅 소프트웨어가 이를 서비스 워크플로에 내재화.

## 🧭 요약 및 시사점

1. **IT 관리는 우산 규율이다.** 시스템(네트워크·HW·SW)을 감독하는 IT 관리 위에서, ITSM은 "서비스를 어떻게 제공하는가"를, ITOM/ITAM은 운영·자산을, ITIL/COBIT/Agile은 방법론을, 거버넌스·컴플라이언스는 통제·책임을 담당한다.

2. **IT의 역할은 "지원"에서 "차별화"로 이동했다.** 헬프데스크에서 리더십 파트너로. 전략 기획·네트워크·위험·예산이 IT 관리자의 4대 책임이며, 생산성·혁신·데이터 컴플라이언스·의사결정이 그 기여점이다.

3. **컴플라이언스는 두 축으로 접근한다.** *관리 소프트웨어*는 컴플라이언스를 서비스 워크플로에 내재화해 **운영**하고, *리포팅 소프트웨어*는 그 활동을 **추적·문서화·입증**한다. 두 도구 모두 자동화·통합·감사 추적으로 위험을 낮추고 확장성을 높인다.

4. **주기적 검토에서 지속적 관리로.** 컴플라이언스는 이제 이벤트가 아니라 서비스 전달 전 과정에 상시 내재된 워크플로가 되어야 한다.

5. **실행 도구로서 JSM.** Atlassian은 노코드·자동화·AI·통합·중앙집중 추적을 통해 IT 관리와 컴플라이언스를 하나의 플랫폼으로 운영할 것을 권장한다.
