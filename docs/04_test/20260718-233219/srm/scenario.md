### TC-GF6-001 · Content 설정 팝업 개별 미리보기 완전 제거
- 근거: @docs/02_plan/screen/service-request.md (5.4절 "개별 미리보기 제거"), @docs/03_develop/plan/service-request.md (6차 요구사항 1번)
- 절차:
  1. text 컴포넌트의 Content 설정 팝업을 열고 하단까지 스크롤 → "미리보기" 섹션이 더 이상 없는지 확인(입력 위젯만 존재)
  2. 캔버스 카드 자체 렌더링(카드가 실제 컴포넌트 모습을 보여주는 것)은 그대로 정상 동작하는지 확인

- 기대 결과: Content 설정 팝업에 개별 미리보기 없음, 캔버스 카드·A1 축소 미리보기는 정상 동작(회귀 없음)

### TC-GF6-002 · 라벨 팝업 "테두리 없음" 체크 시 테두리색 UI 숨김·값 보존
- 근거: @docs/02_plan/screen/service-request.md (5.8절 "체크 시 테두리색 입력 UI 자체를 조건부 렌더링"), @docs/03_develop/plan/service-request.md (6차 요구사항 4번)
- 절차:
  1. 라벨 생성 팝업에서 테두리색을 임의 값(예: `#dc2626`)으로 지정
  2. "테두리 없음" 체크 → 테두리색 입력 UI가 사라지는지 확인
  3. 체크 해제 → 테두리색 입력 UI가 다시 나타나고 값이 `#dc2626`으로 그대로 복귀하는지 확인(초기화되지 않음)
- 기대 결과: 체크 시 UI 숨김, 해제 시 UI 재노출+값 보존

### TC-GF6-003 · 5차 결함 수정 — showBorder=false여도 라벨 텍스트는 표시
- 근거: @docs/02_plan/screen/service-request.md (5.8절 "오버레이 렌더링 여부와 테두리 스타일 적용 여부를 분리"), @docs/03_develop/plan/service-request.md (6차 요구사항 3번, dev-lead 지정 회귀 확인 포인트)
- 절차:
  1. 라벨을 "테두리 없음" 체크한 상태로 생성
  2. 이 라벨을 컴포넌트 1개 이상에 지정 → 캔버스에 테두리 선은 안 보이지만 라벨 텍스트는 표시되는지 확인(직전 5차에서는 오버레이 전체가 사라져 텍스트도 함께 안 보였던 결함)
  3. 그 라벨의 참조를 0개로 만듦(컴포넌트에서 라벨 해제) → 오버레이 자체(텍스트 포함)가 완전히 사라지는지 확인
- 기대 결과: showBorder=false+참조 1개 이상이면 텍스트만 표시(테두리 없음), 참조 0개면 텍스트도 없음

### TC-GF6-004 · 라벨 텍스트 legend 스타일(테두리 선 위에 걸침)
- 근거: @docs/02_plan/screen/service-request.md (5.8절 "fieldset/legend처럼 테두리 선 위에 걸치는 스타일"), @docs/03_develop/plan/service-request.md (6차 요구사항 2번)
- 절차:
  1. showBorder=true인 라벨(참조 1개 이상)의 텍스트가 오버레이 박스 안쪽 좌측 상단이 아니라 테두리 선 위에 걸쳐 보이는지(불투명 배경으로 그 아래 테두리 선이 자연스럽게 가려지는지) 확인
  2. TC-GF6-003의 showBorder=false 라벨 텍스트도 (테두리가 있었다면 걸쳤을) 동일한 위치에 표시되는지 확인
- 기대 결과: showBorder 여부와 무관하게 텍스트가 legend 스타일 위치(테두리 선 위)에 표시

### TC-GF6-R01 · 이전 차수 항목 회귀 확인
- 근거: @docs/03_develop/plan/service-request.md (6차 완료 기준 "기존 SRM 회귀 없음")

  6. 9방향 정렬 위젯 재확인
  7. 칩 스트립 스타일(showBorder 무관 항상 유지) 재확인
