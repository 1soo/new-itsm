# 통합 테스트 시나리오 — asset (배치1: 전이버튼 라벨, 유지보수 요청 2026-07-16)

## 사전 조건
- 빌드 테스트 통과(service-request 시나리오에서 1회 수행한 결과 공유)
- 계정: am@itsm.local(ASSET_MANAGER) — 공통 비밀번호 `Admin@1234`
- ASSET은 타임라인 actor 변경 대상 아님(FE 단독, 버튼 라벨만). "폐기" 버튼은 도착 상태명과 동작 동사형이 동일해 변경 대상 아님

## 시나리오

### TC-ITAM-001 · 빌드 테스트(공유)
- 근거: 통합테스트 선행 항목
- 절차: service-request 시나리오 TC-SRM-001 결과 참조(동일 빌드)
- 기대 결과: 오류 없이 성공

### TC-ITAM-002 · 생애주기 단계 전이 버튼 라벨(동작 동사형) 표시
- 근거: @docs/02_plan/screen/asset.md SCR-ITAM-003 "전이 버튼 라벨" 표
- 전제: am@itsm.local로 신규 자산 등록
- 절차:
  1. PLANNING 버튼 텍스트 확인("계획 수립") 후 클릭
  2. PROCUREMENT 버튼 텍스트 확인("구매 진행") 후 클릭
  3. OPERATION 버튼 텍스트 확인("운영 전환") 후 클릭
  4. MAINTENANCE 버튼 텍스트 확인("유지보수 전환") 후 클릭
- 기대 결과: 각 단계 버튼 텍스트가 도착 상태명이 아닌 표의 동작 동사형 라벨과 정확히 일치

### TC-ITAM-003 · "폐기" 버튼 라벨 변경 없음 확인(회귀)
- 근거: @docs/02_plan/screen/asset.md SCR-ITAM-003 "폐기 버튼" 설명
- 절차: 자산 상세에서 "폐기" 버튼 텍스트 확인 후 클릭(확인 다이얼로그 진행)
- 기대 결과: 버튼 라벨이 기존과 동일하게 "폐기"로 표시(도착 상태명과 동작 동사형이 같아 변경 없음)

### TC-ITAM-004 · 전이 완료 토스트 문구 회귀 확인(도착 상태명 유지)
- 근거: @docs/02_plan/screen/asset.md SCR-ITAM-003 "전이 버튼 라벨" 절
- 절차: TC-ITAM-002 전이 완료 토스트 문구 확인
- 기대 결과: 토스트는 버튼 라벨이 아닌 기존 도착 상태명(`statusLabel`)을 그대로 사용

### TC-ITAM-005 · 타임라인/이력 영향 없음 회귀 확인
- 근거: @docs/03_develop/plan/asset.md "ASSET은 타임라인 actor 대상 아님"
- 절차: 자산 상세 화면 좌측 생애주기 이력 영역 확인
- 기대 결과: 이번 변경으로 인한 이력 표시 변경이 없음
