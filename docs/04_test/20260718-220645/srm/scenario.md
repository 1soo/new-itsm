# 통합 테스트 시나리오 — srm (그리드 폼 빌더 5차: 라벨 경계 테두리 보정)

## 사전 조건
- 빌드 테스트 통과(FE `npm run build`. BE/DB 변경 없음 — 계획서 명시, dev-be/dev-db 소집 없었음)
- Process Owner 권한 계정(`po@itsm.local` / `Admin@1234`, SCR-SRM-007 Form 설정)
- 신규 카탈로그 항목을 생성해 격리된 캔버스에서 검증한다.
- playwright는 새 창(새 browser context)에서 수행, storage 초기화

## 시나리오

### TC-GF5-B01 · 빌드 테스트
- 근거: 통합 테스트 공통 선행 항목
- 절차:
  1. `source/frontend`에서 `npm run build` 실행(BE/DB 변경 없어 BE 빌드는 생략)
- 기대 결과: 오류 없이 빌드 성공

### TC-GF5-001 · 라벨 1개 참조 시에도 테두리 표시(4차 대비 기준 변경)
- 근거: @docs/02_plan/screen/service-request.md (5.8절 "적용 컴포넌트 수... 1개 이상에 지정되고 showBorder=true(기본값)이면 테두리를 그린다"), @docs/03_develop/plan/service-request.md (5차 요구사항 3번)
- 절차:
  1. 라벨 하나를 생성(기본값, "테두리 없음" 체크 안 함)
  2. 컴포넌트 1개에만 이 라벨을 지정
  3. 캔버스에 그 컴포넌트 영역에 테두리가 표시되는지 확인(직전 4차에서는 2개 미만이라 표시되지 않았던 케이스)
- 기대 결과: 참조 1개만으로도 테두리 표시

### TC-GF5-002 · 테두리 4px 바깥 확장
- 근거: @docs/02_plan/screen/service-request.md (5.8절 "각 변 2px씩(전체 4px) 확장한 사각형"), @docs/03_develop/plan/service-request.md (5차 요구사항 1번)
- 절차:
  1. TC-GF5-001의 테두리가 컴포넌트 셀 경계보다 살짝 바깥쪽에 그려지는지 확인(그리드 배치 좌표 자체는 셀 경계 그대로 유지되는지, 즉 컴포넌트 카드 크기·위치는 변하지 않고 시각적 테두리만 확장되는지)
- 기대 결과: 테두리가 셀 경계 바깥으로 시각적으로 확장(약 4px), 실제 그리드 좌표는 불변

### TC-GF5-003 · 테두리 좌측 상단 라벨 텍스트 표시
- 근거: @docs/02_plan/screen/service-request.md (5.8절 "좌측 상단에 라벨 텍스트를 textColor로 절대 위치 표시"), @docs/03_develop/plan/service-request.md (5차 요구사항 2번)
- 절차:
  1. TC-GF5-001의 테두리 오버레이 좌측 상단에 라벨 텍스트가 표시되는지, 색상이 해당 라벨의 textColor와 일치하는지 확인(칩 스트립의 텍스트 표시와는 별개로 추가 표시되는지)
- 기대 결과: 테두리 좌측 상단에 textColor 색상의 라벨 텍스트 표시

### TC-GF5-004 · "테두리 없음"(showBorder) 체크박스
- 근거: @docs/02_plan/screen/service-request.md (5.8절 "showBorder... 체크 시 캔버스 그룹 테두리만 숨기고"), @docs/03_develop/plan/service-request.md (5차 요구사항 4번)
- 절차:
  1. 라벨 생성/수정 팝업에 "테두리 없음" 체크박스가 있는지 확인
  2. 체크한 상태로 저장 → 이 라벨을 컴포넌트 여러 개(2개 이상)에 지정해도 캔버스에 테두리가 전혀 그려지지 않는지 확인
  3. 체크 해제(기본값)로 되돌려 저장 → 참조 1개 이상이면 다시 정상 표시되는지 확인
  4. 칩을 클릭해 재수정 팝업을 열었을 때 "테두리 없음" 체크 상태가 프리필되는지 확인
- 기대 결과: 체크 시 참조 개수 무관 테두리 없음, 해제 시 1개 이상이면 정상 표시, 칩 재오픈 시 체크 상태 프리필

### TC-GF5-005 · showBorder=false여도 칩 스트립 스타일 유지
- 근거: @docs/02_plan/screen/service-request.md (5.8절 "칩 배지의 테두리·틴트 배경은 showBorder 값과 무관하게 항상 표시"), @docs/03_develop/plan/service-request.md (5차 요구사항 4번)
- 절차:
  1. TC-GF5-004에서 "테두리 없음" 체크한 라벨의 칩이 팔레트 상단 스트립에서 테두리+틴트 배경 스타일 그대로 표시되는지(흐려지거나 스타일이 사라지지 않는지) 확인
- 기대 결과: 캔버스 테두리만 숨겨지고 칩 스트립 스타일은 그대로 유지

### TC-GF5-006 · 이동/리사이즈/라벨 재지정/showBorder 토글 시 즉시 재계산
- 근거: @docs/02_plan/screen/service-request.md (5.8절 "즉시 재계산한다"), @docs/03_develop/plan/service-request.md (완료 기준)
- 절차:
  1. 라벨이 지정된 컴포넌트를 다른 셀로 이동 → 테두리(및 텍스트 위치)가 즉시 새 위치로 재계산되는지 확인
  2. 컴포넌트를 리사이즈 → 테두리 크기가 즉시 재계산되는지 확인
  3. 다른 컴포넌트에 같은 라벨을 추가 지정 → 테두리 범위가 즉시 확장되는지 확인
  4. showBorder를 토글(체크→해제) → 테두리가 즉시 나타나는지 확인
- 기대 결과: 네 가지 조작 모두 테두리가 즉시 재계산·반영됨

### TC-GF5-007 · 오버레이는 캔버스 전용(외부 미노출, 회귀 없음)
- 근거: @docs/02_plan/screen/service-request.md (5.8절 "Content 설정 개별 미리보기·A1 축소 미리보기·요청 제출 폼에는 나타나지 않는다")
- 절차:
  1. 라벨이 지정된 컴포넌트의 Content 설정 팝업 내 개별 미리보기에 테두리 오버레이가 나타나지 않는지 확인
  2. 카탈로그 항목 저장 후 CatalogManagePage의 A1 축소 미리보기에 테두리가 나타나지 않는지 확인
  3. 요청 제출 폼(SCR-SRM-002)에도 나타나지 않는지 확인
- 기대 결과: 캔버스 외 3곳 모두 오버레이 미표시(회귀 없음)

### TC-GF5-R01 · 4차까지 항목 회귀 확인
