# 통합 테스트 시나리오 — srm (그리드 폼 빌더 3차 정정 재검증: 3분할 폐기 → 캔버스=미리보기 통합)

## 사전 조건
- 빌드 테스트 통과(BE `./gradlew build`, FE `npm run build`)
- Process Owner 권한 계정(`po@itsm.local` / `Admin@1234`, SCR-SRM-007 Form 설정)
- 직전 통합 테스트(`docs/04_test/20260718-182813`)에서 생성한 카탈로그 항목 재사용 또는 신규 생성
- 이번 재검증은 "3분할 폐기 → 캔버스=미리보기 통합" 변경분만 다룬다. guide-text/guide-file 분리, radio/checkbox 배치방향·여백, placeholder, date/file 롤백 등은 코드 변경이 없어 스킵한다(직전 통합 테스트 `20260718-182813`에서 이미 PASS 확인).
- playwright는 매 TC마다 새 창(새 browser context)에서 수행, storage 초기화

## 시나리오

### TC-GF3R-B01 · 빌드 테스트
- 근거: 통합 테스트 공통 선행 항목
- 절차:
  1. `source/backend`에서 `./gradlew build` 실행
  2. `source/frontend`에서 `npm run build` 실행
- 기대 결과: 둘 다 오류 없이 빌드 성공

### TC-GF3R-001 · Form 설정 팝업 2분할(팔레트/캔버스)·미리보기 패널 없음
- 근거: @docs/02_plan/screen/service-request.md (5.6절 "좌 팔레트 / 우 8×n 그리드 캔버스 2분할... 별도 우측 미리보기 패널은 폐기")
- 전제: Process Owner로 `/admin/service-catalog` "Form 설정" 팝업 오픈
- 절차:
  1. 팝업 내부에 팔레트/캔버스 2개 영역만 있는지 확인(별도 "미리보기" 라벨의 우측 패널 없음)
- 기대 결과: 팔레트+캔버스 2분할, 전체 레이아웃 미리보기 패널 없음

### TC-GF3R-002 · 캔버스 카드 = 실제 렌더링(아이콘·캡션 없음)
- 근거: @docs/02_plan/screen/service-request.md (5.2절 "카드가 실제 렌더링될 모습을 그대로 보여준다... 유형 식별 아이콘 제거")
- 절차:
  1. text/select/radio/guide-text/date 등 여러 유형을 캔버스에 배치
  2. 각 카드 안에 컴포넌트 유형 아이콘·캡션 텍스트(빈 점선 placeholder)가 아니라 실제 input/select/radio 옵션 목록/안내 텍스트 등이 그대로 렌더링되는지 확인
- 기대 결과: 카드 콘텐츠 영역에 실제 렌더링 모습 표시, 유형 아이콘·캡션 텍스트 없음

### TC-GF3R-003 · 캔버스 카드 렌더링은 순수 시각적(상호작용 불가, 드래그만 동작)
- 근거: @docs/02_plan/screen/service-request.md (5.2절 "순수 시각적 표현이며 값 입력·상호작용은 되지 않는다... 카드 자체가 여전히 드래그 이동 대상")
- 절차:
  1. 캔버스에 렌더링된 text input을 클릭 후 타이핑 시도 → 값이 바뀌지 않는지 확인
  2. 캔버스에 렌더링된 select를 클릭 → 드롭다운이 열리지 않고(또는 열려도 선택이 반영 안 됨) 카드가 드래그 가능한지 확인
  3. 카드를 클릭+드래그해 다른 빈 셀로 이동 → 정상 이동되는지 확인
- 기대 결과: 렌더링된 컨트롤과 상호작용 불가(pointer-events-none), 카드 드래그 이동은 정상 동작

### TC-GF3R-004 · 배치·리사이즈·Content 설정 변경이 카드 렌더링에 즉시 반영
- 근거: @docs/02_plan/screen/service-request.md (5.6절 "캔버스 카드의 렌더링은 편집 중(적용 전)인 components 상태를 실시간 반영")
- 절차:
  1. text 컴포넌트의 Content 설정에서 placeholder를 변경 → 캔버스 카드 안 렌더링(placeholder 텍스트)이 즉시 갱신되는지 확인
  2. radio 컴포넌트의 배치 방향(가로→세로)을 변경 → 캔버스 카드 안 옵션 배치가 즉시 갱신되는지 확인
  3. 컴포넌트를 리사이즈(1×1→2×1) → 카드 콘텐츠 렌더링 영역도 커진 크기에 맞게 갱신되는지 확인
- 기대 결과: Content 설정·리사이즈 변경사항이 적용 버튼 클릭 전에도 캔버스 카드에 즉시 반영

### TC-GF3R-005 · 카드 hover 시 설정/삭제 버튼 오버레이 정상 동작
- 근거: @docs/02_plan/screen/service-request.md (5.2절 "설정·삭제 액션 버튼은 렌더링된 콘텐츠 위에 오버레이 형태로 띄운다... 액션 버튼 클릭은 드래그 시작과 분리 처리")
- 절차:
  1. 카드에 hover → 우측 상단에 설정(Settings2)·삭제(Trash2) 버튼이 반투명 배경으로 오버레이되어 나타나는지 확인
  2. 설정 버튼 클릭 → Content 설정 팝오버가 정상적으로 열리는지(카드 드래그로 오인되지 않는지) 확인
  3. 삭제 버튼 클릭 → 카드가 정상 삭제되는지(드래그로 오인되지 않는지) 확인
- 기대 결과: hover 시 버튼 노출, 클릭 시 각각 팝오버 오픈/삭제가 정상 동작(드래그 미발생)

### TC-GF3R-006 · Content 설정 팝업 내 개별 미리보기는 여전히 상호작용 가능
- 근거: @docs/02_plan/screen/service-request.md (5.4절 "Content 설정 팝업 안 미리보기는 상호작용 가능해 실제 값 입력을 테스트할 수 있다")
- 절차:
  1. text 컴포넌트 Content 설정 팝업을 열고 하단 개별 미리보기 영역에 값을 입력 → 정상 입력되는지 확인(캔버스 카드와 달리 상호작용 가능)
  2. 이 미리보기 입력값이 "기본값" 필드나 캔버스 카드 렌더링에는 반영되지 않는지 확인(로컬 상태 전용)
- 기대 결과: Content 설정 팝업 내 미리보기는 값 입력 가능, 캔버스 카드 렌더링과 명확히 구분됨

### TC-GF3R-007 · 기존 SRM 회귀 확인(요청 처리함 등 무관 기능)
- 근거: 직전 통합 테스트 이력(`docs/04_test/20260718-182813/srm/result/srm.md`)
- 절차:
  1. 요청 처리함(SCR-SRM-004) 카테고리별 건수·미분류 마지막 고정 재확인
  2. 카탈로그 항목 저장 후 요청 제출 화면(SCR-SRM-002)에서 실제 렌더링(placeholder/guide-text/guide-file/date-file 박스/radio 배치) 회귀 없음 재확인
- 기대 결과: 이번 유지보수와 무관한 기능 전부 정상 동작(회귀 없음)
