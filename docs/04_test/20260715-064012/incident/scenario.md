# 통합 테스트 시나리오 — incident (런타임 업그레이드 후 기능 회귀)

## 사전 조건
- auth 도메인 로그인 성공 전제. 신규 기능 검증 아님(기존 동작 회귀 확인).

## 시나리오

### TC-INC-101 · 인시던트 목록 조회(빈 목록)
- 근거: @docs/01_analyze/feature/incident.md (SCR-INC-001)
- 전제: incident 테이블은 트랜잭션 데이터 미시딩(설계상 정상)
- 절차:
  1. `/incidents` 접근
- 기대 결과: "인시던트가 없습니다" 정상 렌더링, 콘솔 에러 없음

### TC-INC-102 · 인시던트 등록 및 상세 조회
- 근거: @docs/01_analyze/feature/incident.md (SCR-INC-002/003, 식별키 INC-YYYY-####)
- 전제: TC-INC-101 완료
- 절차:
  1. "인시던트 등록" → 요약/설명/심각도(SEV3) 입력 → 등록
  2. 상세 화면 렌더링 확인(타임라인, MTTD/MTTA/MTTR 지표 영역)
- 기대 결과: INC-2026-0001 생성, 상세 화면에 등록 타임라인 이벤트 자동 기록, 콘솔 에러 없음
