# 통합 테스트 시나리오 — esm (런타임 업그레이드 후 기능 회귀)

## 사전 조건
- auth 도메인 로그인 성공 전제. 신규 기능 검증 아님(기존 동작 회귀 확인).

## 시나리오

### TC-ESM-101 · 부서 서비스 포털 카탈로그 조회 및 요청 제출
- 근거: @docs/01_analyze/feature/esm.md (부서별 카탈로그 5건, 동적 양식)
- 전제: 로그인 상태, 부서별 카탈로그 시드(인사/법무/시설/재무) 존재
- 절차:
  1. `/esm/portal` 접근, 부서 필터(전체/인사/법무/시설/재무) 및 카탈로그 5건 노출 확인
  2. "좌석 배정 요청"(시설) 선택 → 동적 양식(희망 층: 3F/4F/5F) 확인 → "3F" 선택 후 제출
- 기대 결과: 요청 생성되어 상세 화면(`/esm/requests/{id}`)으로 이동, 콘솔 에러 없음
